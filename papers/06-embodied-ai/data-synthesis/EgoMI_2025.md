## 一句话总结

EgoMI 用一套 VR 头显 + 双手柄的"无机器人"采集设备，同步录制人类操作时**头部主动视线轨迹**与**双手末端轨迹**，再配合训练-free 的关键帧记忆机制 SPARKS，把预训练 VLA（π0）两阶段微调成一个 29 维相对笛卡尔动作策略；在**不采集任何本体机器人数据、不做任何视觉增强/对齐/重渲染**的前提下 zero-shot 迁移到轮式半人形机器人的"主动头 + 双臂"，在需大范围搜索与空间记忆的桌面/货架/记忆任务上分别达到 **36/40、35/40、31/40**，显著超过无头相机的 20D 基线（29/40、0/40）与单帧无记忆基线（21/40）。

## 一、问题与动机

从人类示范学习是机器人技能获取的一条可扩展路径，但**具身鸿沟（embodiment gap）**始终是核心障碍。作者观察到一个被大多数系统忽视的现象：人在操作时会**主动协调头部与手部运动**——不断重新定位视点、用注视去搜索并锁定任务相关物体、通过侧头消解遮挡。这类"任务驱动"的头部运动会带来剧烈且大幅的视点变化，而**固定的外部相机或静止的头部相机无法复现这一行为**，从而导致严重的分布偏移，退化策略性能。

现有缩小具身鸿沟的思路各有短板：把动作抽象到末端执行器空间（UMI）会丢弃头部/身体信息；wrist-mounted 相机或顶视投影只能局部缓解；而对于**需要"搜索/张望"、需要跨大工作空间**的任务，策略无法复现被示范的头部行为。此外，标准策略普遍缺乏**对历史观测的持久空间记忆**，在头部快速转动时会发生 context loss（关键上下文丢失）。

因此本文主张：**只要忠实捕获并重定向头 + 手的第一人称运动与观测，就足以弥合具身鸿沟**——无需视觉增强、in-painting 或视点重渲染。

## 二、核心方法

EgoMI 由四部分组成：无机器人采集硬件、示范数据的坐标重定向与清洗、29D 动作/状态表示 + 相对参数化、以及 SPARKS 关键帧记忆 + 两阶段微调的策略训练。

### 2.1 采集硬件与"视线代理"

- 核心是 **Meta Quest 3S** VR 头显，提供操作者头部与双手柄的 6-DoF 追踪；头显上方**刚性固连一台 ZED 2i 相机**录制随头运动的第一人称视频。
- 每个手柄做了两处定制：（1）安装点承载 wrist 相机，视角与机器人腕部相机对齐；（2）标准法兰接口可直接接一只现货夹爪（Robotiq 2F-85）。手柄扳机被映射为对夹爪的实时 drive-by-wire 控制，从而**采集真实的连续夹爪开合动作**。
- Table I 强调 EgoMI 是唯一同时满足"无机器人采集 + 头姿追踪 + 真实夹爪动作 + 全身重定向"的系统。
- **视线代理**：因缺乏眼动追踪，作者在 passthrough 视野中心叠加一个固定十字准星（reticle），要求操作者用它对准操作目标与放置点，从而把**头部朝向当作视觉注意中心的可靠代理**（"look-then-reach"）。这把任务相关特征驱动到观测空间正中，形成 object-centric 表征；作者指出**去掉 reticle 后训练出的策略常常完全失败**。

用大白话说：这套设备就是给人戴上"机器人的眼睛和手的替身"——录下来的头怎么转、手怎么动、夹爪怎么捏，直接就是机器人能照着做的指令，不需要真机器人在场采数据。

### 2.2 数据重定向与清洗

高吞吐转换流水线会做视频时序检查、SE(3) 平移/旋转增量的平滑性阈值来过滤坏数据。关键一步是把 VR 世界坐标系下的原始轨迹**重定向到机器人的规范坐标系**，以缩小本体感受鸿沟。具体地，对每条 episode 用第一帧对齐：

- **前向估计**：取双末端第一帧的前向轴 $z_L(t)=R_V^L(t)e_z,\ z_R(t)=R_V^R(t)e_z$（$e_z=[0,0,1]^\top$），投影到 xy 平面归一化后取偏航角的**圆均值** $\theta$，定义再定向为绕 z 轴的偏航旋转 $R_z(\theta)$。
- **基座原点**取第一帧头部的 x-y 位置 $t_B=[p_{H,x},\,p_{H,y},\,0]^\top$。
- **VR→base 变换** $T_V^B=\begin{bmatrix}R_z(\theta)&t_B\\0&1\end{bmatrix}$，配合标定后的法兰/TCP 偏移，得到 base 系下的末端与头部位姿：

$$T_B^{L/R}(t)=T_B^V\,T_V^{L/R}(t)\,T_{\text{flange}}^{L/R}\,T_{\text{flange}\to\text{TCP}},\qquad T_B^{H}(t)=T_B^V\,T_V^{H}(t)\,T_{\text{head}\to\text{cam}}$$

用大白话说：VR 采集时的"世界"是任意摆放的，这一步把每段示范都摆正——让第一帧头在原点、身体朝向对齐机器人正前方，于是所有轨迹都变成机器人坐标系里能直接复现的量。

### 2.3 29D 动作/状态表示与相对参数化

- **数据集绝对表示（world-frame，29D）**：左手 = 6D 旋转 $r_6^L$ + 位置 $p_3^L$ + 夹爪 $g_1^L$（10 维），右手同构 10 维，头部 = $r_6^H$ + $p_3^H$（9 维），合计 **10+10+9=29**。旋转用 6D 表示（旋转矩阵前两列），因其**连续、利于神经网络优化**，优于欧拉角/四元数。
- **模型输入（相对、inter-gripper，29D）**：把右手保留在 world 系，左手与头部表示为**相对右手**的位姿：

$$^R T_L=(^W T_R)^{-1}\,{}^W T_L,\qquad {}^R T_H=(^W T_R)^{-1}\,{}^W T_H$$

再用同样的"6D 旋转 + 3D 位置"序参数化 $(r_6^{LR},p_3^{LR})$ 与 $(r_6^{HR},p_3^{HR})$。动作 chunk 采样时，对除夹爪外的所有 SE(3) 分量做绝对→相对变换（沿用 UMI 思路）。部署时逆变换回 world-frame 绝对指令，预测的 6D 旋转经 **Gram-Schmidt 正交化**还原为合法 $3\times3$ 旋转矩阵。

### 2.4 SPARKS：空间感知的鲁棒关键帧选择

主动头部会产生剧烈视点跳变，关键信息常首次出现在与当前帧差异极大的视角下（如侧头找物）。只看单帧或固定窗口会严重 context loss。SPARKS 在时刻 $t$ 的因果头姿历史窗口 $\{\, {}^W T_H(\tau)\mid \tau\le t\,\}$ 中给每个过去帧打分：

$$J(\tau)=\underbrace{\phi\big(\angle(\hat z_H(\tau),\hat z_H(t))\big)}_{\text{viewpoint novelty}}+\underbrace{\psi(t-\tau)}_{\text{recency}}+\underbrace{\rho\big(\angle(\hat z_H(\tau-1),\hat z_H(\tau))\big)}_{\text{motion smoothness}}$$

其中 $\hat z_H$ 是头部相机变换的前向轴。三项分别鼓励：视点新颖（与当前差异大→提供互补信息）、时间近、以及帧间角速度小（避开运动模糊帧，认为注视稳定时信息量更高）。只有超出多样性阈值（角位移 $>\alpha\cdot\text{FOV}$ 或平移位移 $>\delta$）的帧才进入 FIFO 缓冲。SPARKS **离线预计算**关键帧索引以保证训练时 IID 小批采样，部署时**在线** $O(L)$ 逐步运行（$L$ 为回看长度）。选出的关键帧图像被**直接作为额外 context 图像 token 喂给 PaliGemma 视觉-语言主干**，无需改动核心网络或引入可学习记忆模块。

用大白话说：SPARKS 是一个"零训练"的相册筛选器——它从头部转动的历史里挑出几张"最不一样、最清晰、又不太旧"的照片塞给模型当记忆，让模型即便当前正对着别处，也记得刚才侧头看到了什么。

### 2.5 两阶段微调与部署

- 从预训练 **π0** 权重初始化；但 π0 原本输出绝对关节位置，与目标表示不匹配，故做**两阶段微调**：(1) 用约 **200 小时** EgoMI 多任务数据把 π0 适配到 29D 相对笛卡尔动作空间；(2) 再用任务专属数据端到端微调至最优。
- **部署**：动作控制频率 25Hz，策略在 RTX 5090 上异步推理约 40ms 延迟，输出未来 **40** 步末端 + 头部位姿，采用 temporal chunk ensembling。用**可微 IK（Pyroki）**把左手/右手/头三个目标映射到关节配置，可微 IK 把不可达位姿当作代价最小化问题→"graceful degradation"（尽量逼近而非报错），从而**无需人工轨迹过滤即可鲁棒跨具身迁移**。躯干/臂与头由两个独立 Pyroki 进程并行求解，解耦架构使 29D 表示保持平台无关，支持不重训的 zero-shot 迁移。
- **机器人本体**：改装 Rainbow RBY1（轮式、6-DoF 躯干、2×7-DoF 臂），顶部装 I2RT YAM 作为 6-DoF 主动颈，颈上装 ZED 2i 作头相机，可同时追踪并复现 6DoF 头 + 手运动。

## 三、实验结果

全部为真实世界实验，**训练数据 100% 来自 VR 采集设备，零本体遥操作数据**。每任务采集约 1–1.5 小时域内数据，策略训练 40k 步（约 50 小时/个）。评测聚焦两点：主动头姿重定向 + 头相机观测对大范围双手操作的作用；SPARKS 对部分可观测下记忆任务的必要性。

### 3.1 采集精度对比（Table I，重定向误差 mm）

| 系统 | 误差 (avg±std) | 无机器人采集 | 头姿追踪 | 真实夹爪动作 |
|---|---|:---:|:---:|:---:|
| Gello | N/A | ✗ | ✗ | ✓ |
| ALOHA | N/A | ✗ | ✗ | ✓ |
| UMI | 8.855±3.228 | ✓ | ✗ | ✗ |
| AirExo-2 | 1.737±1.713 | ✓ | ✗ | ✗ |
| ViA | N/A | ✗ | ✓ | ✓ |
| **EgoMI** | **2.126±1.216** | ✓ | ✓ | ✓ |

EgoMI 是**唯一四项全满足**的系统，误差远低于 UMI、与 AirExo-2 同量级（且方差更小）。

### 3.2 搜索任务（对比 29D vs 20D）

- **29D 策略**：含主动头 SE(3) 动作输出 + 头相机图像。
- **20D 策略**：去掉头动作与头相机，仅用腕相机 + 夹爪 SE(3) 指令。

| 任务 | 29D（含主动头） | 20D（仅腕相机） |
|---|:---:|:---:|
| 桌面搜索（30″×60″，≤8 干扰物，含双手交接） | **36/40** | 29/40 |
| 货架搜索（≈2.4m 高货架，2 罐，含空中交接） | **35/40** | **0/40** |

消融：若给模型头相机图像流但**固定头相机、禁止主动重定位视点**，桌面任务骤降到 **2/20**——说明起作用的是"主动头运动"而非仅仅多一路头相机图像。货架任务中 20D 因无法定位屏外目标而**立即失败（0/40）**。

### 3.3 记忆任务（对比 SPARKS vs 单帧）

正前方桌上有 soup can 与 drink can 及放置篮；左侧 90° 处有一张**初始不可见**的侧桌，上面只放二者之一。机器人须先**侧头查看→记住是哪个物体→转回正面拾取正确物体**（若正确物在左侧还需先做双手交接再放入右侧购物篮），显式要求头部转动期间维持持久记忆。

| 策略 | 成功率 |
|---|:---:|
| Single-Timestep（仅当前头 + 腕帧） | 21/40（接近随机） |
| **SPARKS-Augmented**（保留过去关键帧） | **31/40** |

单帧策略退化为 Markov 假设、常常"不侧头就直接乱抓"；SPARKS 让策略稳定地先侧头、把关键帧存入记忆，再据此消解要拾取哪个物体。

## 四、局限性

1. **穿戴系统笨重**，长时间使用体验差，难以支撑超长采集。
2. **物理具身仍有残差**：机器人主动头的可动范围可能**超过人类自然头动范围**，重定向时反而受限于人的示范范围。
3. **SPARKS 为固定启发式打分**，不做自适应的记忆更新；更智能的条件化（如可学习记忆）可能进一步提升。
4. 实验均为真机小样本评测（每任务 40 次量级），**任务多样性与统计规模有限**；也无与 EgoMimic/DexCap/ARCap 等第一人称数据方法在同一 benchmark 上的直接可比数字。
5. 依赖固定十字准星做视线代理，**没有真实眼动追踪**，注视意图仅由头朝向近似，对"眼动与头动解耦"的场景（余光观察）可能失效。

## 五、评价与展望

**优点**：(1) 抓住了一个被主流忽视的正确变量——**主动头/视线本身就是弥合具身鸿沟的关键信息**，并用干净的消融（固定头相机 2/20、去 reticle 完全失败）把因果讲清楚；(2) 工程上把"无机器人采集 + 真实夹爪 + 头姿"四要素首次凑齐，且**全程零本体数据、无视觉增强/对齐/重渲染**就实现 zero-shot 迁移，方法论上比依赖大量视觉对齐/in-painting 的路线更简洁；(3) SPARKS 作为**训练-free 的关键帧记忆**，把"空间记忆"问题降维成"给 VLM 多喂几张精选 context 图像"，即插即用、无需重训记忆模块，在货架/记忆任务上带来 0/40→35/40、21/40→31/40 的巨大差距。

**局限与开放问题**：与 EgoMimic（AR 眼镜 + 3D 手部追踪）、DexCap（动捕手套 finger-level）、ARCap（AR 反馈引导）相比，EgoMI 的独特点在于**显式建模头动 + 记忆**，但缺乏跨方法的统一定量比较，难判断增益中"头动"与"高质量采集设备"各占多少。SPARKS 的固定启发式是明显的改进入口：viewpoint novelty / recency / smoothness 三项权重人工设定，未来可换成可学习的记忆读写或任务条件化的注意力。此外，把注视当作头朝向的代理在"look-then-reach"任务成立，但对需要余光/眼动独立的精细任务是硬约束，接入真实眼动信号是自然延伸。整体看，这是一篇**观点清晰、工程扎实、消融有说服力**的系统论文，其"主动感知是弥合具身鸿沟核心信息"的立论，对第一人称人类数据驱动的操作预训练方向有直接启发。
