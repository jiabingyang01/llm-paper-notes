## 一句话总结

IGen 把一张普通的"开放世界"RGB 图像(网上随手可得的照片)自动变成大规模机器人视觉-动作训练数据:先用单目几何基础模型 Metric3Dv2 + 物体级 3D 生成模型 TRELLIS + DINOv2/K-means 空间关键点把 2D 像素重建成结构化 3D 场景,再让 VLM 以关键点为锚做 $SE(3)$ 末端位姿的任务规划,最后在 Isaac Sim 里通过点云渲染合成时序一致的观测帧;仅用合成数据微调 $\pi_0$,真机 "Place Bottle" 任务成功率从 0% 提到 75%,1000 条纯合成样本达到 66.7%(超过同采集时间预算下人类遥操的 58.3%),而生成一条数据只需 8.3 GB 显存、18.6 秒(比 Cosmos-Predict2 快约 200 倍)。

## 一、问题与动机

通用机器人策略(VLA)的训练对数据规模呈指数级需求,但真机数据采集人力成本高、且被绑定在特定环境里,难以覆盖多样场景。与此相对,互联网上的开放世界图像海量、廉价,天然覆盖大量与操作任务对齐的真实场景,但它们缺少配套的机器人动作标注,导致这块视觉富矿一直无法直接用于策略学习。

已有两条路线都不够理想:

- **Real-Sim-Real / digital-twin(数字孪生)**:必须先对物理工作台做显式重建才能造数据,依赖场景特定的多视角采集,无法利用任意一张野外图像,可扩展性受限。
- **视频生成模型(如 Cosmos、TesserAct)**:能预测未来观测帧,但无法给出显式可执行的机器人动作,难以处理长程/复杂任务,且计算开销巨大。

IGen 的目标是补上"最后一公里":把非结构化的开放世界图像转成 grounded(物理接地)、可执行、可规模化的视觉-动作数据,让单张野外图像成为机器人策略预训练的有效来源。

## 二、核心方法

IGen 的流水线分三个阶段:场景重建 → 空间行为规划 → 观测合成。

### 1. 从像素到结构化 3D 表征(Scene Reconstruction)

给定单张 RGB 图,先用 VLM 识别与任务相关的物体,再用 SAM 得到物体掩码 $\mathcal{M}$,并用 DINOv2 提取场景特征。对特征嵌入与 3D 坐标做 K-means 聚类,得到一组空间关键点:

$$\mathcal{K} = \{ k_j \in \mathbb{R}^3 \mid j = 1, 2, \dots, K \}$$

**用大白话说**:关键点就是给场景里"哪儿能抓、哪儿是目标"打上一批带 3D 坐标的锚钉,后面让 VLM 直接对着这些锚钉编号来下指令,避免它凭空瞎猜空间位置。

背景与物体分开处理:

- **背景**:用 inpainting 模型(Inpaint Anything)把目标物体从原图抹掉、补全成背景图 $\mathbf{I}_{bg}$,再结合深度图 $\mathbf{D}$ 与相机内参 $\mathbf{K}$ 把所有像素反投影成稠密点云 $P_{bg} \in \mathbb{R}^{H \times W \times 6}$(含颜色)。
- **物体**:因为单目视角下物体是残缺的,用 3D 生成重建模型 **TRELLIS** 把被操作物体补全成完整 3D 形状与外观,再用 6D 位姿估计(Any6D)把补全后的物体摆回原位。
- **空间随机化**:从支撑面(如桌面)上采一批可行放置点,作为物体摆放的候选位姿,实现空间域随机化。

### 2. 面向行为生成的空间规划(Action Planning)

把标注了关键点 $\mathcal{K}$ 的图像、关键点 3D 坐标、任务指令一起喂给 VLM。VLM 通过高层任务规划把整体任务拆成 $N$ 个子阶段:

$$\mathcal{S} = \{ S_i \}_{i=1}^{N}$$

每个子阶段对应一段以 Python 写的、基于末端 $SE(3)$ 位姿的可编程低层控制函数:

$$\mathcal{F} = \{ f_i \}_{i=1}^{N}$$

其中每个 $f_i$ 是一个"关键点条件求解器":它从空间锚点 $\mathcal{K}_i$ 计算参考末端位姿,而平移距离、抬升高度等函数参数由 VLM 从 3D 关键点坐标里推断出来。抓取前阶段(pre-manipulation)的末端位姿由 grasp 模型(GraspGen)预测,夹爪开合由物体点云在夹爪主轴方向上的宽度决定。抓取后,物体上的关键点被视为"刚性附着在末端"的动点,其余关键点作为静态锚点。

从初始状态 $x_0$ 出发,运动规划器逐阶段递推下一状态:

$$\hat{x}_i = f_i(\hat{x}_{i-1}), \quad \hat{x}_0 = x_0$$

再用运动规划器(Curobo)生成可行轨迹并在仿真中执行,得到动作序列 $\mathcal{A} = \{ a_t \}_{t=1}^{T}$,每个 $a_t$ 含末端位姿与关节角。

**用大白话说**:VLM 不直接吐动作数值,而是先"写一段调用抓取/移动/旋转 API 的 Python 脚本",脚本里的具体坐标从关键点算出来。这样既借用了 VLM 的语义与推理能力,又保证了几何上的可执行性和精度。

### 3. 面向机器人学习的经验合成(Observation Synthesis)

把仿真机器人(Isaac Sim)放到规划位姿 $p_{robot}$,虚拟相机放在 $p_{cam}$。沿动作序列得到末端位姿轨迹 $\mathcal{T} = \{ \mathbf{T}_t \}_{t=1}^{T}$($\mathbf{T}_t \in SE(3)$),每一时刻渲染 RGB+深度并反投影出机器人点云 $\mathcal{P}_{robot}$,背景建模为静态点云 $\mathcal{P}_{bg}$。

被抓物体按末端位姿做刚体变换。设抓取发生在 $t_g$、抓取时段为 $\mathcal{T}_{grasp}$,物体点云演化为:

$$
P_{obj,t} =
\begin{cases}
P_{obj,t}, & t \notin \mathcal{T}_{grasp} \\
\mathbf{T}_t\, (\mathbf{T}_{t_g})^{-1}\, \mathbf{T}_{obj,t_g}\, P_{obj,t}, & t \in \mathcal{T}_{grasp}
\end{cases}
$$

**用大白话说**:夹爪没夹住时物体不动;一旦夹住,物体就"焊死"在末端上,随手一起动——用抓取瞬间的相对位姿差把物体点云跟着末端搬过去。

三部分拼成完整任务点云序列:

$$\mathcal{P}_{task} = \mathcal{P}_{bg} \cup \mathcal{P}_{obj} \cup \mathcal{P}_{robot}$$

再通过虚拟相机 $\mathbf{C}$ 渲染整段点云序列,得到与动作序列时序同步的视觉观测 $\mathcal{O}$。视觉观测 + 动作序列 = 可用于训练的成对视觉-动作数据。整条链路无需仿真物理引擎参与交互(simulation-free 的点云合成),因此非常轻量。

## 三、实验结果

评测围绕三问:能否从多样真实图像生成逼真数据?能否高效生成对齐环境、跟随指令的动作?能否只用单图合成数据、无任何人类遥操就训出可部署策略?

### 3.1 场景重建保真度(vs Real-to-Sim,在 Simpler/DreamGen Bench 上)

IGen 重建的场景与原始真实图的一致性显著优于数字孪生 Real-to-Sim。LPIPS 相似度最多提升 **5.13×**。

| 任务 | PSNR↑ (R2S / IGen) | SSIM↑ (R2S / IGen) | LPIPS₁↓ (R2S / IGen) | LPIPS₃↓ (R2S / IGen) |
|---|---|---|---|---|
| Pick Carrot on Plate | 18.15 / **28.26** | 0.676 / **0.837** | 0.286 / **0.052** | 0.206 / **0.042** |
| Put Eggplant in Basket | 16.69 / **23.28** | 0.771 / **0.835** | 0.246 / **0.083** | 0.168 / **0.055** |
| Put Spoon on Towel | 15.93 / **26.75** | 0.604 / **0.862** | 0.428 / **0.062** | 0.298 / **0.043** |
| Stack Cubes | 18.29 / **29.72** | 0.683 / **0.875** | 0.335 / **0.056** | 0.249 / **0.039** |
| **平均** | 17.26 / **27.00** | 0.683 / **0.852** | 0.324 / **0.063** | 0.230 / **0.045** |

(LPIPS₁/₂/₃ 分别用 AlexNet/VGGNet/SqueezeNet 计算;表中列出 LPIPS₁ 与 LPIPS₃。)

### 3.2 行为生成质量与效率(vs Cosmos-Predict2 / TesserAct)

用 GPT-4o、Qwen-3-VL-Plus、GLM-4.5V 三个评估模型,各方法生成 40 段视频,统计 *Instruction Following* 与 *Physics Alignment* 得 1 分的比例。IGen 在两项上均领先;在 Qwen-3-VL-Plus 评估下,IGen 的指令跟随成功数约为 baseline 的 **两倍**。

计算效率对比(生成一段机器人行为视频):

| 方法 | GPU 显存 | 平均生成时间 |
|---|---|---|
| Cosmos-Predict2 | 33.0 GB | 942.00 s |
| TesserAct | 13.0 GB | 275.40 s |
| **IGen** | **8.3 GB** | **18.60 s** |

即在更省显存的前提下,比 TesserAct 快约 **30×**、比 Cosmos-Predict2 快约 **200×**。

### 3.3 真机策略学习(纯图像 → 无遥操数据)

硬件:Franka Research 3 机械臂 + 前置 Microsoft Kinect(仅用 RGB)。基座策略为 $\pi_0$,用 LoRA 微调 30k 步(batch 8,输入单张 $224\times224$ RGB + 关节角状态,预测 10 步相对关节角 action chunk,单 A40 约 10.8 小时)。从单张图对每个场景生成 1000 条空间随机化样本。

| 设置 | Place Bottle 成功率 |
|---|---|
| $\pi_0$ 零样本(0 条) | ≈ 0% |
| 微调后(Place Bottle) | 0.0% → **75.0%** |

数据规模对比(同"采集时间预算"下,IGen 合成 vs 人类遥操):

| 样本量 | IGen 合成 | 人类遥操 |
|---|---|---|
| 100 条 | **44.5%** | 19.4% |
| 1000 条 | **66.7%** | 58.3% |

结论:纯 IGen 合成数据、零人类采集,即可超过相同时间预算的人类遥操,验证了"开放世界图像可作为机器人策略训练的有效数据源"。

### 3.4 消融(Figure 17)

- **去掉场景重建**:点云不完整,操作失败。
- **去掉关键点引导**:缺少关键点约束的空间计算,动作定位不准。
- **去掉多步规划**:直奔目标位置,不做避障,轨迹不合理。

## 四、局限性

- **依赖多个外部大模型的串联**:Metric3Dv2 深度、SAM 分割、DINOv2 特征、TRELLIS 物体重建、Any6D 位姿、GraspGen 抓取、VLM 规划任意一环出错都会传播,单目深度尺度误差、物体补全幻觉、6D 位姿偏差都会直接污染动作标注,论文未系统量化各环节失败率。
- **物理保真度靠点云刚体变换近似**:抓取后物体"焊死在末端",本质是运动学层面的刚体拼接,没有接触力、摩擦、形变、液体等真实物理;对倒水/柔性/精细装配等任务只能给出视觉上合理但物理上未必成立的轨迹(论文自身也用"Physics Alignment"这类由 VLM 打分的软指标,而非物理引擎校验)。
- **抓取与规划的可执行性上限受先验模型约束**:抓取位姿来自 GraspGen 先验、夹爪宽度靠点云主轴估计,对小物体、透明/反光物体、多物体堆叠场景的鲁棒性存疑。
- **任务范围偏短程刚体操作**:真机验证集中在抓放、浇水、敲击等相对短程任务,双臂、灵巧手、长程多阶段、可变形物体等未覆盖。
- **相机/内参假设**:开放世界图像用 Metric3Dv2 给的通用内参或已知机型内参,内参估计误差会影响 3D 尺度与最终动作的绝对精度。

## 五、评价与展望

**优点**:
1. **切中要害的定位**:抓住"互联网图像海量但无动作标注"这一核心矛盾,给出一条不依赖显式物理重建、也不依赖视频生成模型的第三条路——用 3D 重建 + VLM 关键点规划 + 点云渲染合成,把"图像→可执行动作+观测"的闭环打通。
2. **效率优势明显**:8.3 GB / 18.6 s 的开销比视频生成路线低一到两个数量级,这对"造几百万条预训练数据"是决定性的。simulation-free 的点云渲染是效率来源,也是它相对 digital-twin 仿真流水线的差异化。
3. **关键点 + 可编程控制函数的设计干净**:让 VLM 输出"调用 API 的代码 + 从关键点算参数",而非直接回归位姿数值,兼顾语义推理与几何精度,和 ReKep、VoxPoser、OmniManip 等 keypoint/约束式规划一脉相承,但把它接到了数据合成而非在线执行上。
4. **真机结果有说服力**:纯合成数据反超同时间预算的人类遥操,是本文最硬的证据。

**与其他公开工作的关系**:
- 与 **RoLA(Robot learning from any images)** 最接近,都主张"从任意图像造数据",但 RoLA 依赖较精确的物理属性估计做场景建模,IGen 走点云 + 生成重建的轻量路线,回避了物理重建。
- 与 **Cosmos-Predict2 / TesserAct / RoboDreamer** 等生成式世界模型路线正交:它们生成像素但缺显式动作,IGen 反过来先规划显式 $SE(3)$ 动作再渲染观测,天然带动作标注、且长程稳定。
- 与 **MimicGen / DexMimicGen / DemoGen** 等"从少量演示扩增数据"不同,IGen 的起点是单张图而非机器人演示,采集成本更低,但也因此更依赖重建与规划的正确性。

**开放问题与可能的改进方向**:
- **引入真实物理反馈**:把点云刚体近似换成带接触/力学的可微仿真或残差物理校正,能显著扩展到倒液、插拔、柔性等任务,并让"Physics Alignment"从 VLM 软打分升级为物理硬校验。
- **失败样本过滤与数据质量闭环**:合成数据难免混入几何/位姿错误样本,若能加自动可行性校验或用真机少量数据做判别式过滤,数据质量与下游成功率有望进一步提升。
- **规模化验证**:目前每场景 1000 条、任务偏短程;若把开放世界图像量级拉到十万/百万张、任务拉长,验证 scaling law 是否成立,才能真正支撑"通用策略预训练"的叙事。
- **跨本体/双臂/灵巧手迁移**:当前只在 Franka 单臂验证,末端 $SE(3)$ + 关键点表征理论上可迁移到更复杂本体,值得进一步实证。
