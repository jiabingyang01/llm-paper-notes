## 一句话总结

EMMA 让机器人**完全不采集移动遥操作数据**，仅靠"戴 Aria 眼镜的人类全身移动操作视频 + 静态机器人操作数据"协同训练，就学会了导航加操作耦合的移动操作策略；在 4 个真实任务上，用 1 小时人类数据替换 1 小时移动遥操作数据把 Handover Wine 全任务成功率从 52% 提到 82%（相对提升约 30 个百分点），并能迁移到只在人类数据里出现过的新场景（54% 成功率）。

## 一、问题与动机

移动操作（mobile manipulation）同时要求导航与操作两种能力,是机器人学的难点。当前主流的模仿学习路线（如 Mobile ALOHA）依赖**移动遥操作** 采集全身演示,但这种采集方式极其昂贵、难以规模化,导致数据多样性不足、在不可预测的真实环境中鲁棒性差。

作者的核心观察：机器人操作数据只能靠遥操作采（需要真实机器人),但**人类的移动操作数据可以很便宜地采**——让人戴上 Meta Project Aria 智能眼镜,以第一人称视角边走边做家务即可,不需要机器人、也不需要遥操作。此前用人类视频的工作大多局限于**桌面静态操作**;本文首次把这一思路推广到需要"边导航边操作"的**移动**场景,提出 Egocentric Mobile MAnipulation (EMMA)。

要落地必须跨过两道具身鸿沟：(1) 人类全向行走、头-视线解耦,而机器人是差速驱动（differential-drive）底盘,只能走直线和圆弧,且传感器与朝向固定绑定;(2) 人手运动在运动学与分布上都与机器人末端执行器不同。

## 二、核心方法

EMMA 是一个"全栈系统",由三部分组成：**动作重定向管线**（把人类移动操作动作翻译成差速底盘双臂机器人）、**统一的异构协同训练架构**、以及**辅助阶段识别机制**（推理时在导航/操作两种模式间切换控制)。

### 硬件与数据流

- **采集端**：Meta Project Aria 眼镜,通过 Machine Perception Service (MPS) 同时提供外感知（宽视场第一人称 RGB $I_{ego}$）与本体感知（双手追踪 + SLAM 全局定位)。
- **机器人**：两只 6-DoF ViperX 300s 机械臂装在高度可调支架上,底盘为 AgileX TRACER 差速 AGV（最高 2 m/s),整机 1.75 m 高,借鉴 EgoMimic 的 "Eve" 平台;Aria 眼镜以贴近成人头-手配置的方式装在机器人上,以缩小人机相机设备鸿沟。每只手腕装 Intel Realsense D405。
- 人类数据流：双手 3D 位姿 $H_p \in SE(3)\times SE(3)$、头部位姿 $H_d \in SE(3)$（SLAM 世界系）;机器人数据流：双臂关节 $R_q \in \mathbb{R}^{2\times 7}$（含夹爪）、末端位姿 $R_p \in SE(3)\times SE(3)$;共享第一人称 RGB $I_{ego}$,机器人另有手腕相机 $I_{wrist}$。

### 2.1 数据重定向与对齐

对导航用**基于优化的重定向**,对操作用**坐标系对齐**。

**导航动作标签**：把头部位姿 $H_d$ 投影到地平面得 2D 底盘位姿 $h^t_{base}=(x^t,y^t,\theta^t)$,作为人类"意图路径"的路点。因为差速底盘欠驱动、直接照搬人类路点是病态的,作者转成一个轨迹优化：给定人类路点序列 $\{h^k_{base}=(x^d_k,y^d_k,\theta^d_k)\}_{k=1}^K$,求速度指令 $\mathbf{z}=[(v_1,\omega_1),\dots,(v_K,\omega_K)]$ 使

$$\min_{\mathbf{z}} \sum_{k=1}^{K}\left[\lambda_{pos}\lVert p_k(\mathbf{z})-p^d_k\rVert_2^2 + \lambda_{yaw}\,\mathrm{wrap}(\theta_k(\mathbf{z})-\theta^d_k)^2 + \lambda_{smooth}\big((v_k-v_{k-1})^2+(\omega_k-\omega_{k-1})^2\big)\right]$$

约束为差速运动学 $x_{k+1}=x_k+v_k\cos(\theta_k)\Delta t$、$y_{k+1}=y_k+v_k\sin(\theta_k)\Delta t$、$\theta_{k+1}=\theta_k+\omega_k\Delta t$,并 $-1.6\le v_k\le 1.6$ m/s、$-1.5\le\omega_k\le 1.5$ rad/s;权重 $\lambda_{pos}=25,\ \lambda_{yaw}=2,\ \lambda_{smooth}=1$。

> 用大白话说：人类走的路人机器人走不出来（会拐得太急/横移),所以不硬抄坐标,而是让机器人"在自己能做到的差速运动里,尽量贴合人走过的位置和朝向,同时别猛加速",解出一条平滑、可执行的机器人速度指令序列。

**导航上下文**：把 $H_d$ 投影得 $h^t_{base}\in SE(2)$,维护一个基于位移的路点历史 $\mathcal{H}_t=\{h^{t-k_i}_{base}\}_{i=1}^{K_h}$,每移动 0.5 m（位移 $\lVert h^t_{base}-h^{t-k}_{base}\rVert_2 \ge d_{thresh}$)才加一个新路点——按位移而非时间采样,保证空间分辨率一致;再统一变换到当前第一人称系 $\hat{\mathcal{W}}_t=\{T^{-1}_{ego}\cdot h^{t-k_i}_{base}\}$ 作为策略的上下文输入。

> 用大白话说：给策略喂"我刚才走过的一串脚印"(相对当前视角),让它知道自己正沿什么路径前进。

**操作动作对齐**：借鉴 EgoMimic,把人和机器人的上半身动作块都变换到**观测时刻相机参考系**(人用 SLAM 估计,机器人用手眼标定),使预测都相对当前视角;再对人、机两个数据源 $\mathcal{D}_H,\mathcal{D}_R$ **各自独立**做 Z-score 归一化,以吸收生物力学与传感器带来的持续分布差异。

### 2.2 人机异构协同训练

架构基于 decoder-only Transformer,采用 stem-trunk-head 结构（建立在 HPT 之上)：

- **Stems**（浅网络,把原始观测编码成定长 token)：对主第一人称 RGB $I_{ego}$ 用**共享视觉 stem**(人、机共用),强制视觉对齐;机器人手腕相机 $I_{wrist}$ 用单独 stem（人类数据没有这一路)。
- **Trunk**：标准多层 decoder-only Transformer,处理各活跃 stem 拼接的 token 序列,前面预置 $M$ 个可学习 token（动作块长度)。
- **Heads**（浅 MLP,把 trunk 前 $M$ 个输出映射到动作/辅助空间),共四个：机器人双臂关节动作 $\mathbb{R}^{K\times 14}$、人类笛卡尔末端动作 $\mathbb{R}^{K\times 3}$、机器人底盘导航动作 $(x,y,\omega)\in\mathbb{R}^{K\times 3}$、以及辅助输出的任务阶段 $p\in\{0,1\}^K$。
- **协同训练**：人类 batch 激活"人类本体感知 + 共享 ego 视觉 stem + 人类操作头 + 导航头 + 共享阶段头";机器人 batch 激活"机器人本体感知 + 手腕图像 stem + 共享 ego 视觉 stem + 机器人操作头"。共享 trunk 与 ego 视觉 stem 被所有源更新,被迫学到通用表征;**导航头主要从重定向后的人类数据里学**,从而把人类导航策略迁移到机器人。

> 用大白话说：两种数据共用一个大脑（trunk)和一只眼睛(共享视觉 stem),但各接各的"手"(操作头)——机器人从没导航过,导航能力完全靠人类数据"隔空"教出来。

### 2.3 辅助阶段识别与控制调制

移动操作天然在"导航"与"操作"两个阶段交替,需要不同控制策略。

**阶段检测**（Algorithm 1,无监督)：对每帧算头速 $v_{head}$、手速 $v_{hand}$,比值 $r=v_{hand}/(v_{head}+\epsilon)$;取 $\mathcal{M}=\{i: r_i>\tau_{ratio}\wedge v_{head,i}<\tau_{head}\}$(手动头静的高比值帧),在这些帧的头部位置上拟合 $K{=}2$ 的 GMM;若某帧头部位置在 GMM 下的概率密度超过 $\tau_{pdf}$ 则判为操作(phase 0),否则导航(phase 1)。经验取 $\tau_{ratio}=2.0,\ \tau_{head}=0.4$ m/s,$\tau_{duration}=30$。

> 用大白话说：人在"干活"时通常是站住不动、手在忙(手速/头速比很高),用这个比值把"站着操作"的时刻挑出来,再用 GMM 找出这些操作发生的空间热点,以此自动切分导航段和操作段——完全不需要人工标注。

**阶段感知控制**：用预测阶段调制导航动作块,防止乱动——操作阶段时,导航动作从 0 插值到第一个未来导航路点(即基本不动底盘);导航阶段时,把任何未来的操作阶段位置替换成当前导航终点。这样消除了操作时手部动作串扰进底盘的噪声,并让两阶段平滑切换。

## 三、实验结果

设置：ViperX 双臂 + AgileX 差速底盘;4 个长程任务（Table Service、Handover Wine、Grocery Shopping)加 Push Chair;每模型/任务 50 次试验,95% Clopper–Pearson 置信区间,两比例 Z 检验（$p<0.05$ 判显著),单次限时 2 分钟,碰撞记为失败,累计 1150 次真机 rollout。主基线 **Mobile ALOHA**(代表移动遥操作数据),为公平比较改用与 EMMA 相同的 HPT 骨干,只隔离"数据来源"这一变量。

**主结果(累积子任务成功率,末项即全任务)**:EMMA 完全不用移动遥操作数据,却在 Grocery Shopping 与 Handover Wine 上显著优于 Mobile ALOHA（$p<0.05$),Table Service 两变体持平。

| 任务 / 子任务 | EMMA | Mobile ALOHA |
| --- | --- | --- |
| **Grocery Shopping**：取果汁+薯片 | 0.84 | 0.32 |
| 双臂取爆米花 | 0.84 | 0.28 |
| 双臂拎起购物袋 | 0.68 | 0.16 |
| 导航到桌 | 0.68 | 0.04 |
| 放袋到桌(全任务) | **0.60** | 0.00 |
| **Handover Wine**：取酒杯 | 0.86 | 0.72 |
| 导航到人 | 0.82 | 0.52 |
| 递交(全任务) | **0.82** | 0.52 |
| **Serve Utensils**：拿餐具 | 0.84 | 0.62 |
| 导航到餐桌 | 0.38 | 0.30 |
| 摆放(全任务) | 0.30 | 0.12 |
| **Serve Croissant**：牛角包入盘 | 0.46 | 0.66 |
| 全任务 | 0.26 | 0.26 |

Mobile ALOHA 的典型失败(定性)：导航时撞到电视、双臂抓取失误导致把椅子推歪;EMMA w/o retargeting 则因执行运动学不可行轨迹而在导航中"丢失接收人"。

**H3 数据规模律(Handover Wine,固定 50 条静态机器人演示,逐渐加移动数据)**:加人类数据的收益远高于加等时长的移动遥操作数据,差距随数据量从 10% 拉大到 30%。

| 追加移动数据时长 | EMMA(加人类数据) | Mobile ALOHA(加遥操作) |
| --- | --- | --- |
| 15 min | 0.36 | 0.26 |
| 30 min | 0.42 | 0.32 |
| 45 min | 0.56 | 0.44 |
| 60 min | **0.82** | 0.52 |

**场景泛化(未见新场景,接收人在更大 5m×2m 区域随机站位,仅采 30 min 人类演示)**:EMMA 全任务 54%（子任务 0.84 / 0.56 / 0.54);Mobile ALOHA 只在原始环境的遥操作数据上训练,换环境后连第一步抓取都完不成(0.04 / 0.02 / 0.02)。

**消融(Handover Wine,Fig 9)**:两个组件都关键。

| 方法 | 取酒杯 | 导航到人 | 递交 |
| --- | --- | --- | --- |
| EMMA(完整) | 0.86 | 0.82 | 0.82 |
| w/o 动作重定向 | 0.88 | 0.56 | 0.52 |
| w/o 阶段切换 | 0.24 | 0.08 | 0.08 |

去掉重定向,导航段成功率掉约 30 个百分点（轨迹不可行、丢目标);去掉阶段切换更是近乎全崩(操作时底盘乱动、碰撞、掉物)。

**阶段识别精度(Mean over Frames,人工标 10% 帧比对)**:Table Service 0.962、Handover Wine 0.965、Handover Wine 新场景 0.929、Grocery Shopping 0.963,全部 $\ge 0.92$。

**Push Chair(需臂-底盘紧耦合,不做阶段切换)**:统一 EMMA(10 min 机器人 + 20 min 人类)在"初始双臂接触"达到 1.00,整体与 30 min 纯遥操作的 Mobile ALOHA 相当,并优于把导航/操作拆成两个独立模型的消融;而只有 10 min 机器人数据的 Mobile ALOHA 初始接触仅 0.46。说明单策略协同训练更能利用人类数据去学臂-底盘耦合。

## 四、局限性

1. **视觉/空间分布假设**：框架假定机器人部署时的视觉与空间分布落在(或接近)人类演示所覆盖的范围;当人机相机或视角差异变大时该假设会失效。
2. **紧耦合任务**：人类导航策略的直接迁移不足以支撑需要臂-底盘紧耦合的任务(如 Push Chair 必须掺入机器人数据才行),纯人类数据在这类任务上力有不逮。
3. 底盘限定为差速驱动,重定向管线针对该运动学定制;换全向/腿式底盘需重新设计。评测均为真机、单一实验室场景,规模(50 试/任务)偏小。

## 五、评价与展望

**优点**：(1) 命题清晰且实用——把"贵的移动遥操作"整段替换成"便宜的第一人称人类数据",并用规模律证明每小时人类数据的边际价值高于每小时遥操作;(2) 三个组件各司其职、消融充分,尤其无监督阶段识别(头/手速比 + GMM)简洁且 MoF 均 >0.92,是把桌面级 human-data 方法推向"移动"场景的关键补丁;(3) 场景泛化实验有说服力——新场景只喂人类数据就能到 54%,直接体现了人类数据"多样性便宜"的规模化优势。

**与公开工作的关系**：数据范式上承接 EgoMimic(Aria 第一人称人-机协同训练)与 Mobile ALOHA(移动遥操作,这里作为被替换的对象/基线);架构上建立在 HPT 的异构 stem-trunk-head 之上;导航重定向可视作把"人类头部轨迹→差速可行轨迹"的轨迹优化嵌入模仿学习,与用 RL/关键点匹配做导航的路线互补。相较近期大规模 VLA(RT-2、OpenVLA、π0 等)偏静态操作,本文补上了"移动 + 全身"这一维度。

**开放问题与改进方向**：(1) 视角/相机鸿沟仍靠"把眼镜按成人配置装到机器人上"硬对齐,更本质的做法可能是显式的跨视角/跨具身表征学习或域随机化,以放宽第 1 条局限;(2) 阶段识别的阈值($\tau_{ratio},\tau_{head},\tau_{pdf}$)按 Table Service 经验选定,跨任务的自适应/端到端可学切换值得探索;(3) 紧耦合任务仍需少量机器人数据,如何用更少机器人样本(甚至零样本)完成臂-底盘耦合是核心难点;(4) 差速重定向可推广到全向/轮足/人形底盘,并结合生成式增广进一步扩充人类数据分布覆盖。总体上,这是"人类第一人称数据替代昂贵机器人遥操作"这条主线在移动操作方向上一份扎实、可复现、结论明确的推进工作。
