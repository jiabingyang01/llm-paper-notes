## 一句话总结

DLR 提出一个 **Discover-Learn-Reinforce 三阶段框架**,用轻量 RL 策略充当 VLA 预训练的"数据引擎":先用 VAE 从人类演示里发现多个隐式行为 pattern,再对每个 pattern 做行为克隆得到条件策略,最后用稀疏成功奖励在线强化——核心洞见是把"互信息多样性目标"**解耦到人类成功状态流形上**,从而在同一任务里生成多条高成功率且互不相同的轨迹,避免标准 RL 的 mode collapse。在 LIBERO 上,轨迹的 Mean Pairwise Distance 达 26.4(单模态 RL 仅 10.5),下游 OpenVLA 微调平均成功率 16.72% 对比单模态 O2O-RL 的 9.42%(π₀ 为 16.45% 对 13.45%),且展现出单模态 RL 缺失的正向数据规模效应。

## 一、问题与动机

VLA 大模型预训练的效果高度依赖操作轨迹数据的**规模与多样性**。当前主流数据来自人类遥操作,昂贵、难扩展,且人在演示时天然只求"把任务做成",很少刻意展示多种可行解法,导致行为多样性内生受限。

强化学习(RL)可以自主探索、生成数据,甚至超越人类演示的质量,是一条有吸引力的扩展路径。但**标准 RL 只优化任务成功这一个目标,会收敛到单一最优执行 pattern(mode collapse)**:对于一个狭窄任务,单一解法或许够用,但作为通用 VLA 的预训练语料,它覆盖的状态-动作空间太窄,提供的先验太贫乏,无法支撑下游泛化。世界模型生成轨迹是另一条路,但当前世界模型在精细、长时序操作上仍受累积误差之苦。

因此论文把 RL 训练的目标**从"找一个最优策略"重构为"发现一组高成功率的行为 pattern 库"**,让每种行为都成为高质量的数据生成器,支持多样化采样用于 VLA 预训练。

## 二、核心方法

### 2.1 问题设定与朴素做法

在 MDP $\mathcal{M}=\langle \mathcal{S},\mathcal{A},\mathcal{P},\mathcal{R},\rho_0,\gamma \rangle$ 下,采用稀疏成功奖励,轨迹回报为

$$R(\tau) = \gamma^{T-1}\,\mathbb{I}_{\text{succ}}(\tau)$$

其中 $\mathbb{I}_{\text{succ}}$ 是任务是否成功的二值指示;折扣因子鼓励更短(更高效)的成功轨迹。为生成多样行为,引入隐变量 $z\in\mathcal{Z}$ 得到 pattern 条件策略 $\pi_\theta(a\mid s,z)$,推理时从均匀先验 $p(z)$ 采样,目标是最大化对 pattern 与轨迹双重取期望的回报:

$$J(\theta)=\mathbb{E}_{z\sim p(z),\,\tau\sim\pi_\theta(\cdot\mid z)}\big[R(\tau)\big]$$

**用大白话说**:每个 $z$ 对应一种"做法",希望每种做法都能把任务做成,且各不相同。

沿多样性驱动 RL 的思路,最直接的做法是把任务奖励与状态-隐变量互信息 $I(Z;S)$ 相加:

$$\max_\theta\ \mathbb{E}[R(\tau)]+\beta\, I_\theta(Z;S)$$

互信息不可直接算,用变分下界得到密集内在奖励:

$$r_{\text{div}}(s,z)=\log q_\phi(z\mid s)-\log p(z)$$

其中判别器 $q_\phi(z\mid s)$ 是对 pattern 的近似后验。

### 2.2 核心洞见:任务奖励与多样性目标的冲突

论文识别出这种朴素相加存在**根本冲突**,是本文最有价值的分析:

- **判别器对未见状态"失明"**。$q_\phi$ 是 on-policy 训练的,只在策略已访问过的状态上有意义。对从未探索的新状态,判别器无法区分 pattern,预测退回先验,即 $d^\pi(z\mid s)\approx p(z)\Rightarrow r_{\text{div}}\approx 0$。于是当智能体尝试探索"解决任务所必需但陌生"的新状态时,密集多样性奖励骤降,而稀疏任务信号又不能立即补偿——探索被当作净惩罚,策略被逼留在熟悉状态里。
- **盲目最大化多样性会跑偏任务**。策略可能学会做多种"不同但都无用"的行为(如伸向多个错误目标)只为刷多样性奖励。

论文在附录 E 用实验坐实了这点(Figure 10):MI-shaped 目标训练出的轨迹**看上去在 XY 平面很分散,却无法可靠完成关抽屉任务**;而只用稀疏奖励的 DLR 反而成功且各 pattern 轨迹非平凡地不同。

### 2.3 解耦的多样性目标

核心解法:**多样性不应在所有轨迹上追求,只在已经成功的轨迹上追求**。定义成功轨迹集 $\mathcal{T}_{\text{succ}}$ 及其覆盖的成功状态流形 $\mathcal{S}^\star=\bigcup_{\tau\in\mathcal{T}_{\text{succ}}}\{s_0,s_1,\dots\}$,目标状态分布 $d^\star(s)\triangleq p(s\mid \mathbb{I}_{\text{succ}}(\tau)=1)$。松弛后的完整目标为

$$\max_{\theta,\phi}\ \underbrace{\mathbb{E}_{z,\tau\sim\pi_\theta(\cdot\mid z)}[R(\tau)]}_{\text{Reinforce}}+\beta\underbrace{I_\theta(Z;\mathcal{S}^\star)}_{\text{Discover}}-\alpha\underbrace{D\big(d^\pi(s)\,\|\,d^\star(s)\big)}_{\text{Learn}}$$

**用大白话说**:多样性只在"确实能把任务做成"的状态区域里做文章,不去干扰对成功解的初始发现;发散项 $D(\cdot\|\cdot)$ 作为软惩罚,把策略拉回成功状态流形上。由于 $\mathcal{S}^\star$ 未知,论文用一批固定的高质量人类演示 $\mathcal{D}_{\text{human}}$ 作为它的稳定代理,从而把三项拆成可实现的三阶段。

### 2.4 三阶段实现

- **Stage 1 — Discover**:用 VAE 从人类数据 $\mathcal{D}_{\text{human}}$ 中发现隐 pattern,训练编码器 $q_\phi(z\mid s)$ 以最大化 $I(Z;S)$,把状态聚成语义上不同的模式;编码器随后冻结。
- **Stage 2 — Learn**:用冻结的 $q_\phi$ 给每个状态贴标签 $z=\arg\max_{z'}q_\phi(z'\mid s)$,在带标签数据上做行为克隆训练条件策略:

$$\max_\theta\ \mathbb{E}_{(s,a,z)\sim\tilde{\mathcal{D}}_{\text{human}}}\big[\log \pi_\theta(a\mid s,z)\big]$$

BC 隐式最小化 $D(d^\pi(s)\|d_\mathcal{D}(s))$,即把初始策略对齐到成功状态流形的经验估计。
- **Stage 3 — Reinforce**:仅用环境稀疏奖励在线微调,不再需要 MI 密集塑形。从均匀先验采样 $z$,用 PPO 让每个 pattern 条件策略收敛到各自成功流形内的最优版本:

$$\max_\theta\ \mathbb{E}_{z\sim p(z),\,\tau\sim\pi_\theta(\cdot\mid z)}\big[R(\tau)\big]$$

### 2.5 多样性保持的理论保证

针对 Stage 3 的 PPO 微调,论文给出 Theorem 1(pattern 保持定理)。定义跨 pattern 泄漏 $p^j_{\text{leak}}(\theta):=\Pr_{\tau\sim p_\theta^j}[\tau\in\cup_{k\neq j}\mathcal{T}_k^+]$。在四条假设(失败护城河 failure moat、pattern 初始分离、PPO 目标-KL 裁剪、正则性)下,对每个 pattern $j$ 与所有 Stage 3 迭代 $t$:

$$\Big\| \mathbb{E}_{\tau\in\cup_{k\neq j}\mathcal{T}_k^+}\big[\nabla_\theta \log p_{\theta_0}^j(\tau)\,R(\tau)\big] \Big\| \le \sqrt{B}\sqrt{\delta_0+\sqrt{E/2}}$$

**用大白话说**:失败轨迹回报为 0 不产生梯度;"抢别人 pattern"的跨模式梯度被 KL-to-init 预算 $E$ 与初始分离度 $\delta_0$ 上界卡住;于是每步 PPO 更新被自身成功轨迹 $\mathcal{T}_j^+$ 主导,更新保持局部化,收敛到 $\mathcal{T}_j^+$ 内的局部最优——各 pattern 不会塌缩合并成一个。

## 三、实验结果

**设置**:LIBERO 基准(Bo Liu et al., NeurIPS 2023),纯仿真。用 LIBERO-90(90 个任务)作为数据源与 VLA 预训练源,再在 LIBERO-Spatial / Object / Goal / Long 四个从未见过的下游套件上微调评测。轻量 RL 策略为 ResNet18 + MLP(约 10M 参数,PPO);pattern 数 $\lvert Z\rvert=3$、均匀采样。VLA 模型为 π₀(约 3B)与 OpenVLA(约 7B),用 SFT 预训练。基线为 **O2O-RL**(offline-to-online 单模态 RL:先 BC 初始化再 PPO 在线微调)。两方法数据量对齐以公平比较质量而非数量;下游按低数据快适应设定(少于 3 个 epoch,早停)。

**轨迹多样性(Table 1)**——DLR 在几乎所有度量上远超单模态基线:

| 度量指标 | O2O-RL | DLR |
| --- | --- | --- |
| Mean Pairwise Distance (↑) | 10.487 | **26.405** |
| Endpoint Variance (↑) | 0.092 | **1.170** |
| Direction Variance (↑) | 0.083 | **0.085** |
| Path Length Variance (↑) | 17.193 | **26.546** |

状态访问分布对多模态目标的 KL(Figure 1):DLR 0.052,O2O-RL 0.582——DLR 更好地覆盖多个模式。Figure 5 用 DTW 距离刻画同一"关抽屉"任务:O2O-RL 收敛为单一确定策略(DTW=6.755),DLR 方差更大(DTW=7.851)。定性上(Figure 4),DLR 在同一初始状态下发现两种做法,如把书顺时针 vs 逆时针旋转、拉门 vs 先拉后推、把挡门杯子先取回放后 vs 向前推开。

**下游快适应成功率(Table 2,%)**——预训练数据的多样性转化为更强的下游泛化,尤其在 OOD 套件上:

| 数据源 | Spatial | Object | Goal | Long | Avg |
| --- | --- | --- | --- | --- | --- |
| **π₀ / 无预训练** | 0.60 | 0.00 | 0.20 | 0.00 | 0.20 |
| **π₀ / O2O-RL** | 4.40 | 15.20 | 19.20 | 15.00 | 13.45 |
| **π₀ / DLR** | **6.00** | **18.60** | **22.40** | **18.80** | **16.45** |
| **OpenVLA / 无预训练** | 1.60 | 2.60 | 17.04 | 2.20 | 5.86 |
| **OpenVLA / O2O-RL** | 11.60 | 3.04 | 19.11 | 3.94 | 9.42 |
| **OpenVLA / DLR** | **19.88** | 3.67 | **34.20** | **9.21** | **16.72** |

**数据规模效应(Figure 7,LIBERO-Spatial Task 5)**:随 RL 生成数据量增大(每任务 25→50→100→200),DLR 成功率呈上升趋势(2.0→…→46.0),而单模态 O2O-PPO 反而下降(36.0→6.0→4.0)——**正向数据规模效应正是单模态 RL 所缺失的**,支撑了"多样性数据引擎可扩展"的主张。

**与朴素 MI 目标的对照(附录 E,Figure 11)**:用 MI-shaped 内在奖励(即 2.2 节冲突的目标)预训练,3 个 epoch 微调后在四个套件上仅得 4.0–15.0%,而 DLR 预训练得 49.8–55.4%,每个 epoch 都大幅领先,验证了"解耦到成功状态流形"相对朴素互信息塑形的必要性。(注:该消融的绝对数值与 Table 2 的整基准平均口径不同,这里仅取"DLR ≫ MI-shaped"的相对结论。)

## 四、局限性

1. **纯仿真验证**。所有实验仅在 LIBERO 仿真上完成,无任何真机结果;RL 数据引擎在真实动力学与传感噪声下能否保持多样性与成功率是未知数。
2. **pattern 数需预设**。$\lvert Z\rvert=3$ 为人工固定,论文未给出自动确定 pattern 数目或发现"任务实际有几种做法"的机制。
3. **并非无人监督**。框架把成功状态流形 $\mathcal{S}^\star$ 用固定人类演示做代理,因此仍需 bootstrap 一批人类数据;论文向往的"从以人为中心转向算法生成"只是部分实现。
4. **绝对成功率偏低**。下游平均成功率约 16%(受低数据、少于 3 epoch 的快适应设定所限),读者难以判断在充分微调下的天花板;且 OpenVLA 在 Object 套件上 DLR(3.67)与基线(3.04)几乎持平,增益并非全面。
5. **多样性仅限任务内行为**。DLR 只在单任务内制造多种做法,不生成新任务/新场景;论文亦承认"结合自动任务与环境生成"是留待未来的工作。
6. **理论假设难验证**。Theorem 1 依赖 failure moat、pattern 初始分离、KL 预算等假设,这些在实际训练中是否成立、$\delta_0$ 与 $E$ 的量级如何,论文未做实证核验。

## 五、评价与展望

**优点**。(1)问题重构干净有力:把 RL 从"求单一最优"改造成"造多样数据引擎",契合 VLA 预训练对覆盖广度而非单点最优的真实需求。(2)对"任务奖励 + 互信息多样性"朴素相加为何失败的分析(on-policy 判别器对未见状态失明→惩罚探索)是本文最扎实的贡献,并配了 Figure 10 的"看似多样却全失败"的负结果佐证,说服力强于纯定性叙述。(3)"把昂贵 RL 从大 VLA 卸载到 10M 小策略、再用其 rollout 预训练大模型"这一算力解耦范式,工程上很务实。(4)有 Theorem 1 给多样性不塌缩提供了理论说法。

**缺点与开放问题**。(1)基线略单薄:主要对比自家单模态 O2O-RL 与 MI-shaped 消融,未与显式技能发现类方法(DIAYN、DADS 等,论文以"它们只优化内在动机、任务成功率低"为由略去)或世界模型/生成式数据增强路线做正面成功率对照,难以横向定位。(2)DLR 与近期同思路工作关系密切——同一批作者的 arXiv 2509.19752("Beyond Human Demonstrations: Diffusion-based RL to generate data for VLA")以及 SimpleVLA-RL、RLDG、VLA-RL 等都在探索"RL 生成数据 / RL 微调 VLA",本文的差异化主要在"多模态而非单模态",值得读者串起来看。(3)三阶段流水线组件多(VAE 聚类 + BC + PPO),超参 $\alpha,\beta,\lvert Z\rvert$ 的敏感性与工程稳定性未充分暴露。

**可能改进方向**。把 $\lvert Z\rvert$ 换成可增长/自适应的 pattern 发现(如非参数聚类或按数据自动确定模式数);把成功状态流形的代理从固定人类演示改为在线自举的成功缓冲,朝更少人类依赖推进;把 pattern 条件从 proprioceptive 状态扩展到语言指令条件,以在任务间共享 pattern 语义;并在真机或更强下游微调预算下检验多样性增益是否持续。总体看,这是一篇"洞见清晰、分析扎实、验证受限于仿真"的数据合成方向工作,其"多模态成功轨迹 > 单模态最优轨迹"的结论对以 RL 造数据的 VLA 预训练具有方法论参考价值。
