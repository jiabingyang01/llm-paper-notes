## 一句话总结

ROPA 把人体图像生成里的 Pose-Guided Person Image Synthesis 迁移到机器人上：用 ControlNet 微调 Stable Diffusion 2.1，以"骨架位姿图"为条件，为双臂机器人合成新位姿下几何一致的第三人称 RGB 与深度图，并同步生成对应的关节动作标签（对接触阶段用约束优化保证物理可行），从而离线扩增双臂模仿学习数据；在 PerAct2 的 5 个仿真任务与 3 个真机任务上，把 ACT 基线的成功率从 41.3% 提升到 68.0%（Coordinated Lift Ball，RGB）。

## 一、问题与动机

双臂操作（bimanual manipulation）需要覆盖广泛机器人位姿、接触状态与场景上下文的示教数据,但真实世界采集既贵又慢,是可扩展性的瓶颈。数据增广是一条低成本扩数据的路子,但已有工作大多局限于:

- **单臂**场景（如 MimicGen/DexMimicGen 从仿真 rollout 合成新状态-动作对）;
- **腕部相机（eye-in-hand）** 视角(如 RoVi-Aug),或者只做**新视角合成而不产出新的动作标签**(如 VISTA)。

把这些技术扩展到双臂 + 深度图 + 第三人称(eye-to-hand)设定会遇到三个新难点:(1)两条手臂 + 周围场景的**全局视觉一致性**;(2)增广后仍要保证**动作标签正确、可在真机执行**;(3)更高的自由度(DOF)。ROPA 的核心洞见是:把"给定目标骨架位姿、合成一致人像"的思路搬到机器人上——以机器人关节构型为条件驱动扩散模型,生成新机械臂位姿的真实第三人称图像,同时严格维护"增广视觉观测"与"有效动作"之间的对应,使得在增广集上训练的策略仍能在真机上执行。

## 二、核心方法

ROPA 是一个**离线** 增广流程(不需要交互式采集),分三阶段:①在示教数据上微调 Stable Diffusion(PGRIS);②为新位姿渲染骨架位姿图(Skeleton Pose Generator);③生成动作标签、重构数据集(Action Labeling)。策略侧统一用 ACT(Action Chunking with Transformers)。

### 2.1 Pose Guided Robot Image Synthesis(PGRIS)

问题定义:给定源相机图像 $I_t^s$、目标骨架位姿图 $I_t^p$、语言目标 $g$,学习映射

$$I_t^d = f_\psi(I_t^s, I_t^p, g)$$

其中 $f_\psi$ 由 Stable Diffusion 2.1 + ControlNet 实现。源图与目标位姿图经预训练 VAE 编码为潜变量 $z_s = E(I_t^s)$、$z_{pose} = E(I_t^p)$,语言目标经文本编码器得 $c_{text} = \xi(g)$。可训练的 ControlNet 模块 $\mathcal{C}_\phi$ 处理条件输入 $(z_s, z_{pose})$,通过零初始化卷积把位姿感知特征注入被冻结的 U-Net 去噪器 $\epsilon_\phi$。训练目标为标准的噪声预测损失:

$$\mathcal{L} = \mathbb{E}_{z_d, z_s, z_{pose}, g, n, \epsilon}\left\| \epsilon - \epsilon_\phi\big(z_{d,n},\, z_s,\, c_{text},\, n;\, \mathcal{C}_\phi(z_s, z_{pose})\big) \right\|_2^2$$

其中加噪潜变量按 DDPM 调度 $z_{d,n} = \sqrt{\alpha_n}\, z_d + \sqrt{1-\alpha_n}\,\epsilon$。推理时从随机高斯潜变量出发,在 ControlNet 引导下用 DDIM 迭代去噪,再解码 $I_t^d = D(z_d^*)$。

> 用大白话说:这就是"给扩散模型看一张原始机器人照片 + 一张想要的骨架姿势图 + 一句任务指令,让它画出'机器人摆成新姿势'的照片"。零初始化卷积保证训练早期 ControlNet 不会把有害噪声灌进冻结的主干,让位姿条件"温柔地"生效。

### 2.2 Skeleton Pose Generator(骨架位姿图怎么来)

为了大规模、可扩展地渲染"几何一致的机器人骨架",作者借鉴 OpenPose 的人体关键点惯例,把每条手臂的运动学骨架显式表示成**圆柱(骨段)+ 球体(关节)**,并在球体上贴 GENIMA 的**条纹纹理**以提供关节朝向/旋转状态的视觉线索(消融显示这一步很关键)。具体做法:用 PyRender 加载两条手臂的 URDF,按源相机的内外参配置虚拟相机,把手臂摆到目标关节角后渲染,得到与真实相机视图几何对齐的骨架图 $I_t^p$。目标关节角由"对源末端位姿施加变换 $\Delta\rho$ 后解逆运动学(IK)"得到。

> 用大白话说:与其让扩散模型凭空理解 14 维关节向量,不如把关节构型"画成一根根带条纹的骨头棒",这种带纹理的骨架图比裸关节数字、比纯白骨架都更能引导扩散模型画对姿势和朝向。

### 2.3 Action Labeling and Dataset Construction(动作标签 + 约束优化)

只会造图还不够,增广出来的位姿必须配上**合法**的动作标签,尤其双臂场景要防止两臂碰撞或不协调扰动。做法是把任务分解为**无接触(contactless)** 与**接触密集(contact-rich)** 两类状态:

- **接触检测** 采用 Zhang et al. 的力学方法:用自回归模型建模外部力矩残差 $\mu_{ext} = \mu_{motor} - \hat{\mu}_{model}$,当实际力矩连续若干时间步超过动态阈值即判定为接触。相比视觉接触检测,它在第三人称相机下接触点被遮挡时依然可用,也省去外部分割/VLM 的开销。
- **扰动采样**:对源关节位姿施加末端变换 $\Delta\rho = {}_{p_s}T_{p_d}$,再解 IK 得到扰动后关节角。无接触阶段用**均匀采样**;接触阶段用**约束优化**(沿用 D-CODA 的约束框架 + Dual Annealing 全局优化),保持接触关系不被破坏。优化目标同时惩罚"扰动过小 / 末端离桌面太近 / 两末端离得太近 / IK 无解"等情况:平移量约束在 $[0.05, 0.1]$ m,无接触旋转约束在 $[-28.7°, 28.7°]$。扰动 $\Delta\rho$ 直接转成扰动动作 $\hat{\mathbf{a}}_t = (\hat{\mathbf{a}}_t^l, \hat{\mathbf{a}}_t^r)$。
- **数据集构造**:复制原数据集,每隔 $k$ 个时间步($t+k, t+2k, \dots$,取 $k=8$)把原状态**替换** 为增广的分布外(OOD)状态,以缓解 behavior cloning 的误差累积;增广集与原数据集合并后训练策略。

### 2.4 多模态与多视角扩展

- **深度增广**:把 PGRIS 的输入从 RGB 换成深度 colormap,并把 ControlNet 的条件改成"RGB 目标图 + 目标位姿图"两张 RGB 拼成 6 通道多条件输入,从而保证生成的深度图与 RGB 几何一致。
- **多视角**:把最多 4 路相机观测**平铺(tile)** 成一张合成图一起生成,以保持多视角一致性(与 GENIMA 观察一致)。

## 三、实验结果

评测基于 PerAct2(建于 RLBench 之上)的 5 个仿真任务(CLB/CLT/CPB/BSR/CPID)与 3 个真机任务(双 UR5 + Robotiq 2F-85 + Intel RealSense D415),共 **2625 次仿真试验 + 300 次真机试验**;策略均为 ACT,成功率对 3 个随机种子取平均。基线含 ACT(无增广)、ACT(+100 条真实示教的"更多数据")、以及新视角合成方法 VISTA(ZeroNVS)。

### 3.1 仿真主结果(Table I,成功率 %)

| 方法 | CLB | CLT | CPB | BSR | CPID |
|---|---|---|---|---|---|
| RGB ACT(无增广) | 41.3 | 10.7 | 43.3 | 21.3 | 17.3 |
| RGB Fine-tuned VISTA | 52.7 | 1.6 | 54.3 | 10.3 | 15.7 |
| RGB ACT(更多数据) | 48.0 | 12.0 | 42.7 | 17.3 | 10.3 |
| **RGB ROPA(本文)** | **68.0** | **30.7** | **62.7** | **24.3** | **30.7** |
| RGB-D ACT(无增广) | 56.7 | 14.3 | 49.7 | 26.7 | 17.6 |
| RGB-D ACT(更多数据) | 33.3 | 9.7 | 47.0 | 20.3 | 16.3 |
| **RGB-D ROPA(本文)** | **72.0** | **35.0** | **61.3** | **29.0** | **35.0** |

关键观察:(1)ROPA 在全部 5 个任务、RGB 与 RGB-D 两种模态下都超过所有基线,在需要精细误差容忍的任务(如抓托盘 CLT、操作绳子 BSR)上优势尤为明显——生成 OOD 状态带来了更强的**误差恢复** 能力;(2)**单纯加数据并不保证提升**("更多数据"在 RGB-D 下甚至比无增广更差),凸显增广质量的重要性;(3)加入深度(RGB-D)相比纯 RGB 有小幅但一致的提升。

### 3.2 多相机 / 泛化 / 消融(均在 CLB 任务,%)

| 多相机(Table II) | CLB | | 泛化(Table III) | CLB | | 消融(Table IV) | CLB |
|---|---|---|---|---|---|---|---|
| 单相机 ACT(无增广) | 41.3 | | Zero-Shot | 50.7 | | Joint Position Only | 46.7 |
| 单相机 ROPA | 68.0 | | Few-Shot(10 demos) | 58.3 | | White Skeleton | 32.0 |
| 四相机 ACT(无增广) | 72.0 | | Fine-tuned(100 demos) | 68.0 | | OpenPose Skeleton | 62.6 |
| 四相机 ROPA | **80.0** | | | | | **ROPA(条纹骨架)** | **68.0** |

消融结论:用**带纹理的骨架位姿图** 明显优于直接喂 14 维关节向量(46.7→68.0),说明骨架表示编码了更丰富的空间上下文与运动学约束;纯白骨架因失去视觉对比与结构线索而大幅退化(32.0);OpenPose 式骨架已很强(62.6),再叠加 GENIMA 条纹纹理进一步升到 68.0。泛化实验中,模型在另外 5 个 PerAct2 任务上训练后直接迁移到 CLB:直接在目标任务上微调效果最好(68.0),但零样本/少样本也能达到合理水平。

### 3.3 真机结果(Table V,20 次试验计成功次数)

| 方法 | Lift Ball | Lift Drawer | Push Block |
|---|---|---|---|
| RGB Fine-tuned VISTA | 11/20 | 1/20 | 18/20 |
| RGB ACT(无增广) | 13/20 | 4/20 | 14/20 |
| **RGB ROPA(本文)** | **17/20** | **10/20** | **20/20** |
| RGB-D ACT(无增广) | 15/20 | 5/20 | 17/20 |
| **RGB-D ROPA(本文)** | **19/20** | **13/20** | **20/20** |

真机上 ROPA 在全部任务显著更稳:基线在球接触瞬间常"冻结"或抓取力过大触发安全限位,或臂对不齐导致松脱;ROPA 抓取更可靠稳定,并在初始条件不利(物体距离更远、朝向更刁钻)时更容易恢复。

**实现细节**:ControlNet 用 Stable Diffusion 2.1,LR 1e-5、weight decay 1e-2、150 epoch、batch 24、512×512,在单张 A100(80GB)上训练,合成用单张 P100;ACT 图像 128×128,单张 4090 训练。

## 四、局限性

- **遮挡下观测受限**:图像生成依赖机器人位姿,在显著遮挡场景中,基于位姿的生成会削弱有效性。
- **依赖相机标定**:骨架渲染需要精确的相机内外参(PyRender 虚拟相机对齐),标定误差会传导到生成质量。
- **扩散模型伪影**:可能生成伪影或质量差的物体;作者指出可通过引入额外条件(如检测到的边缘、分割掩码)缓解。
- **深度增广缺乏基线对比**:作者坦言没有为深度增广给出基线——因为现有深度估计模型(如 DepthAnything)微调需要一批参考图,而这些图本身要靠 ROPA 生成的 RGB 得到,构成循环依赖。
- **动作可行性依赖 IK 与力矩阈值**:约束优化需 LM-IK 判定可行性,接触检测依赖力矩残差阈值,阈值/标定敏感性未充分量化。

## 五、评价与展望

**优点**:(1)思路简洁且迁移得当——把成熟的 Pose-Guided Person Image Synthesis 范式迁到机器人,骨架位姿图 + ControlNet 是一个可复现、几何可控的条件通道;相比 VISTA/RoVi-Aug 只做新视角、不产新动作标签,ROPA 的关键增量是**"图-动作成对增广"**,这正是模仿学习真正需要的。(2)对**第三人称 + 双臂 + RGB-D** 这一相对冷门但实用的设定做了系统覆盖,深度一致性通过多条件 ControlNet 处理得比较干净。(3)"接触/无接触分解 + 约束优化保接触关系"把物理可行性显式纳入,复用了 D-CODA 的约束框架和 Zhang et al. 的力矩接触检测,工程上务实。(4)消融很有说服力,明确回答了"骨架 vs. 裸关节""纹理是否有用"两个关键问题。

**缺点与开放问题**:(1)骨架条件本质是**几何/运动学** 约束,它保证机械臂画得对,却不直接约束**被操作物体** 的物理响应——被抓物体、可变形物、接触后果仍靠扩散模型"脑补",这也是伪影的根源,分割/物理先验条件是自然的下一步。(2)动作标签正确性高度依赖 IK 解与力矩阈值,论文未量化"增广动作真实可执行率",OOD 状态若动作标注有偏,反而可能引入有害监督。(3)与生成式世界模型/仿真 rollout 增广(MimicGen/DexMimicGen 系)相比,ROPA 是**纯 2D 观测层增广**,不产生新的物理轨迹,覆盖范围受限于"末端位姿小扰动 + IK 可解"(平移 0.05–0.1 m),对大范围重规划/新技能无能为力。(4)评测规模偏小(每任务 3 seed、真机 20 试),VISTA 作为唯一生成式基线在若干任务上表现异常低(CLT 仅 1.6),基线是否被充分调优值得存疑。(5)成本上,骨架渲染需 URDF + 标定 + PyRender 管线,迁移到新平台有一定工程门槛。

总体看,ROPA 是"位姿引导生成式增广"方向在双臂 + RGB-D 上一个扎实、务实的落地,最大价值在于**同时给出一致图像与可执行动作标签** 这一闭环;要走向更强,需要把物体级物理一致性、动作标签可靠性验证与更大范围的状态覆盖补齐。
