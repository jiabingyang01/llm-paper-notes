## 一句话总结

CLAP 先用 VQ-VAE 把机器人真实轨迹量化成一套"可执行的动作 token 词表"（Act-VAE），再用 **对比学习** 把无标注人类视频的视觉状态转移对齐到这套物理接地的词表上（VD-VAE），从而绕开传统 Latent Action Model "视觉纠缠"的老问题；在此对齐词表上训练自回归 VLA（CLAP-NTP）并配 Rectified-Flow 动作头 + Knowledge Matching 做低延迟部署（CLAP-RF），在 LIBERO 上取得 97.2 平均成功率、真机双臂五任务均值 62.7，且加入人类视频后 Make Bouquets(OOD) 从 10% 提升到 45%。

## 一、问题与动机

通用 VLA 模型受制于机器人数据相对人类视频的稀缺。用海量无标注人类视频来补充是公认的方向，主流做法是 **Latent Action Model（LAM）**：用逆动力学（inverse dynamics）预测相邻帧之间转移所需的"潜动作"，做自监督学习。

作者指出 LAM 的一个根本缺陷 —— **视觉纠缠（visual entanglement）**：这些潜空间只从视觉动态里学出来，因而容易被背景变化、物体形变等与操作无关的视觉因素污染，编码的不是"操作技能"而是"外观变化"。结果潜空间与机器人物理动作空间不同构（isomorphic），必须靠复杂的后处理才能把视觉潜变量映射到机器人控制上，严重限制了从人类视频到机器人执行的技能迁移。

CLAP 的核心主张：不要让潜动作空间从视觉重建里自生自灭，而是先建立一套由机器人真实轨迹量化出的、物理可执行的动作词表，再用对比学习把人类视频的视觉转移"拉"到这套词表上。这样潜表示天然对齐机器人指令、过滤视觉噪声。

## 二、核心方法

CLAP 训练分三阶段（外加一套部署后训练策略）。机器人本体是双臂 7-DoF，动作向量 $\mathbf{a}_t = [\mathbf{p}^L,\boldsymbol{\theta}^L,g^L,\mathbf{p}^R,\boldsymbol{\theta}^R,g^R]^\top \in \mathbb{R}^{14}$（每臂末端位置 $\mathbf{p}\in\mathbb{R}^3$、欧拉角姿态 $\boldsymbol{\theta}\in\mathbb{R}^3$、夹爪开合 $g\in\mathbb{R}^1$）。

### 1. Act-VAE：把机器人动作量化成"物理语言"

用 Transformer 编解码器的 VQ-VAE 把连续动作段 $\mathbf{a}_{t:t+H-1}$ 编码、经可学习码本 $\mathcal{C}_{act}=\{e_k\}_{k=1}^K$ 最近邻查表得到离散 token 序列。目标函数：

$$\mathcal{L}_{Act} = \|\mathbf{a} - D_\psi(\mathbf{z}_q)\|_2^2 + \|\mathrm{sg}(E_\phi(\mathbf{a})) - \mathbf{z}_q\|_2^2 + \beta\|E_\phi(\mathbf{a}) - \mathrm{sg}(\mathbf{z}_q)\|_2^2$$

（重建项 + 码本项 + commitment 项，$\mathrm{sg}$ 是 stop-gradient）。

**用大白话说**：这一步先给机器人自己的动作造一部"字典"—— 把连续的手臂轨迹压成一串离散词。码本大小 $K$ 和序列长度 $N_q$ 决定这部字典的表达力，作者用一个信息论的率失真（rate-distortion）分析 $r = \frac{N_q\log_2 K}{N_a D_a \log_2(\mathrm{R}/\sqrt{\mathrm{MSE}})}$ 找"肘点"，最终选 $N_q{=}16,\ K{=}256$（重建 PSNR 40 dB）来平衡"重建够准"和"词表够紧凑供 VLM 学"。这部字典之后被冻结，作为后面对齐的锚点。

### 2. VD-VAE：对比学习把视觉转移对齐到动作词表

Vision-Dynamic VQ-VAE 是一个逆动力学模型，把视觉转移 $\mathbf{o}_t\to\mathbf{o}_{t+H}$（用冻结 DINOv3 特征）映射到 **两条解耦的潜流**：动作相关潜 $\mathbf{z}_{v,a}$ 与动作无关潜 $\mathbf{z}_{v,i}$。关键设计是：$\mathbf{z}_{v,a}$ 用 **冻结的 Act-VAE 码本 $\mathcal{C}_{act}$** 量化（强制它落进机器人控制空间），而 $\mathbf{z}_{v,i}$ 用另一套可学习环境码本 $\mathcal{C}_{env}$ 吸收背景等干扰；再用前向解码器重建未来帧特征。

对齐用 SigLIP 式对比损失，把连续的视觉动作潜 $\mathbf{z}_{v,a}$ 与来源相关的参考动作潜 $\mathbf{z}_a^+$ 拉近：

$$\mathcal{L}_{contrastive} = -\log\sigma\!\left(\frac{s_p - b}{\tau_c}\right) - \sum_{j=1}^{M}\log\!\left(1 - \sigma\!\left(\frac{s_{n,j} - b}{\tau_c}\right)\right)$$

其中 $s_p=\mathrm{sim}(\mathbf{z}_{v,a},\mathbf{z}_a^+)$，$s_{n,j}=\mathrm{sim}(\mathbf{z}_{v,a},\mathbf{z}_{a,j}^-)$，$\tau_c$ 为温度、$b$ 为可学习偏置。对机器人样本 $\mathbf{z}_a^+=\mathrm{sg}(E_{act}(\mathbf{a}_{gt}))$ 直接取真值动作的 Act-VAE 编码；对人类视频（无动作标签）则用 EMA 教师编码器对干净转移构造 stop-gradient 伪正样本，在线分支吃增广特征，避免自我对比。再对 $\mathbf{z}_{v,i}$ 加 L1 稀疏约束逼它只吃干扰。总损失：

$$\mathcal{L}_{VD} = \mathcal{L}_{rec}(\hat{\mathbf{f}}_{t+H}) + \lambda_{vq}\mathcal{L}_{VQ} + \lambda_{con}\mathcal{L}_{contrastive} + \lambda_{reg}\|\mathbf{z}_{v,i}\|_1$$

**用大白话说**：给一段人类视频的"前后两帧变化"，模型要判断"这变化对应机器人字典里的哪个动作词"。对比学习像老师拿着机器人真实动作当标准答案，把视觉变化往标准答案上靠；同时把背景、光照这些无关的东西打包塞进另一条无关潜流丢掉。这样人类视频最终被翻译成一串机器人也能执行的伪动作 token $\hat{\mathbf{z}}_a$。

### 3. CLAP-NTP：token 空间的自回归策略

以 Qwen3VL-4B 为骨干，把动作 token 当作语言序列的延续来自回归预测，联合建模子任务与动作索引 $Y=[\mathbf{y}_{sub},\mathbf{z}_a]$。机器人样本用真值 $\mathbf{z}_a$，人类视频用 VD-VAE 推断的伪 token $\hat{\mathbf{z}}_a$：

$$\mathcal{L}_{NTP} = -\sum_{\ell=1}^{L_y}\log P_\theta(y_\ell \mid y_{<\ell}, \mathbf{o}_t, \mathcal{I})$$

因为动作 token 与语言 token 同构，NTP 训练保留了 VLM 的推理与指令跟随能力，还能从人类视频里吸收语义证据实现物体泛化。

### 4. 部署后训练：CLAP-RF + Knowledge Matching

- **CLAP-RF**：自回归逐 token 解码太慢，于是在 NTP 骨干上挂一个 DiT 结构的 Rectified-Flow 动作头 $f^a$，通过交叉注意力查询骨干的 KV 缓存，直接预测低延迟连续动作 chunk，RF 损失 $\mathcal{L}_{RF}=\mathbb{E}\big[\|(\mathbf{a}-\boldsymbol{\epsilon})-f^a(\mathbf{a}^\rho,\rho,\mathbf{h}_t)\|^2\big]$。作者发现融合 VLM 早/中层特征（而非只用高层）对精细动作更好。
- **Knowledge Matching（KM）**：微调易遗忘预训练先验，用反向 KL 把可训练 token 策略锚在冻结参考策略上：$\mathcal{L}_{KL}=\sum_\ell D_{KL}(p_{\phi_{policy}}(y_\ell\mid c_\ell)\,\|\,p_{\phi_{ref}}(y_\ell\mid c_\ell))$，$\mathcal{L}_{post}=\mathcal{L}_{RF}+\alpha\mathcal{L}_{KL}$。
- 两阶段流程：Stage 1 用全部目标数据（机器人 token + 人类伪 token）适配 NTP；Stage 2 因连续动作 chunk 只在机器人数据里，故只用机器人轨迹训 RF 头并加 KM 正则。

**数据配方**：清华精选 AgiBot World Beta（约 100k episodes、1500 小时双臂）、自采 Astribot S1（VR 遥操 27000 episodes、约 50 小时、90 物体）、DROID（约 76000 条，单臂当右臂、左臂补零）、Ego4D 人类视频（取 UniVLA 精选子集 90 小时）。

## 三、实验结果

### 真机双臂五任务（Astribot S1，原始设置，Table I）

|---|---|
| π₀ | 54.0 |
| π₀.₅ | 60.0 |
| UniVLA | 35.0 |

CLAP-RF 取得最高均值。分项上 CLAP-NTP 在指令跟随类任务（Make Bouquets 达 50% C-2）强，CLAP-RF 在需高精度/接触丰富的任务（Pack the Doll 抓取 80→90、Fold T-shirt 30→40、Close 子任务 70）更好，印证"离散 NTP 擅长感知推理、连续 RF 擅长精细控制"的设计假设。

## 四、局限性

1. **纯人类视频学全新任务仍难**：对齐主要捕获高层规划逻辑，对未见活动的精细局部动力学推断力弱，需至少一部分机器人数据接地。
2. **手-爪形态鸿沟**：人手复杂灵巧动作难以无歧义映射到平行夹爪；作者在花束视频里让演示者刻意用"像夹爪"的手势来缓解，反过来说明广义灵巧操作仍是硬伤。
3. **多阶段训练工程复杂**：Act-VAE / VD-VAE / 对比对齐 / 策略头分开训，pipeline 冗长，作者把端到端统一列为未来工作。
4. 估计手姿态基线（用 WiLoR 把手姿态直接回归成机器人 EE 轨迹）不可靠、噪声大甚至拖累策略（Fig 11），侧面支持"用视觉动态学潜动作比直接回归噪声手姿态更可扩展"的设计取向，但也意味着方法回避了"直接可执行手姿态"这条路。

## 五、评价与展望

**优点**：(1) 问题定位精准 —— "视觉纠缠"确实是 LAM 系（Genie/LAPA/UniVLA 一脉）的通病，用"先建机器人动作 VQ 词表、再对比对齐"把因果方向反过来，直觉上比"视觉重建后再事后映射"更干净，消融也验证对比项贡献 −6.2 分实打实。(2) 动作相关/无关潜流的显式解耦 + L1 稀疏，是对 disentanglement 的合理工程化。(3) NTP 负责语义规划、RF 负责精细控制的"双头"分工，兼顾了指令跟随与低延迟，是当前 VLA 落地的务实折衷；KM 反向 KL 抗遗忘也拿到了 OOD +10 的实证。

**与公开工作的关系**：与 UniVLA、LAPA 同属"从人类视频学潜动作"路线，但核心差异在于锚点方向 —— UniVLA/LAPA 的潜空间源自视觉动态本身，CLAP 则以机器人动作码本为锚反向对齐，论文直接以 UniVLA 为主要对照并在真机/LIBERO 上占优。RF+DiT 动作头与 π₀/π₀.₅、FLOWER 的 flow-matching 思路同源；KM 与 Knowledge Insulating（KI）都为抗遗忘，作者实验称 KI 过于严格（LIBERO 仅 88.3），KM 更宽松有效。动作分词上引用了 BEAST、FASTerVQ、Being-H0 等并行工作。

**开放问题与可能改进**：(1) 对齐锚点强依赖机器人动作码本质量，码本一旦不能覆盖某类技能，人类视频里的该技能就无处对齐，跨到差异大的新本体时词表迁移性存疑。(2) 伪正样本用 EMA 教师构造，其质量与对比温度、增广强度耦合，论文未给对温度/EMA 率的敏感性分析。(3) 灵巧手/多指本体上，把动作压进平行夹爪式离散词表会损失自由度，需要更高容量的分层码本。(4) 多阶段 pipeline 若能端到端联合优化 Act-VAE 与对比对齐，有望减少码本与视觉对齐之间的失配。(5) 人类视频的语义收益已被消融坐实，但"语义 vs 灵巧"的分解（Fig 11）显示灵巧收益有限，如何让人类视频也贡献低层动力学而非仅语义，是后续关键。
