## 一句话总结

不改动 VLM 的任何词表、架构或新增网络层，只是把机器人动作编码成空格分隔的整数文本、让 VLM 用原生文本生成能力直接输出，配合动作集成（ensemble prediction）与掩码动作增强（masked action augmentation）两个训练/推理技巧，VLA-0 在 LIBERO 上以 94.7% 的平均成功率超过所有同等（无大规模动作预训练）数据规模的 VLA，并逼近经过大规模预训练的 OpenVLA-OFT（97.1%），在真实 SO-100 机械臂任务上以 60% 对 47.5% 超过预训练过的 SmolVLA。

## 一、问题与动机

当前构建 VLA 的主流路线可归为三类：

1. **离散 Token VLA**（RT-2、OpenVLA）：把连续动作离散化到若干 bin，每个 bin 映射为 VLM 词表中的一个 token，用标准交叉熵训练。缺点是分辨率受词表容量限制（精细控制可能需要成千上万个 bin），且反复复用词表 token 会破坏 VLM 原有的语言语义。
2. **生成式动作头 VLA**（π0、SmolVLA）：在 VLM 之上接一个 diffusion 或 flow-matching 的动作头，VLM 只负责输出隐向量，动作头负责解码为连续动作。这种方式提升了动作保真度，但引入了一个需要从头训练的新网络，容易损害 VLM 原本的语言理解和 grounding 能力。
3. **自定义架构 VLA**（OpenVLA-OFT 的专用 ACT head、π-FAST 基于 DCT 的动作 tokenization）：架构改动更大，训练流程也更复杂。

作者反问：有没有可能完全不碰 VLM 本身，直接把动作（坐标、关节角等）表示成数字字符串，靠 VLM 原生的文本生成能力输出？这一想法此前并非全新——LLARVA 和 HAMSTER 都用文本形式预测过动作，但二者都只是把文本预测用在**中间阶段**（先生成 2D 轨迹规划，再由额外模块转成最终动作），而不是端到端直接预测最终执行动作。VLA-0 的贡献在于证明：只要有合适的训练/推理 recipe，端到端、无架构改动的文本动作预测本身就能做到 SOTA。

## 二、核心方法

**基座模型**：公开可获取的 Qwen2.5-VL-3B（30 亿参数），选择理由是同尺寸下性能强、计算开销小、开放权重便于复现；论文强调方法本身不依赖特定 VLM。

**输入结构**：System Prompt + 图像（仿真中用第三人称视角 + 手腕相机图像；真实实验中用左右两个相机图像）+ 任务指令文本（如 "put the banana on the plate"）。System Prompt 明确规定输出格式：分析输入图像，预测未来 $H$ 个时间步的动作，每个动作 $D$ 维，输出 $H\times D$ 个取值范围 $[0,B]$ 的整数，空格分隔，不要输出任何其它内容。$H,D,B$ 依数据集设定。论文还比较了把多路图像拼接成一张复合图 vs. 作为独立输入两种方式，发现性能几乎无差异。

**动作解码（离散化为整数文本）**：连续动作先归一化到固定整数区间 $[0,B]$，VLM 为每个动作维度生成一个整数。直觉上可以写作

$$a_i^{\text{int}} = \mathrm{round}\!\left(\frac{a_i - a_{\min}}{a_{\max}-a_{\min}} \times B\right)$$

用大白话说：这一步就是把连续的物理量（比如末端执行器坐标）换算成"格子编号"，让 VLM 把它当作普通的数字文本来续写。和离散 token VLA 不同，这里提高分辨率只需要增大 $B$（例如从 250 提到 1000），完全不需要扩充 VLM 词表——因为数字本身就是已有词表里的普通 token。

**动作集成（Ensemble Prediction）**：借用 ACT（Action-Chunking Transformer）的做法。VLM 每次推理会一次性预测未来 $n$ 步的动作序列，因此对同一个时刻 $t$，会分别在第 $t, t-1, \dots, t-n+1$ 次推理中都被预测到过（作为该次输出序列中的第 $1, 2, \dots, n$ 个动作）。最终执行的动作取这些历史预测的平均：

$$\hat{a}_t = \frac{1}{n}\sum_{k=0}^{n-1} a_t^{(t-k)}$$

其中 $a_t^{(t-k)}$ 表示第 $t-k$ 步推理时对时刻 $t$ 做出的预测。大白话说：与其只信任"刚做出的这一次"预测，不如把过去若干次推理里都覆盖过同一时刻的预测做平均，相当于滑动窗口投票，抑制单次自回归采样的抖动噪声。

**掩码动作增强（Masked Action Augmentation）**：VLM 是自回归生成的，后面的 token 天然依赖前面已经生成的 token。如果不加干预，模型可能学会"看到前几个数字就顺着数列自动补全"这种捷径，而不去认真依赖图像观测和语言指令。训练时对目标动作字符串随机遮蔽字符，强迫模型在生成每一个数字时都重新基于视觉/语言条件推理，而不是单纯做数列续写。

**训练细节**：对基座 VLM 做全参数微调，标准交叉熵 loss 覆盖整个词表；Adam 优化器，64 epoch，batch size 192，学习率 5e-6；8 张 A100 上训练约 32 小时。

**评测设置**：仿真用 LIBERO（Spatial/Object/Goal/Long 四个 suite，每 suite 10 个任务，每任务 50 个 episode）；真实世界用 SO-100 机械臂 + LeRobot 框架，4 个任务（block 重定向、推苹果、香蕉 pick-place、纸杯蛋糕 pick-place），每任务采集 100 条演示。真实推理在 RTX 5090 上用标准 PyTorch 实现，流式输出，速度约 4 Hz；为简化实现，真实实验没有做动作集成（因为要并行跑 8 个模型实例）。

## 四、局限性

论文 Conclusions and Limitations 一节明确指出两点未来方向：

- 尚未验证 VLA-0 在**大规模动作数据预训练**下的表现——目前所有对比都基于同等（无大规模预训练）数据量，VLA-0 能否像 π0.5-KI、OpenVLA-OFT 那样从大规模预训练中进一步获益仍是未知数。
- **推理速度**是明显短板：标准 PyTorch 实现在 RTX 5090 上仅 4 Hz，且真实实验中因动作集成需要 8 个并行模型实例而没有启用，作者留给量化、蒸馏等优化手段作为未来工作。

此外，从论文本身数据也能观察到：真实世界评测只覆盖 4 个任务、每任务 100 条演示，样本规模有限；且仅在单一骨干（Qwen2.5-VL-3B）上做了完整消融，未验证结论在其他 VLM 骨干上是否同样成立。

## 五、评价与展望

**优点**：VLA-0 的核心贡献不是提出新算法，而是用扎实的消融实验证明了一个此前被学界忽视的"最简假设"——直接复用 VLM 原生文本生成能力做动作预测——只要配合恰当的 recipe（动作集成 + 掩码动作增强 + 合适的分辨率），就足以匹配甚至超过专门设计的动作头或离散 token 方案。零架构改动意味着完整保留了 VLM 预训练时学到的语言理解和视觉 grounding 能力，理论上对下游泛化和多任务迁移更友好；同时工程实现和复现成本也远低于需要额外训练生成式动作头的方案。

**与其他公开工作的关系**：相比 RT-2/OpenVLA 式离散 token 方法，VLA-0 用"整数文本 + 大分辨率区间"替代"新增词表 token"，规避了分辨率-词表容量的权衡，也不会污染原词表语义；相比 π0/SmolVLA 式生成式动作头，VLA-0 不需要额外训练一个 diffusion/flow-matching 网络；相比 OpenVLA-OFT 的定制 ACT head 或 π-FAST 的 DCT tokenization，VLA-0 的架构侵入性最低。与最相近的 LLARVA、HAMSTER 相比，二者都只用文本预测中间的 2D 轨迹规划，而 VLA-0 首次证明端到端、直接生成最终执行动作的文本路线本身即可达到 SOTA，这是其区别于既有文本动作类工作的关键点。

**开放问题**：其一，当对比方法也叠加大规模预训练后（如 OpenVLA-OFT 全表排名 1.5），无预训练的 VLA-0 只能屈居第二，说明该简单方案在数据规模进一步扩大后是否仍保持相对优势尚待验证，这也是作者自己列出的首要未来方向；其二，自回归逐 token 生成天然比并行输出的动作头慢，4 Hz 的推理速度对不少高频精细操作任务可能不够用，这是文本动作路线的结构性代价，蒸馏、投机解码、并行解码等工程手段能在多大程度上弥补值得关注；其三，论文未给出与 LLARVA/HAMSTER 等其他文本动作方法的直接定量对比，动作集成、掩码动作增强这两个 recipe 组件相对于其他文本路线方法的增量收益仍不明确；其四，真实世界评测任务数量和场景多样性有限，结论的稳健性有待更大规模、更多任务类型的验证。总体而言，这篇论文对"更复杂的动作表示/架构才能带来更好性能"这一领域先验提出了一个有分量的反例，具有启发性的科学价值。
