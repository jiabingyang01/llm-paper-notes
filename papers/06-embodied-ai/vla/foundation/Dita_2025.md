## 一句话总结

Dita 用一个从零训练的因果 Transformer 直接对带噪动作 token 去噪,并让语言 token、历史图像 patch token 与带噪动作 token 在同一序列中做"上下文条件化"（in-context conditioning）交互,而不是像以往扩散策略那样先融合出单个 embedding 再喂给外挂的小型 MLP/UNet 扩散头;仅 334M 参数(221M 可训练),在 SimplerEnv、LIBERO、CALVIN、ManiSkill2 四个仿真基准上取得 SOTA 或有竞争力的结果,并在真实 Franka 机械臂上以 10-shot LoRA 微调完成复杂长时序、非目标物体扰动、光照变化等场景任务。

## 一、问题与动机

现有 VLA 策略的两条主流路线各有局限:

1. **离散化动作头**(如 RT-1/RT-2、OpenVLA)把连续动作离散化为 token,由自回归 Transformer 逐 token 预测,表达力受限,且对跨本体、跨动作空间的异构数据适应性差。
2. **扩散动作头**(如 Octo、π0)用小型 MLP/UNet 作为去噪网络,只 condition 在单个融合 embedding 上——历史观测和语言先经过 VLM/Transformer 融合成一个 embedding,再喂给 tiny diffusion head 做去噪。这种"早融合"方式使去噪过程难以捕捉动作的细粒度变化(action deltas)和环境的细微差异(如非目标物体摆放、光照变化),而 tiny diffusion head 在大规模跨本体数据集巨大的动作/相机/任务/场景多样性下容易成为表达力瓶颈。

Motivation:既然 Transformer 架构在 NLP/CV 中已展现出可扩展性,能否让扩散去噪过程本身由一个大 Transformer 直接完成,并让历史图像 token 与带噪动作 token 在同一因果序列内做细粒度对齐,从而增强跨具身、跨相机视角、跨场景、跨任务、跨动作空间数据集上的鲁棒性,并支持复杂长时序任务?

## 二、核心方法

### 架构总览

Dita 是一个"上下文条件化"的 Diffusion Transformer(DiT)策略,仅接受第三人称相机图像和语言指令作为输入:

- **多模态 tokenization**:语言指令由冻结的 CLIP 文本编码器 tokenize;图像观测由 DINOv2 抽取 patch 特征,并与 Dita 主干联合端到端微调(因为 DINOv2 训练于网络数据而非机器人数据,分布有偏移)。
- **Q-Former**(深度 4 层、从零训练)在每个 block 内以 FiLM 方式注入语言 token,依据指令上下文将 DINOv2 patch 特征压缩查询为长度 32 的 token 序列,降低计算开销。
- **Sin-PosEmb** 对扩散时间步做正弦位置编码。
- **动作表示**:7 维向量(3 维平移 + 3 维旋转向量 + 1 维夹爪状态);为对齐图像/语言 token 的维度,连续动作向量被补零,扩散去噪过程中仅对这 7 维动作本体加噪。
- **核心设计**:语言 token、图像特征、时间步嵌入被拼接在序列最前部,随后是带噪动作 token,整体送入一个 LLaMA2 风格、从零训练的因果 Transformer(12 层自注意力,hidden size 768)直接对多 token 动作块(action chunk)去噪。这种 in-context conditioning 使去噪可以直接以图像 patch 为条件,让模型捕捉历史观测中的细微动作差异,区别于以往方法先融合 embedding 再用浅层网络(MLP/AdaLN-DiT 头)condition 的方式。

### 训练目标

去噪网络记为 $\mathcal{E}_\theta(c_{lang}, c_{obs}, t, \boldsymbol{x}^t)$,其中 $c_{lang}$ 是语言指令、$c_{obs}$ 是图像观测、$t \in \{1,\dots,T_{train}\}$ 为去噪时间步。训练时对动作 $\boldsymbol{a}$ 采样高斯噪声 $\boldsymbol{x}^t \sim \mathcal{N}(\boldsymbol{0}, \boldsymbol{I})$ 加噪得到带噪动作 token,网络 $\mathcal{E}_\theta$ 学习预测所加噪声,优化目标是预测噪声与真实噪声之间的 MSE。用大白话说:整个训练就是标准的 DDPM 去噪目标,区别只在于"预测噪声的网络"本身就是那个统一处理图像/语言/动作 token 的因果 Transformer,而不是外挂的小型 MLP。

推理阶段迭代 $N_{eval}$ 步:

$$\boldsymbol{x}^{t-1} = \alpha(\boldsymbol{x}^t - \gamma \mathcal{E}_\theta(c_{lang}, c_{obs}, t, \boldsymbol{x}^t) + \mathcal{N}(\boldsymbol{0}, \sigma^2 \boldsymbol{I}))$$

用大白话说:这是标准 DDPM/DDIM 采样公式,$\alpha,\gamma,\sigma$ 由噪声调度器给定,每一步用网络预测的噪声修正当前带噪动作、并加入少量新噪声,迭代 $N_{eval}$ 次后得到最终去噪后的动作块。

### 训练与预训练细节

预训练用 DDPM 目标,$T_{train}=1000$;零样本评估时用 DDIM 加速到 $T_{eval}=20$ 步。根据 ManiSkill2 上的预实验,采用 2 帧历史观测、预测 16 步动作块。AdamW 优化 100,000 步,因果 Transformer 和 Q-Former 学习率 1e-4、DINOv2 学习率 1e-5;batch size 8192(32 张 NVIDIA A100、每卡 256 样本);预训练数据为 OXE 数据集(选取与权重方式与 OpenVLA/Octo 一致)。模型总参数量 334M,可训练参数 221M(CLIP 文本编码器冻结)。

四个仿真基准分工不同:SimplerEnv 做 OXE 预训练模型的 zero-shot 评估;LIBERO 评估单相机下的微调适配性;CALVIN 评估长时序语言条件任务;ManiSkill2(自建 4 万余条轨迹)评估新相机视角泛化,从零训练。

### 架构消融(附录)

- **层数**:2/6/12/24 层因果 Transformer 在 CALVIN(不做 OXE 预训练)上的 Avg.Len 分别为 1.84/2.21/2.56/2.61,而参数量相当、采用 AdaLN 条件化的 DiT(类 Diffusion Policy 结构)仅 1.45,验证了 in-context conditioning 相比 AdaLN 条件化方式的优势。
- **去噪步数**:DDIM 推理步数从 20 降到 10 时在 SimplerEnv 上性能几乎不掉(Pick Coke variant:85.5%→85.3%),仅 2 步时仍有 70.4%,表明 Dita 的 in-context conditioning 并未牺牲推理速度。
- **收敛速度**:相同参数量下,Dita 在 OXE 预训练阶段的 MSE loss 收敛明显快于扩散头基线。
- **图像增强**:CALVIN 上加入 ColorJitter 等图像增强后 Avg.Len 从 3.61 提升到 3.78;LoRA 微调下仅约 11M/221M 参数可训练,容量不足以充分吸收增强带来的扰动,全参数微调在极端环境变化下的成功率比 LoRA 高约 20 个百分点。

### 真机实验

平台:Franka Emika Panda + Robotiq 2F-85 夹爪 + RealSense D435i(第三人称,距机器人约 1.5m);对 OXE 预训练模型用 LoRA 做 10-shot 微调(20,000 步,batch 512,DDPM 100 timesteps)。任务涵盖 Pick&Place(香蕉/猕猴桃)、Pour(倒咖啡豆/倒水)、Stack(叠碗/俄罗斯套娃)、Pick&Rotation、Pull&Push(开关抽屉)以及若干 3 步以上的长时序任务(如"打开顶层抽屉、放入方块、关闭抽屉")。整体上 Dita 在两步任务上达到 63.8% 成功率(第二阶段贡献约占一半),全面优于 Octo、OpenVLA 和同架构的扩散头基线;在包含 3D 旋转、非目标物体扰动、背景/光照变化的场景下依然保持较强鲁棒性,而 Octo/OpenVLA 常在抓取位置或姿态判断上失败。

## 四、局限性

- 模型规模仅 334M(221M 可训练),作者在附录中坦言这一容量不足以充分消化整个 OXE 数据集的多样性,导致在 LIBERO 上不如用更贴近下游分布的 Droid 数据集预训练(63.8% vs 77.9%);扩大模型规模留作未来工作。
- 仅使用第三人称单相机输入(无腕部相机、无深度、无本体感知)。这被作者当作卖点(证明极简输入下也能取得强性能),但添加腕部相机后 LIBERO 平均成绩从 82.4% 大幅提升到 92.3%,说明单一第三人称视角本身仍是明显的性能瓶颈。
- LoRA 微调容量有限(约 11M/221M 可训练参数),难以充分适应真实环境中的图像增强扰动;全参数微调虽表现更好,但训练成本显著更高。
- 因果 Transformer 中观测帧数超过 2 时收敛困难、性能骤降,说明当前架构对更长历史上下文的扩展性有限,需要额外工程手段。
- 失败案例分析显示模型抓取失败后虽可重试,但仍难以修正到正确位置;倒水等动态操作中因运动不稳定而洒出,说明对精细动态操作的鲁棒性仍有限。
- 真机 10-shot 评估每类任务仅 10 条演示,样本量偏小,统计稳健性有限。

## 五、评价与展望

Dita 的核心贡献是把"扩散去噪"与"Transformer 主干"更彻底地融合:不再像 Octo、π0 等工作那样用一个独立的小型 MLP/UNet 扩散头 condition 在单一融合 embedding 上,而是让带噪动作 token 与历史图像 patch token、语言 token 在同一因果注意力序列中交互,实现细粒度、逐 token 的上下文条件化。这一设计使模型能直接"看到"图像 patch 级别的细节来判断动作增量,概念上与将 diffusion 和 Transformer 结合的 3D Diffuser Actor 等工作一脉相承,但 Dita 面向的是跨具身、跨相机、跨任务的大规模异构预训练场景,给出了一个开源、轻量(334M)的基线——这在 VLA 领域普遍朝数十亿参数规模扩展(如 OpenVLA-7B)的背景下颇具价值,说明架构设计而非单纯堆参数同样能提升样本效率和跨域鲁棒性。

与同期扩散类 VLA 工作相比:相较于采用交叉注意力 Transformer 或 DiT 头的双臂操作方案(如 RDT-1B),Dita 的 in-context conditioning 更彻底地取消了"先融合、再喂给去噪头"的中间步骤;相较于基于 PaliGemma 大型预训练 VLM 主干做流匹配(flow matching)的方案(如 π0),Dita 采用从零训练的 LLaMA 风格因果 Transformer,这既是其"轻量、简洁"卖点的来源,也是其容量瓶颈的根源——没有借助大规模视觉语言预训练的语义先验,可能是其在开放词汇泛化、复杂指令理解上不及以 VLM 为基座的 VLA 的潜在原因,但论文并未直接给出这类语义泛化对比实验,是一个开放问题。

其他可能的改进方向:(1)扩大 Transformer 规模并更充分利用 OXE 全量数据,能否进一步拉开与离散化/浅层扩散头方法的差距,值得后续验证;(2)当前架构仅接受单一第三人称视角,如何在保持轻量的同时融合腕部相机、深度或本体感知信息仍需专门设计;(3)观测帧数超过 2 时的收敛性问题,以及更长历史上下文与因果 Transformer 计算成本之间的权衡,是有待解决的工程问题;(4)真机实验样本量偏小,更大规模、更多任务类型的真机对比会让结论更具说服力。
