## 一句话总结

Embodied-R1.5 用单一 8B 参数模型统一了空间认知、任务规划与纠错、指向定位三大具身推理能力，配合三条自动化数据构建流水线（超 15B token）、"SFT+多任务平衡 RL"两阶段训练，以及一个由同一模型兼任规划者/定位者/纠错者的 Planner-Grounder-Corrector（PGC）闭环框架，在 24 个具身 VLM 基准中的 16 个取得 SOTA（21 个主榜均分 70.4%，超过 Gemini-Robotics-ER-1.5 17.0 个百分点），且仅用少量动作数据微调成的 Embodied-R1.5-VLA 在 SimplerEnv Google Robot Visual Matching 上达到 92.4%，超过 π0.5 的 72.7% 逾 20 个百分点，验证了"内化的具身推理可部分替代大规模动作预训练"这一核心论点。

## 一、问题与动机

论文开篇引用王阳明"知行合一"（"Reasoning initiates the action; Action fulfills the reasoning"）作为立论隐喻：大语言模型已在数字世界取得成功,但将智能真正落地到物理世界(即弥合"seeing-to-doing"的鸿沟)仍是核心挑战。作者认为实现通用物理智能需要一个 Embodied Foundation Model（EFM）,将感知、推理、执行统一在单一架构内,并归纳出三条能力维度:空间认知与推理(理解物理世界的语义与空间结构)、任务规划与纠错(组织执行逻辑、监控进度并纠错)、具身指向与定位(将高层推理落地为坐标与轨迹)。

论文指出现有工作存在三个根本瓶颈:

1. **能力碎片化**:多数模型只覆盖部分谱系——有的专注认知(如 Sermanet et al. 的 RoboVQA)、有的专注规划纠错、有的专注定位;即便像 RynnBrain 这类尝试跨维度的工作,也依赖不同规模的多个独立模型分别处理不同任务,而非在单一模型内真正统一。
2. **多任务冲突**:长文本推理、轨迹预测、多样化定位任务的输出格式差异巨大,联合训练时不同能力会相互侵蚀,收敛困难。
3. **长程任务上缺乏闭环自主验证**:现有 EFM 大多停留在 Embodied QA 层面,未验证推理能力是否真正在长程复杂决策中物理落地。

Embodied-R1.5 是作者前作 Embodied-R1(一个纯指向能力的专才模型)的延伸——从"指向专才"跃升为统一三大能力维度的综合 EFM,系统性地解决上述三个瓶颈。

## 二、核心方法

### 2.1 统一能力分类体系

三大能力维度被形式化为一条从感知到决策再到执行的渐进推理链:

- **具身认知与空间推理**:细分为空间关系理解(相对方位、支撑、包含、遮挡)、度量空间推理(距离/尺寸/空闲空间估计)、3D 场景感知(单目深度、跨视角几何一致性)、物体与场景认知。
- **具身规划与纠错**:覆盖任务全生命周期——长程任务分解、下一步规划、过程检测(判断子任务是否完成)、错误定位、错误纠正(生成回退/重规划建议)。
- **具身指向与定位**:四种类型——REG(指代表达定位,给定自然语言描述输出物体坐标)、RRG(区域指代定位,如"能放一个盘子的空区域")、OFG(物体功能定位,定位把手/壶嘴/按钮等功能部件)、VTG(视觉轨迹生成,同时支持 object flow 与 end-effector flow 的 2D/3D 轨迹点序列)。

### 2.2 模型架构

Embodied-R1.5 以 Qwen3-VL-8B-Instruct 为骨干,是一个 8B 参数的 VLM。所有输出(包括坐标、轨迹)统一表达为纯文本 token 序列,坐标归一化到 $[0,1000]$ 区间,轨迹表示为有序坐标序列。作者刻意不引入额外的坐标专用 token,直接以纯文本数字输出坐标,使坐标可作为可复用符号在多步推理链中被反复引用,同时保持视觉锚定。

模型可进一步扩展为直接输出连续动作的 **Embodied-R1.5-VLA**:在 VLM 骨干上挂接一个基于 DiT 的轻量 flow-matching 动作专家,构成双系统架构(System 2 负责推理、System 1 负责动作生成)。动作专家从 VLM 中间层提取视觉-语言特征作为交叉注意力上下文,结合 32 个可学习的未来 query token,通过 action chunking 生成 7 维连续动作轨迹。作者的核心假设是:当 VLM 已充分内化具身推理能力后,从"理解到的意图"映射到"动作"的学习相对简单,更强的具身骨干应显著降低下游动作学习对数据量的需求。

### 2.3 三条自动化数据构建流水线

数据系统整合 34 个数据集、总规模超 15B token。三条流水线针对性填补关键能力缺口:

**流水线一(空间推理,ER1.5-Spatial,约 2 万样本)**:输入 Fractal、BridgeData V2、DROID 等真机数据集的单张 RGB 图,全自动重建结构化 3D 语义场景图。具体流程为:双后端语义理解(Qwen3-VL 生成开放域语义描述 + RAM++ 提供更广类别覆盖)并经 VLM 语义归一化模块合并同义词组;度量尺度单目几何估计使用 MoGe-2 联合预测稠密深度图、表面法线图与相机内参;开放词表实例分割用 Grounded-SAM-2(GroundingDINO 生成候选框、SAM2 精修掩码);像素通过逆透视投影
$$\mathbf{p}^c = D_i(u,v) \cdot \mathbf{K}_i^{-1}[u,\,v,\,1]^\top$$
提升到相机坐标系下的 3D 点(用大白话说:已知每个像素的深度值和相机内参,就能把 2D 像素反算回 3D 空间中的真实点);再通过 RANSAC 在法线图上找主平面(桌面)并做重力对齐的水平面矫正,构造相机到世界坐标系的刚体变换,最终从场景图程序化生成空间推理 QA 对。每一阶段都嵌入了多重质检(检测置信度、掩码面积过滤、桌面高度一致性校验等)。

**流水线二(规划与纠错,ER1.5-Correction,约 80 万样本)**:针对现有数据集只有成功演示、缺乏结构化失败标注的问题,借鉴 RoboFAC、Guardian 的失败分类框架,沿"阶段"(规划失败 vs 执行失败)与"认知层级"(失败检测/定位/纠正)两个正交维度构建数据,产出六类 QA。规划失败通过五种结构化扰动算子(步骤删除、重复、交换、物体替换、动作/位置替换)对正确子任务计划注入错误,并同时生成三个认知层级的 QA、保留原始正确计划作正样本防止过拟合。执行失败通过视频截断模拟中断、描述替换模拟物体/动作错配、ManiSkill 中的物理引擎扰动注入模拟操作失败三种策略构造。数据来源覆盖真机(BridgeData V2、RoboFail)与仿真(ManiSkill、GEMBench)场景,全部经人工抽样复核。

**流水线三(功能定位与轨迹,ER1.5-Pointing,约 40 万样本)**:OFG(物体功能定位)数据一方面从仿真环境(ManiSkill-PartNet、PRISM)零成本渲染获取部件级标注,另一方面从大规模人-物交互数据中提取人手抓握模式作为功能部位的监督信号;此外借鉴 CAPTURE 构造 RegularRearrangement 数据(呈现规律排列图案但缺 1-2 个元素,要求模型推理应指向哪里补全图案),将空间推理与指向能力结合。VTG(轨迹生成)数据针对有 3D 末端位姿元数据的数据集(如 DROID)直接将 3D 轨迹投影到 2D 图像平面,针对无元数据的数据集则微调一个基于 Detectron2 的末端执行器检测器来追踪抓手运动,物体轨迹借助 CoTracker3 追踪被操作物体,并利用 RoboCasa 场景资产生成关节物体的 3D 轨迹数据。作者特别指出:轨迹语义所需的交互动作语义不依赖高保真渲染("语义泛化"),因此低保真仿真数据仍能提供有效的运动语义监督。

### 2.4 两阶段训练:SFT + 多任务平衡 RFT

**Stage 1 SFT**:在 Qwen3-VL-8B-Instruct 上全参数微调,标准因果语言建模目标,所有异构能力输出(自然语言推理、坐标、轨迹序列)统一表示为 token 序列联合优化。AdamW、bfloat16、峰值学习率 $2\times10^{-6}$ 余弦衰减、10% warmup、有效批量 512、最大上下文 8192 token,视觉编码器与 LLM 骨干联合训练(不冻结),训练 1 个 epoch。

**Stage 2 RFT**:作者观察到 GRPO 虽擅长单任务 RFT,但异构多任务训练引入两类互补的失衡——组内(intra-task):组级标准差归一化会偏向低方差样本,难度适中、学习信号最强的样本反而未被充分优化;组间(inter-task):不同能力量级悬殊的奖励尺度会让高密度任务主导梯度。为此设计了一套组合方案:

1. *难度感知数据过滤*:用 SFT checkpoint 的 rollout 通过率过滤训练语料,保留约 20 万条中等难度样本,最大化学习信号。
2. *动态过滤*:当一组内所有 rollout 获得相同奖励(无判别性)时自动检测并屏蔽该组,确保梯度只来自具有奖励区分度的样本。
3. *全局批次奖励归一化*:标准 GRPO 按组级标准差归一化,无法消除跨任务的奖励尺度差异(每组仅来自单一 query);EMA-GRPO 引入了按任务的移动平均,但在数据量差异悬殊时统计量不稳定。本文改为
$$\hat{A}_i = \frac{R_i - \mu_{\mathrm{group}}}{\sigma_{\mathrm{batch}} + \epsilon}$$
(用大白话说:组内均值决定"这条 rollout 在同组里排第几"这个相对顺序,但用整个混合批次的标准差去统一梯度幅度,这样既不丢失组内相对排序信息,也不需要任务标签或历史状态就能让不同任务的梯度量级可比。)
4. *多任务奖励设计*:五类奖励族匹配不同输出结构——精确匹配(0/1,选择题/数值题/数学题)、IoU 奖励(空间定位框)、点距离奖励(点定位,分段线性衰减)、轨迹 RMSE 奖励(2D 轨迹用逐点 RMSE、3D 深度维用 MAE 独立评估后加权平均)、语义相似度奖励(开放式文本,默认用 Skywork-Reward-V2-Qwen3-4B 打分,推理失败时退化为 BLEU)。所有连续型奖励采用统一的分段线性衰减框架:给定距离度量 $d$ 和阈值对 $(\tau_p,\tau_z)$,
$$\phi(d;\tau_p,\tau_z) = \mathrm{clip}\!\left(\frac{\tau_z-d}{\tau_z-\tau_p},\ 0,\ 1\right)$$
(用大白话说:预测足够准时给满分,差得离谱时给零分,中间线性插值,这样比 0/1 二元奖励能提供更稠密的梯度信号。)
5. *自适应思考长度*:仅约束输出需被 answer 结构标签包裹,不强制显式推理链。由于 RL 奖励只评价最终答案质量而不奖励中间推理,模型会自发学习按需分配算力:感知类简单任务(如指向)额外的推理 token 不带来奖励增益,模型学会以近零推理开销直接输出坐标;复杂规划任务中充分思考能显著提升答案质量因而获得更高奖励,模型自发生成结构化思考过程。这种涌现的自适应计算分配恰好契合具身场景的实际需求(高频定位任务要求实时响应,低频规划任务可从更充分推理中获益)。

格式奖励与准确率奖励按 $R=(1-\lambda)R_{\mathrm{acc}}+\lambda R_{\mathrm{fmt}}$、$\lambda=0.1$ 加权组合。训练细节:改进版 EasyR1 框架、clip ratio 下/上限 0.2/0.28、KL 正则采用 K1 无偏估计直接作为奖励惩罚项($\beta=0.01$)、每条 prompt 采样 $n=8$ 条并行 rollout、AdamW 学习率 $3\times10^{-6}$、梯度裁剪 1.0,训练 2 个 epoch。

### 2.5 Planner-Grounder-Corrector(PGC)闭环自主框架

为验证三大能力是否能真正在物理世界闭环落地,作者设计了极简的 PGC 框架:单个 Embodied-R1.5 实例作为统一推理服务,异步调用其不同能力接口驱动整套自主执行栈,不依赖多模型级联或多智能体编排。以"做一杯奶茶"为例:**Planner**(高层)接收用户指令与当前图像观测(及可选的外部上下文,如任务 SOP 文档),先调用规划能力将长程任务分解为结构化子任务序列,执行中每完成一步再做下一步规划、动态调整后续安排;**Grounder**(低层)针对每个子任务自适应组合调用指向能力接口(例如"按下奶茶机开关"需先用 OFG 定位开关这一功能部件,再用 VTG 规划按压轨迹),生成精确空间定位指令传给低层技能执行器;**Corrector** 以异步查询方式持续评估执行状态(基于当前观测与 Memory 中的历史信息),输出 SUCCESS / PROCESS(仍在执行,继续等待)/ FAIL 三种判定。整个任务维护一个简单先进先出(FIFO)的 **Memory** 模块,按固定频率采样图像并记录每个子任务的状态标注。当 Corrector 检测到失败,错误信息写入 Memory 并反馈给 Planner,触发针对该子任务的重试或自适应重规划。因为 Planner、Grounder、Corrector 三个角色均由同一个模型服务,Corrector 可直接复用 Planner 阶段建立的推理上下文与 Grounder 阶段的空间理解,避免了级联系统固有的信息损失、实现更精确的错误归因。PGC 本身只是一个轻量、无状态的控制流调度与 Memory 管理层,不包含任何推理逻辑或启发式规则——所有智能决策均由 Embodied-R1.5 内化的能力驱动,框架能力上限完全取决于模型能力而非系统工程复杂度。

### 2.6 EmbodiedEvalKit

为解决具身 VLM 评测缺乏统一框架(现有 VLMEvalKit、lmms-eval 面向标准 VQA/captioning,无法解析坐标、边界框、轨迹序列等具身特有输出格式)的问题,论文开源了四层模块化评测框架 EmbodiedEvalKit:数据层将 25+ 基准统一为 HuggingFace Parquet 格式;推理层提供模型无关接口(支持 vLLM、HF Transformers、API 后端);解析层将异构定位输出归一化为统一结构化表示;评测层按能力类别计算官方指标并聚合。论文所有实验均通过该统一流水线复现,报告了 24 个基准、13 个基线的结果。

## 三、实验结果

### 3.1 具身 VLM 基准(21 个主榜,均分 70.4%,SOTA 16/24)

|---|---|---|---|---|---|
| Gemini-Robotics-ER-1.5 | 32.6 | 32.0 | 53.7 | 47.0 | 41.3 |
| Mimo-Embodied(最优基线) | 62.0 | 43.0 | 56.8 | 54.6 | 54.1 |
| Embodied-R1(前作) | 51.8 | 26.5 | 55.2 | 40.2 | 43.4 |
| **Embodied-R1.5** | 61.0 | 53.8 | **69.3** | **77.2** | **65.3** |

**指向与定位(表 2,均分,9 个基准)**:Embodied-R1.5 均分 **72.8**,在全部 9 个指向基准上均为 SOTA,较最强基线 Gemini-Robotics-ER-1.5(52.4)高出约 20 分。视觉轨迹(表 4)上,VABench-V RMSE 7.00、ShareRobot-V RMSE 15.79,均优于 Embodied-R1(9.04 / 23.20)。

**认知与空间推理(表 3,均分,8 个基准)**:Embodied-R1.5 均分 **70.2**,领先最强通用 VLM Gemini-2.5-Pro(68.8)与最强具身基线 Mimo-Embodied(65.6),该维度上通用 VLM(Gemini-2.5-Pro、GPT-5.4)本身已较强,优势margin相对最小。

**通用能力保留(表 5)**:7 个标准视觉基准上相对 Qwen3-VL-8B 基座几乎无退化,MMMU(54.8 vs 52.0)和 ScienceQA(93.1 vs 92.3)反而提升,验证具身训练未牺牲通用视觉理解。

### 3.3 零样本操作迁移(不用目标域训练数据)

- **ManiSkill-Affordance**(关节物体功能定位,ManipLLM 协议):可见类别成功率 0.76(ManipLLM 0.59,+17pp),未见类别 0.66(ManipLLM 0.54,+12pp)。
- **RoboTwin2.0**(demo_randomized,7 项双臂任务):Embodied-R1.5 **完全零样本**(训练中未用任何 RoboTwin 数据,仅靠指向预测+统一运动逻辑执行)达到均分 **65.0%**,逼近用 400 条演示微调的 π0.5(72.9%),大幅超过微调后的 π0(15.0%)和 RDT(12.0%);在 Click Bell 任务上零样本达 99%,反超微调 π0.5 的 66%。

Cup Disassembly(需将叠放的杯子逐一分离、不碰倒其余堆叠)所有基线均为 0%,Embodied-R1.5 靠任务分解+逐杯精确指向做到 66.6%。

### 3.5 长程闭环演示与消融

PGC 框架在 ARX Lift2s / Realman RM75 双臂平台上零样本完成"做奶茶"(10 步)、"三层叠杯"(6 步)、"循环扫垃圾"、"开放词表货架拣货"四类长程任务,并在人为主动扰动(反复挪动目标盘子/物体)下展示自主重定位与重执行能力,全程无需人工干预或重置。

消融(§7.5):RFT 相对 SFT-only 在三个维度均有提升(Planning +2.7pp、Pointing +3.8pp、Cognition +1.3pp,表 14);以 Embodied-R1.5 作为骨干相较原始 Qwen3-VL-8B,在 LIBERO 上收敛速度显著更快(1 万步:92.0% vs 82.0%,+10pp)且收敛后仍领先(8 万步:97.0% vs 96.0%),在 GR00T 与 OpenVLA-OFT 两种动作专家上结论一致;RL 配方消融显示"难度过滤+动态过滤+全局批次归一化"的完整方案在三维度上取得最佳综合平衡,优于 vanilla GRPO 与 EMA-GRPO。此外论文用配套技术报告 ForceFlow 展示了视觉-力觉切换(approach 阶段靠 Embodied-R1.5 的视觉泛化能力,接触阶段切换到力觉策略做力控)在清洗花瓶、插拔充电器等接触密集型任务上的可行性。

## 四、局限性

论文在结论中明确指出三点未来方向,亦即当前局限:

1. **仅处理 2D 图像输入**,尚未引入原生 3D 感知(点云、深度图),在杂乱或遮挡场景下的空间推理能力有进一步提升空间。
2. **VLA 扩展仅采用轻量动作头**,推理 token 与动作生成之间更紧密的耦合方式(而非当前的"冻结推理+外挂动作专家"范式)是值得探索的方向。
3. **PGC 闭环框架仅在桌面操作场景验证**,扩展到移动操作与导航、更长时间尺度与更丰富环境是重要的下一步。

此外,从方法设计看还存在几点未被论文充分讨论的局限:自动化数据构建流水线高度依赖外部现成模型(MoGe-2、Grounded-SAM-2、Detectron2、CoTracker3、RAM++)的输出质量,这些组件自身的误差会传导进最终 QA 标注,论文虽有多重质控但未对误差累积做定量分析;开放式文本奖励依赖外部奖励模型(Skywork-Reward-V2-Qwen3-4B)打分,存在被 RL 优化"攻击"奖励模型的潜在风险(reward hacking),论文未做专门的鲁棒性验证;PGC 的 FIFO Memory 是"最小可用"设计,论文自己也承认更复杂任务可能需要更精细的记忆策略。

## 五、评价与展望

**优点**:本文最突出的贡献是用单一 8B 模型真正统一了空间认知、规划纠错、指向定位三个此前被认为需要不同架构或不同规模模型分别处理的能力维度(对比 RoboBrain 通过 chain-of-thought 融合空间/时间推理、MIMO-Embodied 融合驾驶与操作、Pelican-VL 与 RynnBrain 追求统一具身推理但仍依赖多模型级联),并提出了一套面向异构多任务 GRPO 训练的具体工程方案(全局批次奖励归一化 vs. EMA-GRPO 的按任务移动平均)——这是对同期 heterogeneous multi-task RLHF/RLVR 训练不稳定问题的一个有价值的实证解法,而非仅停留在数据配比层面。PGC 框架把"单模型身兼多角色"这一设计推到了闭环执行的实测场景(10 步奶茶制作、主动人为扰动下的自适应重规划),提供了比大多数 EFM 工作止步于 Embodied QA 更进一步的验证深度。最后,VLA 侧的结果(LIBERO 上零动作预训练超过有预训练的 OpenVLA-OFT;RoboTwin2.0 零样本 65.0% 逼近 400-demo 微调后的 π0.5 的 72.9%)为"强具身推理骨干可大幅降低下游动作学习的数据需求"这一假设提供了目前较有说服力的证据,呼应了近期 π0.5、Gemini-Robotics-ER-1.5 等工作将具身推理数据共训练以缓解遗忘、但走得更彻底(直接验证了骨干强弱对下游动作学习收敛速度和数据效率的因果影响)。

**与其他公开工作的关系**:在评测口径上,论文自建 EmbodiedEvalKit 并声称所有基线都经统一流水线复现,一定程度上缓解了具身 VLM 领域评测碎片化、跨论文数字不可比的老问题,但这也意味着表中数字未必与各基线原论文报告的数字完全一致,读者引用时需留意口径差异。方法论上,其"分段线性衰减奖励+多任务归一化"的组合与近期一系列 RLVR 工作(GRPO、EMA-GRPO)一脉相承,创新点主要在于针对具身场景异构输出类型(点/框/轨迹/文本)的奖励族设计与全局批次归一化的工程选择,而非提出全新的 RL 算法框架。

**开放问题与可能的改进方向**:(1)论文尚未探讨在原生 3D 输入(点云/多视角)下重新训练是否能进一步放大当前 2D-only 设计在遮挡场景中暴露的短板;(2)PGC 中 Planner/Grounder/Corrector 三角色由同一模型异步串行调用,当任务并发度提升或多臂协同时,单模型串行推理的延迟是否会成为闭环响应速度的瓶颈,论文未做系统的延迟/吞吐分析;(3)RoboTwin2.0 零样本结果显示,在部分较难任务(如 Move Stapler 42%、Shake Bottle 89% vs 微调后 π0.5 的 42%/97%)上零样本指向执行与微调策略仍有明显差距,提示"纯指向+运动基元"的零样本范式更适合以抓取放置为主的任务,面向精细操作或强接触任务时仍需专门的模仿学习或力控策略(如论文自己引入的 ForceFlow 力觉切换)兜底;(4)难度感知过滤、动态过滤依赖 SFT checkpoint 的 rollout 通过率,这一"自举式"课程学习设计对不同规模/初始能力骨干的普适性、以及是否会引入选择性偏差,值得后续工作系统研究。
