## 一句话总结

作者形式化了动作分词应满足的三条准则——高压缩（P1）、完全可解码（P2）、因果有序（P3），指出 Bin/FAST/QueST 各自只满足其中的子集；OAT 用「寄存器 token（register token）+ 有限标量量化（FSQ）+ nested dropout + 寄存器间因果注意力」同时满足三者,训练出从粗到细排列的离散动作 token 空间,使自回归策略天然支持"前缀解码"的 anytime 推理。在 LIBERO/RoboMimic/MetaWorld/RoboCasa 四个仿真基准上,全量解码的 OAT₈ 全面超过 QueST 与 Diffusion Policy（如 LIBERO 56.3% vs QueST 48.2%、DP 36.6%；RoboMimic 73.1% vs 66.9%/67.1%）,真实机器人两项任务上 OAT₈ 达到 16/20、16/20,同样是四种方法中最优。

## 一、问题与动机

自回归序列模型已成为机器人策略学习的主流范式之一,但要让连续动作可被自回归建模,必须先把动作 chunk 离散化为 token 序列，这一步被称为**动作分词（action tokenization）**：映射 $\mathcal{T}: a_{1:H_a} \mapsto T_{1:H_l}$ 把形状为 $H_a \times D_a$（时间horizon×动作维度）的连续动作块变为长度 $H_l$ 的离散 token 序列,再由对应的反分词映射 $\mathcal{T}^{-1}: T_{1:H_l} \mapsto a_{1:H_a}$ 解码回连续动作供执行。

作者认为一个好的动作分词器应同时满足三条准则：

- **P1 高压缩**：token horizon $H_l$ 要足够小,以支持高效的长上下文自回归建模；
- **P2 完全可解码**：$\mathcal{T}^{-1}$ 必须是良定义的**全函数（total function）**——任意 token 序列都必须能解码出合法的动作块,不能出现未定义/无效输出（这在自回归推理时至关重要,因为策略可能生成任意 token 序列）；
- **P3 因果有序**：token 序列 $T_{1:H_l}$ 应具备与自回归 next-token prediction 归纳偏置对齐的从左到右因果结构——早期 token 编码动作块的粗粒度/全局结构,后续 token 逐步细化局部细节。

论文系统梳理了三类既有方案,发现无一能同时满足三条：

- **逐维度分箱（Bin，如 RT-1/RT-2/OpenVLA 所用）**：每一动作维度独立归一化后均匀分箱为 $N$ 个离散区间。满足 P2（映射良定义）,但 token 数随 $H_a \times D_a$ 线性增长（如 LIBERO 上 $H_a{=}32, D_a{=}7$ 需 224 个 token）,严重违反 P1；且 token 顺序只是"维度×时间"的序列化,不具备粗到细的信息层级,违反 P3。
- **频域变换（FAST，Pertsch et al. RSS 2025）**：先做离散余弦变换（DCT）再用字节对编码（BPE）压缩。低频分量在前、高频在后,部分满足 P3 的直觉,信息密度高,满足 P1；但 BPE 产生变长 token 序列,导致反分词是**偏函数**——任意策略生成的 token 计数不保证等于目标系数总数 $N_{\text{target}} = H_a \times D_a$，无法重塑（reshape）为合法动作块,产生"不可解码"错误,违反 P2。附录进一步分析了这一"结构性错配"：若强行截断/填充生成的系数序列,由于 DCT 系数是有序展平的,插入/丢失个别系数会导致后续系数整体错位（"spectral shift"）,语义上把第 $J$ 关节的系数错移到第 $J{+}1$ 关节,可能造成物理上危险的动作。
- **学习式隐空间 tokenizer（QueST，Mete et al. NeurIPS 2024）**：用时序卷积+因果 Transformer 编码器将动作块压缩为固定长度隐 token 序列（如 $H_l = H_a/4$）,再做向量量化。编码器/解码器都是良定义的神经网络,满足 P1、P2；但隐 token 空间没有显式排序或层级结构,不利于自回归建模,违反 P3。

## 二、核心方法

OAT 是一个学习式自编码器框架，用 Transformer 编码器 + 寄存器 token（register token）+ FSQ + nested dropout 构造有序离散动作表示。

**编码与量化。** 把连续动作块 $a_{1:H_a}$ 与一组固定数量的**可学习寄存器 token** $r_{1:H_l}$ 拼接,送入 Transformer 编码器联合处理；编码结束后只保留寄存器对应的隐向量 $z_{1:H_l}$ 作为瓶颈表示（动作 token 自身的编码结果被丢弃）。寄存器隐向量随后经**有限标量量化（FSQ）** 离散化，得到 token 序列 $T_{1:H_l}$。

**解码。** 反分词 $\mathcal{T}^{-1}$ 由单遍（single-pass）Transformer 解码器实现（结构类似 ACT/ALOHA 的解码器 [Zhao et al.]）：解码器以固定的正弦位置编码为 query，交叉注意力读取离散动作 token，把嵌入投影回连续动作空间，重构出动作块 $\hat a_{1:H_a}$。分词器与反分词器端到端联合训练，损失为重构 MSE：

$$\mathcal{L} = \lVert \hat a_{1:H_a} - a_{1:H_a} \rVert_2^2.$$

用大白话说：把整段动作先"摘要"进几个可学习的寄存器槽位，再把这些槽位量化成离散符号；解码器再把这些符号"展开"还原回动作曲线，训练目标就是让还原结果尽量贴近原始动作。

**两种诱导排序的机制（P3 的关键）：**

1) **Nested Dropout（嵌套丢弃）**：训练时对寄存器 token 施加尾部随机丢弃——每步从分布 $p(\cdot)$ 采样保留数 $K \in [H_l]$，把尾部 $H_l - K$ 个寄存器替换为可学习的 `MASK` token 再送入解码器。这迫使编码器把信息按"优先级"打包进寄存器：靠前的寄存器必须独立承担起在任意前缀长度下都可用的粗粒度重构任务，后面的寄存器只需负责在信息预算增大时补充细节。
2) **因果注意力（Causal Attention）**：寄存器可以自由地注意所有动作 token（获取全局聚合信息），但寄存器与寄存器之间的注意力被因果掩码限制——第 $i$ 个寄存器只能注意到第 $j \le i$ 个寄存器，强制寄存器间形成从左到右的信息流，与自回归 next-token 预测的结构对齐。

**信息论直觉。** 作者借用 Shannon 编码理论解释这种排序为何自然涌现：最优码长与事件概率的负对数成正比,常见/粗粒度的模式应编码得短。动作块数据分布高度非均匀——大多数轨迹共享共同的粗粒度模式,精细偏差则相对稀有；在 nested dropout 下,分词器被迫在不同信息预算下都尽量重构准确,因而学会把信息按频率/重要性递减的顺序分配到 token 序列中，这一排序不是人为规定的，而是从"变长前缀下的重构目标"中自然涌现的。

**自回归 OAT 策略与前缀解码（P3 带来的核心能力）。** 给定观测历史 $o_{1:H_o}$，策略对 token 序列做标准自回归因式分解：

$$p(T_{1:H_l} \mid o_{1:H_o}) = \prod_{i=1}^{H_l} p(T_i \mid T_{<i}, o_{1:H_o}).$$

用大白话说：策略像"说话"一样一个 token 一个 token 地生成动作 token，先说出动作的大致轮廓，再逐步补充细节。因为解码器对任意前缀 $T_{1:K}$（$K \le H_l$）都能解码出一个合法动作块（尾部补 `MASK`），策略可以在生成到第 $K$ 步时提前终止，用 $\text{OAT}_K$ 表示只解码前 $K$ 个 token 得到的动作。这带来 Bin/FAST/QueST 都不具备的**anytime 推理** 能力：短前缀换低延迟、粗动作；长前缀换高延迟、细动作，且这一权衡不需要改变策略架构或训练目标。

## 三、实验结果

**实验设置。** 动作块 horizon $H_a{=}32$，观测历史 $H_o{=}2$，执行时用 receding-horizon 方式只执行每次预测的前 $\tfrac12 H_a{=}16$ 步。OAT 与最强基线 QueST 使用相同超参：2 层 Transformer 编码器（模型维度 256，head 维度 64）、4 层 Transformer 解码器，latent horizon $H_l{=}8$，latent 维度 $D_l{=}4$，FSQ levels $[8,5,5,5]$（隐式 codebook 大小 $\lvert\mathcal V\rvert \approx 1000$）。Bin 用 $N{=}256$ 分箱，FAST 用 $\lvert\mathcal V\rvert{=}1024$ 词表。基线还包括 Diffusion Policy（DP，10-step DDIM，相同 Transformer backbone）。

四个仿真基准：LIBERO（libero10，$D_a{=}7$，50 demo/task）、RoboMimic（lift/square/can，$D_a{=}7$，200 demo/task）、MetaWorld（box close/coffee pull/disassemble/stick pull，$D_a{=}4$，50 demo/task）、RoboCasa（close drawer/coffee press button/turn off microwave/turn off sink faucet，$D_a{=}12$，50 人类+150 机器生成 demo/task）。均为 5 个随机种子 × 每种子 50 次评估 rollout。

**表 1：仿真基准平均成功率（%）**

| Policy | LIBERO | RoboMimic | MetaWorld | RoboCasa |
|---|---|---|---|---|
| DP | 36.6 ± 0.2 | 67.1 ± 1.3 | 19.3 ± 1.6 | 54.0 ± 1.6 |
| Bin | 14.4 ± 0.6 | 39.5 ± 1.2 | 14.5 ± 0.7 | 27.7 ± 0.9 |
| FAST | 23.0 ± 0.5 | 24.0 ± 1.5 | 7.1 ± 0.9 | 13.2 ± 1.1 |
| QueST | 48.2 ± 0.6 | 66.9 ± 0.8 | 17.9 ± 0.9 | 52.3 ± 1.9 |
| OAT₁ | 11.7 ± 0.7 | 50.8 ± 1.4 | 11.3 ± 0.4 | 47.7 ± 1.3 |
| OAT₂ | 39.8 ± 0.9 | 52.5 ± 1.2 | 16.4 ± 0.3 | 50.3 ± 0.8 |
| OAT₄ | 46.4 ± 0.6 | 65.3 ± 0.9 | 19.5 ± 0.8 | 51.7 ± 1.0 |
| **OAT₈** | **56.3 ± 1.0** | **73.1 ± 0.5** | **24.8 ± 0.3** | **54.6 ± 1.1** |

OAT 随解码 token 数 $K$ 增大呈现清晰的单调性能提升，$\text{OAT}_8$ 在全部四个基准都取得最佳成绩。Bin 因 token 序列过长表现最差；FAST 虽压缩率尚可，但反分词偏函数导致的不可解码/不稳定行为使其性能同样很差；QueST 是最强基线，但缺乏排序结构使其在多数场景不及 $\text{OAT}_4$/$\text{OAT}_8$。

**表 2：token 数与推理延迟（LIBERO 上，ms/次推理，NVIDIA A100）**

| Policy | #Tok | 延迟(ms) |
|---|---|---|
| Bin | 224 | 517.2 |
| FAST | 44.2（均值,变长） | 114.4 |
| QueST | 8 | 27.1 |
| OAT₁ | 1 | 10.5 |
| OAT₄ | 4 | 17.4 |
| OAT₈ | 8 | 27.4 |

$\text{OAT}_8$ 与 QueST token 数、延迟相当（全量解码计算量一致），但 $\text{OAT}_1$ 仅用 1 个 token、10.5ms 就能给出可执行动作（RoboCasa 上 $\text{OAT}_1$ 已达 47.7%，超过 Bin/FAST 全量解码），体现了 anytime 权衡的实际价值。

**消融：排序是否重要？** 去掉 nested dropout（记为 $\text{OAT}_\times$，其余架构/训练设置不变,token 预算仍为 8）后性能全面下降：LIBERO 从 56.3 降到 35.2，RoboMimic 从 73.1 降到 61.1，MetaWorld 从 24.8 降到 17.6，RoboCasa 从 54.6 降到 48.5——在多个基准上甚至低于 $\text{OAT}_4$，部分情况低于 QueST，说明排序结构本身（而不仅是压缩率）对自回归学习有实质性增益。

**消融：codebook 大小的影响（LIBERO）。** 用不同 FSQ level 配置控制 $\lvert\mathcal V\rvert$：$[8,6,5]\to240$ 得 29.2%，$[8,8,8]\to512$ 得 53.3%，$[8,5,5,5]\to1000$（默认）得 56.3%，$[8,8,6,5]\to1920$ 得 54.6%，$[7,5,5,5,5]\to4375$ 得 46.9%——性能随容量从很小提升到中等而显著改善，但过度扩大 codebook 会增加 token 熵与稀疏性，反而损害自回归 modelability，导致性能下降。

**真实机器人实验。** 固定基座 ARX-5 机械臂 + 单个 Logitech 摄像头，7 维动作空间（末端位姿+夹爪），每任务 200 条人类遥操作演示，每任务 20 次独立评估。

| Policy | Pick & Place Ball | Stack Cups |
|---|---|---|
| DP | 14/20 | 11/20 |
| Bin | 4/20 | 8/20 |
| FAST | 8/20 | 6/20 |
| QueST | 11/20 | 8/20 |
| OAT₁ | 7/20 | 3/20 |
| OAT₄ | 13/20 | 12/20 |
| **OAT₈** | **16/20** | **16/20** |

真实世界结果与仿真趋势一致：$\text{OAT}_8$ 两项任务均最优；Bin 因长 token 序列导致的高延迟拖累闭环响应；FAST 解码不稳定、动作过于激进；QueST 优于二者但仍受限于无序隐空间。作者观察到 OAT 解码 token 越多，动作越平滑，解码 token 数较少（如 $\text{OAT}_1$、$\text{OAT}_2$）时的典型失败模式是执行精度不足（如接近目标但未能完成精细插入动作），印证了"早期 token 编码粗粒度结构、后期 token 编码精细修正"的设计意图。

## 四、局限性

1. **推理深度在部署时是固定的**：论文承认这是从信息论视角看的次优设计——一个动作块所需的 token 数本应取决于其内在复杂度和所需精度：简单行为应能用更紧凑的表示，复杂、接触丰富的交互可能需要更深的自回归精化。如何在线估计动作复杂度、决定何时追加 token 才有意义仍是未解决的问题，作者将"自适应自回归深度"列为未来工作方向。
2. **单 token（$\text{OAT}_1$）在部分任务上明显不足**：例如 LIBERO 上 $\text{OAT}_1$ 仅 11.7%，远低于 QueST 的 48.2%，说明极端压缩下信息损失过大，需要至少若干个 token 才能与强基线竞争，"1 token 即可用"更多是理论上的可行性而非普遍实用性。
3. **codebook 大小需要手动调参**：FSQ level 配置对性能敏感（过小欠拟合、过大又损害 modelability），论文未提供自动/自适应选择 codebook 容量的方法。
4. **真实机器人验证规模有限**：仅在单一 ARX-5 机械臂上验证了两个相对简单的桌面任务（拾放球、摞杯子），未覆盖双臂、长时序或更复杂接触丰富的真实任务，也未在大规模跨本体预训练场景（如 FAST 最初面向的大规模 VLA 预训练设定）下验证。
5. **全量解码时计算量并不比最强基线更省**：$\text{OAT}_8$ 与 QueST 的 token 数、延迟基本相当，OAT 的核心优势是灵活性（anytime 权衡）与排序带来的性能增益，而非渐近意义上的推理成本降低。

## 五、评价与展望

OAT 的主要贡献是概念性的：把动作分词的设计空间形式化为三条相互制约的准则（压缩、可解码性、排序），并指出现有方法（Bin/FAST/QueST）各自只能满足其子集——这一框架本身具有超越具体架构的参考价值，为后续动作 tokenizer 设计提供了可复用的评价标准。方法层面，nested dropout 诱导粗到细排序的思路借鉴自 Matryoshka 表征学习（Kusupati et al., NeurIPS 2022）与"用 nested dropout 学习有序表征"（Rippel et al., ICML 2014）等公开工作，其新意在于把这一机制与 register token 聚合、FSQ 离散化、寄存器间因果注意力结合，专门针对自回归机器人策略的 next-token 预测结构做了定制设计，并在系统性消融（表 III/IV）中证明了排序机制（而非单纯压缩率）对下游策略性能有独立贡献。

与 FAST 相比，OAT 从架构上根治了"变长 BPE 序列 vs 固定维度目标"的结构性错配问题（论文附录 B 对该错配给出了较细致的分析），代价是需要额外训练一个自编码器，而不是像 FAST 那样直接对预计算的频域系数做无监督 BPE；与 QueST 相比，OAT 复用了几乎相同的编码器/解码器骨架和 codebook 规模，核心差异仅在训练目标中加入 nested dropout 和寄存器间因果掩码，这使得比较相对公允，也让"排序是否重要"这一消融结论较为可信。论文提到的 BEHAVIOR-1K Challenge 2025 冠军方案将 FAST 与连续流匹配 action expert 结合使用，反映了当前 VLA 系统中"离散推理 + 连续解码"混合范式的实际部署趋势；OAT 在这一背景下被定位为一种可以替代 FAST/QueST、同时与扩散/流式连续解码器兼容的辅助监督信号或规划接口。

开放问题包括：(i) 前缀长度的自适应选择——如何在推理时用某种不确定性/复杂度估计来决定该解码多少个 token，而不是像本文那样固定 $K$；(ii) OAT 在更大规模、跨本体的预训练语料（类似 Open X-Embodiment 规模）下是否仍能保持相对于 QueST/FAST 的优势，目前的评测规模（四个中小型仿真基准 + 单臂两个真实任务）尚不足以验证这一点；(iii) OAT 的 register-token 因果注意力设计是否可以进一步与视觉-语言主干的语义结构对齐，从而让 token 排序不仅反映"动作复杂度"，还反映跨模态的语言指令层级，这可能是把该方法推广到更复杂长时序任务的一个自然方向。
