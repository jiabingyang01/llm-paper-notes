## 一句话总结

VGA 用预训练的 3D 几何基础模型 VGGT（Visual Geometry Grounded Transformer）直接替换 VLA 中常见的语言/视频骨干，把机器人操作形式化为视觉到几何的映射 $f(v) \rightarrow G$，配合 Progressive Volumetric Modulation（PVM）模块和"动作 + 3D 属性"联合训练、推理时解耦解码的策略，在 LIBERO 上取得 98.1% 平均成功率，超过 $\pi_{0.5}$（96.9%）、OpenVLA-OFT（97.1%）等主流 VLA 基线以及若干 3D-VLA、World Action Model 基线；在真实 Franka Panda 平台上，训练视角内成功率与 $\pi_{0.5}$ 基本持平（75% vs 77%），但在完全未见的相机视角下反超 $\pi_{0.5}$ 6 个百分点（58% vs 52%）。

## 一、问题与动机

论文的核心论点：抓取、放置、对齐等操作动作本质上由 3D 几何量（位置、旋转、空间关系）决定，但当前主流的 VLA（如 RT-2、Octo、$\pi_0$ 系列）和视频预测式 World Action Model（WAM）所用的骨干都是在互联网规模的 2D 图文或时序像素数据上预训练的，其表示被语义概念或 2D 先验塑造，与操作所需的精确 3D 几何本质并不天然对齐。

作者认为现有的"补救"方案都有结构性缺陷：

- **显式 3D 输入**（深度图、点云）：需要额外传感器，带来噪声、多模态融合复杂度和硬件成本。
- **给 VLM 骨干前置 3D-aware 编码器**（如 SpatialVLA 的 3D-aware encoder、GeoAwareVLA 的 frozen VGGT 编码器）：论文引用已有研究指出 VLM 的隐表示本质上仍是 2D-centric 的，3D 信息最终仍被投影回扁平的 2D 隐空间，形成一个"flawed 3D-2D-3D transformation loop"——3D 几何特征被迫经过 2D 隐空间瓶颈，再解码回 3D 动作，核心推理环节仍被下游 VLM 瓶颈化。

由此提出：与其在 2D-centric 骨干外面"打补丁"式加 3D 信息，不如直接用一个原生 3D 几何基础模型（VGGT）作骨干，让感知、推理、动作三者始终在同一个原生 3D 表示空间内对齐。

## 二、核心方法

### 2.1 统一 Tokenization 与 VGGT 骨干

给定 $N$ 个视角的多视图 RGB 观测 $\{I_t^i\}_{i=1}^N$、语言指令 $\ell$、机器人本体状态 $S_t$：每帧图像经 DINO 编码器得到 $K$ 个 patch token；本体状态经 MLP 投影为 embedding；语言指令经 Qwen-GTE 编码；此外引入可学习的 action query（聚合操作上下文）和 camera query（聚合相机上下文）。所有模态 embedding 拼接为统一 token 序列后送入预训练的 VGGT transformer（LoRA 微调，rank=64），其 Alternating-Attention 机制交替执行：

$$X_k^{(l)} = \text{Attention}(Q=X_k^{(l-1)}, K=X_k^{(l-1)}, V=X_k^{(l-1)}) \quad (\text{偶数层，帧内局部注意力})$$

大白话：偶数层只在同一视角/同一模态的 token 内部做注意力，先理清局部细节。

$$\tilde{X}^{(l)} = \text{Attention}(Q=\tilde{X}^{(l-1)}, K=\tilde{X}^{(l-1)}, V=\tilde{X}^{(l-1)}) \quad (\text{奇数层，跨模态全局注意力})$$

大白话：奇数层让所有视角、语言、动作 token 互相看一遍，融合出统一的 3D 场景理解。最终输出 $\mathbf{V}_t = \tilde{X}^{(L)}$ 作为"原生 3D 表示"，送入 PVM 模块与三个任务头。

### 2.2 三个解码头与联合训练目标

- **动作头**：回归式 transformer（$L_a=12$ 层，沿用 OpenVLA-OFT 设计），采用 chunk size $C=8$ 的动作分块；以可学习噪声 embedding $z \in \mathbb{R}^{C \times D}$ 为输入，经 PVM 调制后输出动作块 $\hat{a}_{t:t+C} = \phi(z^{(L_a)})$。
- **相机头**：取演化后的相机 token 回归相机参数 $g_i = [r_i, t_i, f_i] = \text{MLP}(t_{i,g}^{(L)})$（旋转、平移、焦距）。
- **深度头**：DPT（Dense Prediction Transformer）结构，聚合多尺度骨干 token 重建逐像素深度 $D_i(u) = \Psi\left(\sum_s \phi_s(t_i^{(s)})\right)$。

总损失 $\mathcal{L} = \mathcal{L}_{action} + \mathcal{L}_{camera} + \mathcal{L}_{depth}$（动作为回归损失，相机为 Huber loss，深度为带不确定性加权和梯度项的深度损失）。大白话：相机头和深度头不是为了推理时用，而是训练时给骨干"加作业"——逼它把相机姿态、场景深度学扎实，使共享的 3D 表示更"实在"。推理阶段相机头和深度头被旁路（**解耦推理**），只保留动作头解码，在不增加显式几何重建计算开销的前提下保留联合训练带来的空间感知能力。

### 2.3 Progressive Volumetric Modulation（PVM）

动机：标准交叉注意力（解码器隐状态直接 attend 到 3D 表示）不足以充分利用 3D 表示中的几何结构。PVM 在每层 $l$ 接收视觉-语言条件 $H_a^{(l)}$、动作 query 条件 $H_b^{(l)}$、解码器隐状态 $h_{dec}^{(l)}$，执行两阶段序列化跨模态转导：

$$\tilde{h}_{dec}^{(l)} = \text{Attention}(Q=h_{dec}^{(l)}, K=H_b^{(l)}, V=H_b^{(l)})$$

大白话：第一步让解码器状态先去动作 query 里"提取"与当前动作最相关的上下文。

$$a_{dec}^{(l)} = \text{Attention}(Q=\tilde{h}_{dec}^{(l)}, K=H_a^{(l)}, V=H_a^{(l)})$$

大白话：第二步再用这个中间结果去视觉-语言（即 3D 几何）条件上做二次精炼，先粗筛再细调。最后拼接投影回隐空间 $h_{dec}^{(l+1)} = \text{Linear}([h_{dec}'^{(l+1)}, a_{dec}^{(l)}])$，逐层迭代使几何信息渐进注入动作生成过程。

### 2.4 实现细节

骨干与动作头层数 $L=L_a=12$；LoRA rank=64 做参数高效微调；可训练参数约 500M；单卡 NVIDIA A100-SXM4-80GB 训练，最长训练任务耗时 60 GPU 小时；VGGT 预训练数据包括 Co3Dv2、BlendMVS 等大规模多模态 3D 数据。

## 三、实验结果

**LIBERO 仿真基准**（Spatial/Object/Goal/Long 四套件，各 10 任务，每套件 500 次 rollout）：

| 类别 | 方法 | Spatial | Object | Goal | Long | Avg |
|---|---|---|---|---|---|---|
| VLA | OpenVLA | 84.7% | 88.4% | 79.2% | 53.7% | 76.5% |
| VLA | $\pi_0$ | 90.0% | 86.0% | 95.0% | 73.0% | 86.0% |
| VLA | $\pi_{0.5}$ | 98.8% | 98.2% | 98.0% | 92.4% | 96.9% |
| VLA | OpenVLA-OFT | 97.6% | 98.4% | 97.9% | 94.5% | 97.1% |
| VLA | VLA-Thinker | 98.7% | 99.0% | 95.2% | 96.9% | 97.5% |
| 3D-VLA | SpatialVLA | 88.2% | 89.9% | 78.6% | 55.5% | 78.1% |
| 3D-VLA | GeoAwareVLA | 95.0% | 100% | 99.0% | 93.0% | 96.8% |
| 3D-VLA | GeoVLA | 98.4% | 99.0% | 96.6% | 96.6% | 97.7% |
| WAM | mimic-video | 94.2% | 96.8% | 90.6% | -/- | 93.9% |
| WAM | Motus | 96.8% | 99.8% | 96.6% | 97.6% | 97.7% |
| 本文 | **VGA** | **99.0%** | **99.6%** | **98.6%** | 95.0% | **98.1%** |

VGA 比 $\pi_{0.5}$ 高 1.2%，比 OpenVLA-OFT 高 1.0%（绝对成功率）；比 SpatialVLA（3D-aware encoder）高 20.0%，比 GeoAwareVLA（frozen VGGT 编码器）高 1.3%，比 GeoVLA（显式 3D 信息）高 0.4%。但在 LIBERO-Long 上 VGA（95.0%）落后于 Motus（97.6%）、VLA-Thinker（96.9%）、GeoVLA（96.6%），作者归因于 VGGT 预训练聚焦几何结构、缺乏长序列数据暴露，长时序依赖建模弱于 VLM/WAM 骨干。

**消融实验**（Table 2）：去掉 PVM 掉 2.4 点（98.1%→95.7%）；去掉联合训练（仅 action 监督）掉 0.9 点（98.1%→97.2%）；随机初始化 + 全参数微调仅 86.6%，预训练初始化 + 全参数微调也只有 87.1%（与随机初始化几乎持平），说明全参数微调几乎抹除了预训练 VGGT 带来的 3D 先验优势；而随机初始化 + LoRA 直接崩溃到 6.4%，说明 LoRA 严重依赖一个良好的预训练起点、无法从零学习。最终配置（预训练初始化 + LoRA + PVM + 联合训练）达到 98.1%，是各因素叠加的结果。

**真实世界实验**：平台为 7 自由度 Franka Panda + 平行夹爪（8 维配置/动作空间），三个 RealSense D415 相机（1 个手腕相机 + 2 个固定相机，分别用于同分布 ID 与跨视角 OOD 评估），GELLO 遥操作采集，每任务 80–100 条轨迹；对比基线 ACT、OpenVLA、$\pi_{0.5}$：

| 方法 | ID 平均 | OOD（未见视角）平均 |
|---|---|---|
| ACT | 37% | 7% |
| OpenVLA | 18% | 3% |
| $\pi_{0.5}$ | 77% | 52% |
| VGA（本文） | 75% | **58%** |

ID 评估下 VGA（75%）略逊于 $\pi_{0.5}$（77%），主要差在 Pick Cube 任务（80% vs 85%），作者归因于自采数据存在噪声伪影而 $\pi_{0.5}$ 的大规模预训练能过滤这类不一致性。OOD 评估（测试时把固定相机切换到训练中完全未见的视角）下 VGA 反超 $\pi_{0.5}$ 6 个百分点，作者将其归因于原生 3D 表示不依赖视角相关的 2D 视觉模式。此外还做了语言接地抓取实验（黄瓜/胡萝卜/茄子三种视觉相似蔬菜、不同空间排布、60 条演示），VGA 能稳定按语言指令抓取正确目标。

## 四、评价与展望

**优点**：论文提出了一个简洁但有冲击力的框架性论点——把操作建模为 $f(v) \rightarrow G$ 而非传统"2D 感知 + 语言推理 + 动作"管线，用 3D 几何基础模型 VGGT 整体替换骨干，是对"3D-2D-3D 变换环路"缺陷（即便是 3D-aware 的 VLA 如 GeoAwareVLA/SpatialVLA，骨干仍是 2D-centric VLM）的一次针对性回应。消融实验拆解充分：LoRA vs 全参数、随机 vs 预训练初始化、联合训练 vs 纯动作监督、PVM 有无——四组对照清晰验证了"预训练 VGGT 权重"与"LoRA 轻量适配"缺一不可这一核心结论。真实世界的跨视角 OOD 实验设计干净（同一批演示、仅切换测试相机），6 个百分点的优势是本文较有说服力的实证支撑。

**局限与开放问题**（论文附录 G 亦明确指出一条）：作者自陈 VGA 在需要常识知识或语言推理的任务上存在局限，因为骨干不基于大语言模型——例如 OpenVLA 论文中"把方块放在 Taylor Swift 照片上"这类需要常识关联的指令对 VGA 是困难的，作者提出未来可引入外部推理模块。此外：(1) 与最强 3D-VLA 基线（GeoAwareVLA/GeoVLA）的优势幅度其实很窄（仅 0.4%~1.3% 绝对成功率），"整体替换骨干"相对"给 VLM 加 3D-aware 模块"的增益在 LIBERO 上并不悬殊，真正拉开差距的是相对纯 2D VLA（如 SpatialVLA）以及真实世界跨视角泛化；(2) LIBERO-Long 的短板提示"原生 3D 几何先验"和"长时序规划能力"可能是两种不完全兼容的归纳偏置，二者是否需要不同的预训练数据混合、抑或可由统一骨干同时具备，仍待探索；(3) 真机训练阶段由于 3D 标注成本高只用了动作监督，相机头/深度头的联合训练收益是从仿真消融外推而来，未在真机上直接验证，这是论证链条上相对薄弱的一环；(4) VGA 在 LIBERO 上对 WAM 基线（Motus 97.7%、mimic-video 93.9%）的优势也不大（98.1% vs 97.7%），提示"3D 几何"与"视频/时序动态"作为两种不同的世界模型监督信号，未来融合二者而非二选一，可能是更有潜力的方向。
