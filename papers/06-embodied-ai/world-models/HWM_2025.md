## 一句话总结

在 1X EVE 人形机器人的 100 小时第一视角数据上，作者训练了两类轻量级、开源的动作条件视频世界模型（掩码 Transformer 的 Masked-HWM 与 Flow-Matching 的 Flow-HWM），系统消融了注意力结构与参数共享策略，结论是掩码范式全面优于流匹配（FID 10.13 vs 110.73），且参数共享能在几乎不损质量的前提下把模型缩小 33–53%，使世界模型可在 1–2 张 GPU 上训练与部署。

## 一、问题与动机

人形机器人天然适配为人类设计的环境，且遥操作演示采集相对容易，但要让其在开放、非结构化的真实世界中安全可靠地行动，需要能"预演动作后果"的能力。**世界模型**（World Model）通过预测"给定动作后的未来观测"来提供这种能力：既可作为长时程规划的动力学模型，也可为策略学习生成合成经验。

作者指出当前存在一条明确的鸿沟：

- 主流视频生成模型（Sora、MovieGen 等）面向**娱乐**，重视觉观感而缺乏第一视角、物理可信的预测，且不支持以过去视频为条件。
- 多数机器人/通用世界模型（UniSim、Cosmos、Genie 等）要么**闭源**、要么**算力门槛极高**、要么并非为人形机器人设计。文中给出具体例证：Cosmos 最小的 7B 变体训练需 8 张 H100、推理需 40GB+ 显存；在作者的 2×A6000 上生成 121 帧视频要一个多小时，微调/部署不现实。

由此提出核心问题：**能否构建一个物理可信、面向人形、且轻量到只需两张 GPU 就能训练与部署的世界模型？** 答案即 Humanoid World Models（HWM）——一族开源轻量模型，以人形动作 token 为条件预测未来第一视角视频。

## 二、核心方法

### 2.1 统一问题形式化

给定 $p$ 帧过去 RGB 视频 $v_p\in\mathbb{R}^{p\times 3\times H\times W}$、$p$ 段过去动作 $a_p\in\mathbb{R}^{z}$ 与 $f$ 段未来动作 $a_f\in\mathbb{R}^{z}$，模型预测 $f$ 帧未来视频 $v_f\in\mathbb{R}^{f\times 3\times H\times W}$。动作向量包含关节角、速度与夹爪（手部闭合）状态。视频先经 VAE 压到隐空间 $L_p,L_f$：Masked-HWM 用离散 VQ-VAE，Flow-HWM 用连续 VAE。

作者在同一形式化下实现两条生成范式，从而公平地比较"掩码 vs 流匹配"。

### 2.2 Masked-HWM：掩码视频建模（离散隐空间）

沿用 MaskGIT/MAGVIT 的掩码视频建模思路。把 $v_p,v_f$ 过 VQ-VAE 得到 $\mathbf{L_p},\mathbf{L_f}$，沿时间维拼成 $\mathbf{L}=[\mathbf{L_p};\mathbf{L_f}]$。训练时输入被**破坏 + 掩码**的隐序列：

- 破坏（借鉴 Copilot-4D）：以从 $\mathcal{U}(0,\rho_{\max})$ 采样的比率随机替换 token，$\rho_{\max}=0.2$；
- 掩码（逐帧阈值）：每帧采 $r\sim\mathcal{U}(0,1)$，用调度函数算阈值 $\gamma(r)$，对每个 token 采一概率，低于该帧阈值则掩掉。

训练目标是仅在被掩位置 $\mathbf{M}$ 上的交叉熵：

$$\mathcal{L} = -\mathbb{E}_{\mathbf{M}}\left[\sum_i \mathbf{M}_i \log p(\hat{L}_i \mid \mathbf{L_f})\right]$$

用大白话说：把未来帧的一部分 token 盖住（并对已知 token 加噪扰动增强鲁棒性），让模型根据"没盖住的上下文 + 过去视频 + 动作"猜出被盖住的 token，只在盖住处算 loss。

推理时先把未来隐序列 $\mathbf{L_f}$ 全部掩掉，**逐个隐帧**生成：预测完一帧的 token 后回填作为下一帧的条件；帧内则并行地在 $K$ 步细化中反复"随机重掩—重预测"，逐步改进猜测（实验 $K=2$）。这种并行解码比自回归逐 token 生成快得多。

### 2.3 Flow-HWM：流匹配（连续隐空间）

在连续隐空间学一条把高斯先验搬运到数据分布的**时变速度场**。设视频样本 $\mathbf{X}_1$、噪声 $\mathbf{X}_0\sim\mathcal{N}(0,\mathbf{I})$，采 $t\in[0,1]$ 做线性插值：

$$\mathbf{X}_t = t\mathbf{X}_1 + (1 - (1 - \sigma_{\min})t)\mathbf{X}_0$$

其真值速度为路径对时间的导数：

$$\mathbf{V}_t = \frac{d\mathbf{X}_t}{dt} = \mathbf{X}_1 - (1 - \sigma_{\min})\mathbf{X}_0$$

模型 $u_\theta$ 以 $v_p,a_p,a_f$ 与时间 $t$ 为条件预测瞬时速度，用 MSE 回归真值速度：

$$\mathbb{E}_{t, \mathbf{X}_0, \mathbf{X}_1, a_p, a_f, v_p}\left[\|u_\theta(\mathbf{X}_t, a_p, a_f, v_p, t) - \mathbf{V}_t\|^2\right]$$

用大白话说：在噪声与真视频之间画一条直线，训练网络在这条线上任意一点都能指出"往真视频方向该怎么走"这个箭头（速度）；推理时从纯噪声出发，用一阶 Euler ODE 从 $t{=}0$ 积分到 $t{=}1$（50 步去噪），配合无分类器引导（CFG，尺度 3.0）平衡条件强度。

### 2.4 三维设计空间与四条 token 流

两种范式都把输入拆成**四条 token 流**：过去视频 $v_p$、未来视频 $v_f$、过去动作 $a_p$、未来动作 $a_f$。作者沿三个维度做架构消融：

**(1) 注意力结构。**

- *Base Block*：Masked-HWM 采用因子化时空注意力（借鉴 Genie、Pandora）——视频流各自做 Spatial Attention，四条流通过**联合 Temporal Attention** 交互（空间用 2D RoPE、时间用 1D RoPE）；Flow-HWM 的 Base Block 借鉴 Stable Diffusion 3，每条流先经 timestep 调制（scale/shift $\alpha,\beta$），再用各自的 $W_{QKV}$ 投影后做**联合注意力**，视频 token 用 3D RoPE、动作 token 用 1D RoPE。
- *Split Attention*：两阶段——每条流先在自身序列内做独立 self-attention，再由未来视频 $v_f$ 作 query、对 $\{v_p,a_p,a_f\}$ 做 cross-attention。避免全局联合注意力在长序列/高 token 数下的高昂开销。

**(2) 参数共享**（共享 timestep 调制 $\alpha,\beta,\gamma$、$W_{QKV}$、前馈 MLP）。

- *Full Sharing*：四条流全部共享，最大化压缩；
- *Modality Sharing*：同模态内共享（$v_p,v_f$ 共享一套、$a_p,a_f$ 共享一套），在紧凑与表达力之间折中。

两者都保留前 4 层为完全独立的 Base Block（学模态特定表征），仅在其后 $l-4$ 层做共享（深层做跨模态推理，借鉴 fal.ai 实践）。

### 2.5 数据与规模

- **数据**：1xGPT 数据集（1X Technologies），1X **EVE** 人形机器人执行多种任务的 100 小时第一视角视频，30Hz，每帧配 $a\in\mathbb{R}^{25}$ 动作向量（运动速度、手部闭合、以及腕/膝/肘/肩/颈/髋等关节的 pitch-yaw-roll）。任务设定为以 $p{=}9$ 过去帧预测 $f{=}8$ 未来帧，分辨率 $256\times256$。
- **Masked-HWM**：Cosmos DV8x8x8 离散 tokenizer（时空各 8× 压缩，$256^2$→$32\times32$），24 层/8 头/512 维/MLP 2048，AdamW，LR 从 3e-5 线性衰减，6 万步，单张 A6000。
- **Flow-HWM**：Cosmos 连续 8x16x16 tokenizer（空间 16×、时间 8× 压缩，$256^2$→$16\times16$），17 层/1172 维，patch $p_{lw}{=}2,p_t{=}1$，AdamW LR 1e-4，15 万步，2 张 A6000。

## 三、实验结果

评测用 FID 与 PSNR，在留出验证集上生成的 21,000 帧上计算；为剥离 VAE 重建误差，真值帧也过同一 VAE 编解码。同时报告参数量、推理显存、每秒样本数（解码速度）。

**表 1：Masked-HWM 四变体**（Base Block 取得最佳 FID，Split Attention 取得最高 PSNR，Full Sharing 最省最快）

| 指标 | Split Attention | Base Block | Modality Sharing | Full Sharing |
|---|---|---|---|---|
| 参数量（Billion） | 0.220 | 0.321 | 0.237 | **0.195** |
| 峰值显存（GB） | 2.22 | 2.63 | 2.30 | **2.12** |
| 每秒样本数 | 2.09 | 2.27 | 2.25 | **2.36** |
| FID ↓ | 15.31 | **10.13** | 11.67 | 14.21 |
| PSNR（dB）↑ | **29.37** | 29.02 | 28.97 | 28.66 |

**表 2：Flow-HWM 四变体**（Full Sharing 在 FID/显存/速度上全面最优，Split Attention 取得最高 PSNR）

| 指标 | Split Attention | Base Block | Modality Sharing | Full Sharing |
|---|---|---|---|---|
| 参数量（Billion） | 0.944 | 1.36 | 0.886 | **0.648** |
| 峰值显存（GB） | 4.37 | 5.94 | 4.41 | **3.25** |
| 每秒样本数 | 1.11 | 1.69 | 1.89 | **1.91** |
| FID ↓ | 111.12 | 111.59 | 112.75 | **110.73** |
| PSNR（dB）↑ | **20.50** | 20.42 | 20.50 | 20.43 |

关键结论：

- **掩码范式压倒性优于流匹配**：即便 Flow-HWM 用了更多参数、训练更久，其 FID（约 110）仍远差于 Masked-HWM（约 10）；解码速度也更慢。作者据此认为，在该数据与算力约束下，掩码视频建模是更有效的范式。
- **趋势分化**：Masked-HWM 中联合注意力（Base Block）综合最好；Flow-HWM 中 Split Attention 更受青睐（最高 PSNR）。
- **参数共享几乎无损**：Modality Sharing 以少 26% 的参数追平 Base Block 质量；Full Sharing 在两条范式上把参数分别压缩约 39% 与约 52%（合计 33–53% 区间），并带来最小显存与最快推理，仅有微小质量损失。
- Full Sharing 的 Flow-HWM 参数减半却拿到该组最佳 FID（110.73），说明冗余主要来自各流独立参数。

定性上：Masked-HWM 能学到家具、小物体与整体纹理，手臂等大部件较准，但手指等精细部位偏模糊、易粘连，对光照变化鲁棒；Flow-HWM 能抓住墙/地/门等整体结构，但明显模糊、有斑块伪影，后段帧质量退化、难保直边与圆形，手臂倾向于回退到"标准手臂"外观而无法表现分布外姿态。

## 四、局限性

- **未做下游验证**：全文以 FID/PSNR 衡量视频质量，宣称世界模型可用于长时程规划与合成数据，但**没有任何策略学习、规划或控制任务的实验**验证其作为动力学模型的价值。
- **无动作可控性度量**：动作是核心条件，却没有"动作跟随一致性"这类指标，难判断生成是否真正服从 $a_f$，抑或主要靠视频外插。
- **短时程、单一具身、单一数据集**：仅预测 8 帧未来、只在单一 EVE 机器人的 1xGPT 数据上训练，泛化性与长时程一致性未知。
- **Flow-HWM 质量偏弱**：FID 高达约 110，伪影明显，且会回退到标准姿态而丢失分布外构型，实用性存疑。
- **缺乏与外部基线的横向对比**：只在自家四变体间比较，未与 Genie、iVideoGPT、IRASim、Cosmos 等公开工作在同一协议下对比。

## 五、评价与展望

**优点。** 论文的价值主要在**工程可复现性与系统化消融**：把"掩码 vs 流匹配 × 注意力结构 × 参数共享 × 模态分组"编织成统一实验矩阵，并给出可在 1–2 张 A6000 上跑通的开源配方，对小实验室友好；用 Cosmos 现成 tokenizer 降低了 VAE 训练成本；参数共享"深层共享、浅层独立"的做法给出了清晰且可迁移的经验。

**缺点与开放问题。** 其一，作为"世界模型"却缺失最关键的一环——没有任何规划/策略下游评估，FID/PSNR 与"是否能当动力学模型用"之间存在明显 gap；其二，缺动作可控性指标，无法排除模型在很大程度上忽略动作条件；其三，Flow-HWM 结果偏弱，究竟是流匹配范式本身不适配、还是该实现（隐空间压缩率、层数、训练量）欠调，尚不明朗——SD3、Cosmos 等表明连续流匹配在充分算力下可达高保真，因此"掩码优于流匹配"更可能是**受算力/数据规模约束下的结论**，而非普适定律。

**与公开工作的关系。** HWM 在 Genie（掩码时空 Transformer 的交互式世界模型）、Pandora、iVideoGPT（可扩展视频世界模型）、IRASim（机械臂动作模拟）、UniSim、Cosmos（大规模物理 AI 视频基础模型）之间，选择了"人形专用 + 极致轻量 + 开源"的差异化定位，填补了"物理可信且能在学术硬件上跑"的空白。相较之下，其规模（<1B）与时程都很小，更像是一个方法学基线与配方，而非能力上限。

**可能的改进方向（学术层面）。** ①引入动作跟随/逆动力学一致性等可控性度量；②在同协议下与 Genie/iVideoGPT 等横评；③补做基于生成 rollout 的策略学习或规划实验以兑现"世界模型"承诺；④延长预测时程并研究长时程漂移；⑤扩展到多具身/多数据集验证参数共享结论的稳健性；⑥对 Flow-HWM 做更充分的规模化以厘清范式差异的真实来源。
