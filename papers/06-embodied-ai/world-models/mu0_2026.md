## 一句话总结

μ₀ 把世界模型的预测目标从"稠密像素"和"本体相关动作"之间挪到中间地带——**语义交互点的平滑 3D 轨迹（trace）**：用数据引擎 TraceExtract 把异构的人/机器人视频自动转成"事件字幕化的 3D 轨迹"监督，预训练一个 query 条件、置换等变、以 B 样条控制点做 flow matching 的轨迹世界模型；冻结后接一个动作专家，即可把无动作视频预训练迁移成真机策略，在 3D 轨迹预测的所有指标/时域上均最优、RoboCasa365 仿真平均超 π₀ 5.0 分、UR3 真机 3 任务平均达 91.7%。

## 一、问题与动机

机器人学习面临"数据悖论"：视频数据海量但缺动作标签，动作标签稀缺、昂贵、且跨本体不通用。世界模型是一条绕过瓶颈的路，但**关键问题是"世界模型到底该预测什么"**：

- **像素空间视频模型**（如 Cosmos 等）可扩展、视觉先验广，但把大量模型容量花在稠密外观/背景重建上，难以捕捉操作真正需要的度量几何、接触结构与遮挡模式。
- **直接动作预测**（VLA）受限于标注的本体特异性与稀缺性，跨硬件不通用。

作者主张占据中间地带：预测**物体部件、工具、手、接触区域** 等语义交互点的 3D 轨迹——它紧凑地刻画了"无论用什么机器人都必须发生的运动改变"，天然是 embodiment-agnostic 的运动接口。

现有以运动为中心的方法（2D flow、3D flow、物体轨迹）有三个共性缺陷：(1) 对工具尖端/接触区等小而关键的区域采样不足；(2) 在局部或 2D 图像坐标下工作，把物体运动和相机运动混在一起；(3) 用 episode 级字幕配长演示，而非事件级意图。最接近的前作 TraceGen（Lee et al., CVPR 2026）在固定网格上预测 3D 轨迹、用 episode 级字幕、且推理时需要深度输入，三个维度都受限；μ₀ 正是针对这三点做系统性改进。

## 二、核心方法

μ₀ 由两部分组成：数据引擎 **TraceExtract**（离线把视频变成轨迹监督）与轨迹世界模型 **μ₀**（VLM backbone + Trace Expert），下游再接一个 **Action Expert**。

### 2.1 TraceExtract 数据引擎（三问：What / Where / How）

**What to move —— 语义关键点采样。** 抛弃固定网格(会被背景主导、小物体点太少、易漏掉工具尖端)，改用 DINOv2 patch 特征聚类成 entity 级分组，通过帧间二部匹配传播 entity 身份；给每个 entity 分配与其可见 patch 覆盖成比例的关键点配额(小物体保底)，在实体 mask 上做 farthest-point sampling 得到空间上分散的 query 点。再用一个 movement filter：把关键点 i 的轨迹投到参考相机，计算其在可见帧集上的最大成对位移(轨迹直径 $d_i$)，当 $d_i$ 超过阈值 $\tau_m = 40$ 像素才标为 moving，避免静止/背景点淹没交互信号（深度轴权重 $\lambda_z = 0.1$）。

**Where to move —— 全局-局部 3D 重建。** 用混合 VGGT 方案：先对整段视频均匀抽最多 $T_{\text{sparse}}$ 个 anchor 帧做**一次全局稀疏 pass**，得到公共全局坐标系下的外参与一个平均内参；再把视频切成不重叠 chunk 做**稠密局部 pass**，对每个 chunk 求一个刚体变换 $A^{(c)} \in \mathrm{SE}(3)$ 把 chunk 局部位姿对齐回全局锚点：

$$A^{(c)} = \arg\min_{A \in \mathrm{SE}(3)} \sum_{t \in \mathcal{S} \cap c} \| A\, E^{(c)}_t - E^{\text{sparse}}_t \|^2$$

因为每个 chunk 都**直接**对齐到同一批全局锚点(而非对齐到前一个 chunk)，对齐误差相互独立、不会跨 chunk 累积。3D 点追踪用 TAPIP3D，跨 chunk 用"每点上一有效世界坐标"做渐进传播(在世界坐标而非图像坐标做传播，对 egocentric 大幅相机运动鲁棒)。最后把轨迹重投到每个 chunk 的参考相机，得到**去掉相机运动、屏幕对齐的 3D 轨迹** $\mathbf{T}^{t:t+H}_{\text{ref},n} = [x_{n,i}, y_{n,i}, z_{n,i}]$，并用弧长重参数化归一化轨迹速度，缩小人/机演示的时长差异。

**How to move —— 事件级字幕。** 对有效 moving 关键点的逐帧加速度取均值得到标量运动信号，用 Savitzky–Golay 滤波平滑成 $\tilde{a}_t$，把 chunk 边界放在低加速度谷点：

$$b_i = \arg\min_{t \in [p_i, p_{i+1}]} \tilde{a}_t$$

这样切出的短事件天然对齐 reach / grasp / move / release 等子目标。每个 chunk 由 VLM 从起/中/末帧(可选叠加运动 mask 与 episode 级任务描述)生成分层字幕，再用纯文本 LLM 在滑窗上合并相邻字幕，同时得到细粒度事件字幕与粗粒度任务摘要。

**监督接口。** 每段视频变成元组 $\mathcal{D}_{\text{TE}} = \{( I_t, l_c, Q_t, \mathbf{T}^{t-h:t+H}_{\text{ref}} )\}$，其中 $I_t$ 是观测，$l_c$ 是事件/任务字幕，$Q_t = \{q^t_n\}_{n=1}^{N}$ 是 query 关键点集，$\mathbf{T}_{\text{ref}}$ 含过去与未来 3D 轨迹。μ₀ 学习预测映射 $( I_t, l_c, Q_t, \mathbf{T}^{t-h:t}_{\text{ref}} ) \mapsto \hat{\mathbf{T}}^{t:t+H}_{\text{ref}}$。相比前作 TraceGen 的 3D 轨迹数据集，轨迹策展规模扩大约 **8×**。

### 2.2 μ₀ 轨迹世界模型

**多模态条件 backbone。** 用预训练 SmolVLM2-2.2B 前缀(截取前 $L_{\text{vlm}}=20$ 层文本解码层)编码 RGB + 语言(可选深度)，**冻结**；再挂一个同深度(20 层)、隐藏宽度为 VLM 一半的 Trace Expert，每两层对 VLM 的 KV cache 做 cross-attention，把"语义记忆"(VLM 保留)与"运动计算"(Trace Expert 学习)分离。深度图不在 VLM 原生输入空间内，故经 Turbo colormap 渲染成 RGB、走一个单独可训练 patch stem，再与 RGB token 共享更深的 SigLIP 层。图像统一 resize 到 $512 \times 512$。

**置换等变的 Trace Expert + B 样条目标。** 每个关键点都当作一个可交换 query token，预测不依赖 query 顺序。目标不直接回归 32 步逐帧未来，而是先减锚点、再按每轴 95 分位尺度归一化：

$$\tilde{T}^1_{n,k} = (T^1_{n,k} - c_n)/s_\Delta$$

> 用大白话说：先去掉"当前锚点"这个慢变量(整体平移)，再把方差归一化到跟单位高斯噪声匹配，flow matching 才好学。

然后把每个关键点未来拟合成一条 3 次 B 样条(D=10 个控制点)，用带行权重(屏蔽无效步)的岭回归求控制点、曲线钉在锚点上：

$$P^\star_n = \arg\min_{P \in \mathbb{R}^{D \times 3}} \| M_n \odot (BP - [0; \tilde{T}^1_n]) \|_F^2 + \lambda_{\text{bsp}}^2 \| \Gamma P \|_F^2$$

> 用大白话说：不预测 32 个逐帧点，而是预测 10 个控制点画一条平滑曲线——更紧凑、能压掉追踪抖动、输出维度更低更好去噪；回滚时一个矩阵乘 $\hat{T}^1 = B\hat{P}$ 就解码出整条轨迹。历史 $h=8$、未来 $H=32$。

每个 query token 融合：历史/未来的 segment embedding、当前帧 $(u,v)$ 的 Fourier embedding、以及在该点像素处双线性采样的冻结 DINO 局部特征(经 2 层 MLP 融合，$e_{n,j} \leftarrow W_2\,\mathrm{SiLU}(W_1\,\mathrm{concat}(e_{n,j}, f^{\text{dino}}_n))$)，把全局场景语义与局部点级语义桥接。

**带语义结构的 flow matching。** 同一指令下未来可能有多条合理轨迹、还可能被遮挡截断，确定性回归会把多模态未来"平均糊掉"。故把 Trace Expert 训成 B 样条控制点上的条件 flow 模型。线性概率路径与损失：

$$P^\tau = \tau \epsilon + (1-\tau) P^\star, \qquad \mathcal{L}_{\text{flow}} = \mathbb{E}_{\tau, \epsilon} \big[ \| v_\theta(P^\tau, \tau, F_{\text{cond}}) - (\epsilon - P^\star) \|_2^2 \big]$$

> 用大白话说：从纯高斯噪声出发学一个速度场，把噪声"运输"到干净控制点；条件是 VLM 语义 + 每 query token + flow 时间 τ(经 adaLN-Zero 注入)。生成式而非回归式，才能保住接触丰富的多模态未来。

再加两项结构约束。**Validity/done 头** 预测每步轨迹是否还有效(处理遮挡/跟丢导致的截断)：

$$\mathcal{L}_{\text{done}} = \frac{\sum_{t=1}^{H} \ell_{\text{BCE}}(\hat{d}_{n,t}, y_{n,t})}{N}$$

**Rigidity loss** 让同一 DINO 语义簇(同一物体部件)内关键点的两两距离在整条控制点序列上尽量不变：

$$\mathcal{L}_{\text{rig}} = \mathbb{E}_{\tau, \epsilon} \Big[ \frac{1}{|R|} \sum_{(n,n') \in R} \mathrm{Var}_d \big( \| \hat{P}_{n,d} - \hat{P}_{n',d} \|_2^2 \big) \Big]$$

> 用大白话说：刚体约束——但不用只有合成环境才有的分割 mask，而是复用 TraceExtract 的 DINO 聚类当部件标签来保持局部几何一致。

总损失 $\mathcal{L} = \mathcal{L}_{\text{flow}} + \lambda_{\text{done}} \mathcal{L}_{\text{done}} + \lambda_{\text{rig}} \mathcal{L}_{\text{rig}}$。推理时对 $\tau \in [0,1]$ 用 4 步 Euler 积分 $v_\theta$，再经 B 样条基解码出绝对轨迹。为了在推理时缺历史轨迹/缺深度也能预测，训练用两级 history dropout(整体丢历史 prob 0.2、逐点独立丢 prob 0.3)、深度通道以 prob 0.7 随机置空。

### 2.3 轨迹条件动作专家（迁移到真机）

冻结整个预训练 μ₀(VLM backbone + Trace Expert)，只训一个动作专家，让学到的 3D 运动先验保持 embodiment-agnostic、只把动作监督限定在目标机器人接口上。关键设计：动作专家不等 μ₀ 完全去噪出轨迹，而是从纯噪声控制点 $P^0$ 只跑 **一步** 4 步 Euler，取 Trace Expert 的中间隐状态作运动描述子 $z_{\text{trace}}$；通过**门控 cross-attention** 注入 VLM 特征：

$$z_{\text{guided}} = z + \sigma(g) \cdot \mathrm{CA}(Q = \mathrm{LN}(z),\ K = V = \tilde{h}_{\text{trace}})$$

> 用大白话说：门 $g$ 是可学习标量、初始为 0，一开始几乎不干扰策略；只有运动线索确实有用时门才逐渐打开(弱注入起步)。

动作专家沿用 π₀.₅ 的自注意力架构、以 flow matching 生成连续动作块，额外输入 gripper 相机图(DINOv2 编码)、本体感知(MLP)、语言，噪声动作序列作为 query（$\mathcal{L}_{\text{action}} = \mathbb{E}\|v_\phi(a^\tau, \tau, z_{\text{guided}}, c) - (a - \epsilon_a)\|_2^2$）。

## 四、局限性

作者明确承认：(1) **误差继承感知栈**——语义聚类、3D 重建(VGGT)、追踪(TAPIP3D)、字幕任一环节失败都会产生噪声监督；(2) **轨迹表示只刻画几何与运动，不显式建模力、触觉反馈、接触模式**，对精细操作可能不足；(3) 动作专家评测集中在**桌面操作、有限本体与任务族**，移动操作、灵巧手、更长时域任务尚未验证。此外从审稿角度补充：仿真上 π₀.₅ 仍平均领先(尽管对比非数据对等)、门控注入等设计缺更细粒度分析、真机每任务仅 20 rollouts 统计力有限。

## 五、评价与展望

**优点。** (1) **表征选择干净且有说服力**：把"预测什么"落在语义交互点的度量 3D 轨迹上，回避了像素模型的容量浪费与 VLA 的本体绑定，且被证明是可复用、可迁移的运动接口；(2) **工程组合成体系**：DINOv2 语义采样 + VGGT 全局-局部重建(误差不累积) + TAPIP3D 世界坐标传播 + 加速度切段字幕，构成一条可扩展的无动作标注流水线，把 3D 轨迹策展扩大约 8×；(3) **建模细节到位**：B 样条压缩 + flow matching 保多模态 + validity 头处理截断 + 用 DINO 聚类替代分割 mask 的 rigidity loss，消融显示每一项都有贡献;(4) **迁移范式优雅**：冻结世界模型、只训动作专家、初始为 0 的门控注入，让 video-only 预训练干净地转成真机策略。

**与公开工作的关系。** μ₀ 这个命名显然是对 π₀/π₀.₅（Physical Intelligence 的 VLA）的呼应——同样以 flow matching 生成动作，但把"预训练承重"从动作标签换成 video-only 的 3D 轨迹世界模型。它与最接近的 TraceGen 是同组前作的直接升级：TraceGen 固定网格 + episode 字幕 + 推理需深度，三点都被 μ₀ 端到端修掉。相较 Track2Act/ATM 等 2D flow、3DFlowAction/Dream2Flow 等 3D flow，μ₀ 的差异在于"语义 query + 去相机运动的 reference-frame 3D 轨迹 + 事件级语言 + query 条件生成"这一整套接口，而非稠密网格或稠密场。

**开放问题与改进方向。** (1) 整条流水线依赖多个现成感知模型，误差累积是 scale 的天花板，能否端到端联合或引入自监督一致性校正值得探索；(2) 轨迹是**开环预测**——推理时能否闭环重规划(观测更新后重预测轨迹)以应对扰动，论文未涉及;(3) 不建模接触力/触觉，接插、拧、可变形物的精细阶段可能失效(尽管 Unfold Towel 表现意外地好);(4) "跨本体迁移"更多是概念主张，缺同一 μ₀ 迁移到差异较大本体(移动底盘、灵巧手)的直接实验;(5) model/data scaling 仍单调下降，说明在更大规模上潜力未尽，但也意味着当前数字尚非饱和结论。总体上，这是一篇把"3D 交互轨迹作为世界模型输出"论证得相当完整、且真机验证扎实的工作，代表了 embodiment-agnostic 运动先验预训练的一个有力方向。
