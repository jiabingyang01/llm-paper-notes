## 一句话总结

RynnVLA-002 在统一图文生成骨干 Chameleon 上新增状态/动作 tokenizer，把 VLA 的动作预测目标与世界模型的动作条件视觉预测目标合并进同一套共享词表、同一个自回归 Transformer 里训练，并针对动作 token 设计了专门的注意力掩码（阻止同一 chunk 内动作互相依赖）来抑制误差累积，又外挂一个轻量、并行解码的连续 Action Transformer 头解决离散自回归在真机上的过拟合和抖动问题；最终在无机器人数据预训练的情况下 LIBERO 达到 97.4% 平均成功率，真机 LeRobot SO100 实验中世界模型联合训练把整体成功率提升 50 个百分点（部分设置由 0% 拉到 80%）。

## 一、问题与动机

论文指出标准 VLA 架构存在三个缺陷：(1) 动作只出现在输出侧，模型无法形成动作的内部表征，即"不理解动作"；(2) 缺乏想象力，无法预测候选动作会如何演化，妨碍前瞻和反事实推理；(3) 缺乏显式物理理解，无法内化物体交互、接触、稳定性等动力学。世界模型天然具备"想象力"（预测未来观测），但反过来受限于无法直接输出动作，存在功能性缺口。

已有的"世界模型辅助 VLA"工作（3D-VLA、IRL-VLA、World-Env、World-VLA-Loop 等）通常把世界模型和策略模型作为两个独立组件，用世界模型产生的 rollout 或 reward 去外部优化策略，本质上仍是两个模型的松耦合。RynnVLA-002 想做的是把动作预测目标和动作条件视觉生成目标真正内化进同一个 Chameleon 式自回归骨干、共享同一个 token 空间，让两种能力在同一次训练里互相促进。

本文是作者团队 WorldVLA（Cen et al., 2025）的后续工作。WorldVLA 首次把离散化动作和图文 token 统一进单一词表，但发现两个问题：预训练 MLLM 主要见过图像和文本，动作域的泛化能力有限；自回归生成动作 chunk 时早期错误会传播到后续动作，导致误差累积严重。RynnVLA-002 针对这两点分别提出动作注意力掩码策略和连续 Action Transformer 头。

## 二、核心方法

### 2.1 问题形式化

VLA 目标：策略 $\pi$ 根据语言目标 $l$、上一时刻本体感受状态 $s_{t-1}$ 和观测历史 $o_{t-h:t}$ 生成动作

$$a_t \sim \pi(a_t \mid l, s_{t-1}, o_{t-h:t})$$

世界模型目标：给定历史观测和动作，预测下一时刻观测

$$\hat{o}_t \sim f(o_t \mid o_{t-h:t-1}, a_{t-h:t-1})$$

**大白话说**：一个"根据任务和过去看到的画面挑动作"，一个"根据过去的画面和动作猜下一帧长什么样"，本文让同一组参数 $\psi$ 的模型 $M_\psi$ 既能当 π 用也能当 f 用。

### 2.2 统一 Tokenization

模型从 Chameleon（Meta 的图文早融合统一模型）初始化，涉及四种 tokenizer：图像 tokenizer 是 VQ-GAN，压缩比 16、码本大小 8192，256×256 图像生成 256 个 token，512×512 生成 1024 个 token；文本 tokenizer 是训练好的 BPE；状态/动作 tokenizer 把连续本体状态和动作的每一维离散化为 256 个 bin（bin 宽度由训练数据的取值范围决定）。图像、文本、动作、状态 token 共享一个大小为 65536 的统一词表。

**VLA 数据的 token 序列**：`{text}{state}{image-front-wrist}×M{action}×K`，文本前缀固定为 "What action should the robot take to ⟨task⟩?"，损失为离散动作 token 的交叉熵 $\mathcal{L}_{dis\_action}$。LIBERO 上历史帧数 $M=2$，动作 chunk 大小 $K$ 在 LIBERO-Long/Spatial 上取 10，Object/Goal 上取 5。

**世界模型数据的 token 序列**：`{text}{images-front-wrist}{action}{images-front-wrist}×N`，文本前缀固定为 "Generate the next frame based on the current image and the action."，因为动作已经完全决定下一状态，不需要额外任务指令；损失为离散图像 token 的交叉熵 $\mathcal{L}_{img}$。实验中单次预测轮数 $N=1$ 以保证计算效率。

**训练目标**：VLA 数据与世界模型数据按 1:1 比例混合，单阶段微调、不额外做大规模机器人操作数据预训练（除非专门做预训练消融）。总损失 $\mathcal{L}_{dis} = \mathcal{L}_{dis\_action} + \mathcal{L}_{img}$。

**推理模式是解耦的**：同一次推理只执行一个查询任务——当作 VLA 用时只输出动作（不滚动生成未来图像），避免把自回归图像预测的复合误差带入实时控制回路；当作世界模型用时只预测未来观测。

### 2.3 动作 Chunk 生成的注意力掩码

朴素自回归下，chunk 内后一个动作依赖前一个动作，而动作模态在 MLLM 预训练里几乎没出现过，泛化能力弱，导致误差随生成序列长度累积、chunk 越长成功率掉得越厉害（消融见下）。论文提出一种改造过的注意力掩码（对应原文 Fig. 3(b)）：当前动作 token 只能关注文本和视觉 token，禁止关注同一 chunk 内此前生成的动作 token，使每个动作的生成相互独立、只依赖视觉/语言输入，从而阻断误差在动作序列内的传播。世界模型部分仍用常规因果注意力掩码。

**大白话说**：与其让机械臂"一步错步步错"地按顺序编排整条轨迹的动作，不如让每个时刻的动作都直接看图说话、互不牵连。

### 2.4 连续 Action Transformer 头

离散自回归动作模型在仿真里表现不错，但在真机实验中经常失败，原因有二：其一，大型自回归架构在有限的真机数据上容易过拟合，泛化差；其二，前述注意力掩码让 chunk 内每个动作独立生成，缺乏轨迹连续性，导致抖动、动作不平滑，成功率骤降。

为此论文借鉴 ACT（Zhao et al., 2023）思路，外挂一个专门的 Action Transformer：读入语言、图像、状态的完整上下文 token，用可学习的动作 query 一次前向并行输出整段连续动作 chunk。这个模块比骨干 LLM 小得多，不易过拟合、泛化更好、动作更平滑；并行解码 + 双向注意力也显著减少解码步数、加快推理。用 L1 回归损失 $\mathcal{L}_{conti\_action}$ 监督，总损失

$$\mathcal{L} = \mathcal{L}_{dis} + \alpha\mathcal{L}_{conti} = \mathcal{L}_{dis\_action} + \mathcal{L}_{img} + \alpha\mathcal{L}_{conti\_action}$$

实验中 $\alpha = 10$。离散动作 token 在训练中依然保留（与连续头共存），消融显示这能显著加速连续动作生成的收敛。

## 三、实验结果

### 3.1 LIBERO 仿真基准（Table 1，四个任务集 Spatial/Object/Goal/Long 平均成功率）

|---|---|---|---|
| OpenVLA | ✓ | 离散 | 76.5 |
| UniVLA | ✗ | 离散 | 95.2 |
| π0.5 | ✓ | 连续 | 96.9 |
| OpenVLA-OFT | ✓ | 连续 | 97.1 |
| X-VLA | ✓ | 连续 | 98.1 |
| EO-1 | ✓ | 连续 | **98.2** |
| **RynnVLA-002-Discrete** | ✗ | 离散 | 93.3 |

RynnVLA-002-Continuous 四项子任务分别为 Spatial 99.0、Object 99.8、Goal 96.4、Long 94.4。在完全不使用机器人数据预训练的情况下，成绩逼近甚至超过多数依赖大规模机器人/跨本体预训练的基线（仅次于有预训练的 EO-1）。

### 3.3 关键消融

**离散动作消融（LIBERO，Table 3）**：仅 VLA 62.8% → 加世界模型 67.2% → 只加动作 chunking（无掩码，无世界模型）反而掉到 54.0%（误差累积，Spatial 骤降到 36.7、Long 降到 16.9）→ chunking + 无掩码但有世界模型 76.6% → chunking + 世界模型 + 本文注意力掩码（完整版）78.1%（最佳）。

**连续动作消融（LIBERO，Table 4）**：基础连续动作头 84.5% → 加腕部相机 91.6% → 再加世界模型 94.6% → 再加本体状态 97.4%（完整版，最佳）。

**真机消融（Table 5）**：离散动作 + 世界模型 + 腕部相机 + 本体状态（完整配置但动作类型为离散）三项设置全部 0% 成功率；连续动作但缺腕部相机或缺本体状态同样全部 0%；连续动作 + 腕部相机 + 本体状态但去掉世界模型仅 30.0/10.0/0；连续动作 + 世界模型 + 腕部相机 + 本体状态（完整版）达到 80.0/80.0/50.0——真机上世界模型的贡献远大于仿真（平均提升约 50 个百分点，对应摘要中"提升 50%"的说法），且离散动作在真机上完全失效。

**世界模型质量消融（Table 6，LIBERO 验证集，FVD↓/PSNR↑/SSIM↑/LPIPS↓）**：与 VLA 数据联合训练的 Action World Model 在四个任务集上普遍优于只用世界模型数据单独训练的 World Model，例如 Object 集 FVD 从 1141.6 降到 877.2、SSIM 从 59.59 升到 65.03；Long 集 FVD 从 557.73 降到 427.86。说明 VLA 数据里继承的图像理解能力反过来提升了世界模型的生成质量——双向增益在生成侧同样成立。

**其他消融**：动作 chunk 长度从 5 增到 20 时，朴素注意力掩码下成功率明显下滑，本文掩码下则保持稳定（Fig. 6）；训练时保留离散动作 token 能显著加快连续动作头的收敛速度（Fig. 7）；效率上连续动作生成因并行解码大幅快于离散自回归，如 chunk=10 时连续版本推理频率 15.78 Hz 对离散版本的 2.74 Hz（约 5.8 倍），且性能相当或更优（Table 7）；世界模型预训练（先用世界模型目标冷启动再训 VLA）在四个 LIBERO 子集上均带来提升，Long 集从 23.0 提升到 30.2（Table 8）。

## 四、局限性

论文在"Real-World Scope and Cost"一节自陈：

- 真机评测只在单一 SO100 机械臂上做拾取-放置任务，未覆盖更广泛的机器人平台、长时程移动操作或可变形物体操作。
- 当前自回归图像 token 方案计算代价高：由于图像生成会引入大量 token，单个真机任务的训练大约需要四天。
- 作者将更多机器人平台、更复杂操作任务、更高效的图像 token 生成列为未来方向。

此外，从方法设计看还存在几点隐含限制：推理时动作预测与图像预测是解耦查询的，模型不会用自己生成的未来帧在测试时做显式的滚动规划或搜索，世界模型带来的收益主要体现在训练阶段的共享表征正则化，而非测试时前瞻推理；世界模型的预测轮数固定为 $N=1$（单帧预测），未展示更长视频 rollout 下的想象能力评估。

## 五、评价与展望

**优点**：本文最有价值的贡献是给出了"世界模型与 VLA 双向增益"的可验证证据——不仅是世界模型数据提升了 VLA（Table 3/4/5 的一致趋势），VLA 数据本身也反过来提升了世界模型的视频生成质量（Table 6），这与大多数"世界模型只作为外部 reward/simulator"的工作（如 3D-VLA、IRL-VLA、World-Env、World-VLA-Loop）形成了鲜明对比，是共享骨干、共享词表这一设计选择的直接体现。消融研究做得相当系统（覆盖离散/连续动作、注意力掩码、chunk 长度、腕部相机、本体状态、世界模型预训练、推理效率等 7-8 张表），论证链条完整。在完全不做机器人数据预训练的前提下于 LIBERO 上逼近甚至部分超过依赖大规模预训练的强基线（π0.5、OpenVLA-OFT、X-VLA），说明统一动作-世界目标本身可能是一种有效的学习信号来源，一定程度上替代了机器人专属预训练的作用。

真机上离散动作全面失效（Table 5 三行 0%）而连续动作头显著可用，是一个有实践参考价值的负面结果，与 π0/π0.5/OpenVLA-OFT/FAST 等业内转向连续或 flow-matching 动作头的趋势相互印证。

**局限与开放问题**：真机评测范围较窄（单臂、两类拾取放置任务、每设置仅 10 次试验），统计置信度有限，尚不能验证方法在更复杂长时程或可变形物体任务上的有效性。训练成本高企（约四天/任务），根源在于自回归逐 token 生成图像的效率瓶颈，这也是当前"用图像 token 化世界模型联合训练 VLA"这条技术路线（与 GR-2、VPP、RynnVLA-001、UniPi、GeVRM 等视觉生成式 VLA 同属一脉）普遍面临的可扩展性问题。此外，世界模型目前只用于训练阶段的表征正则化，并未像 Dreamer 系列、Genie 或部分基于世界模型做测试时 rollout/搜索的方法那样在推理时真正做"想象后再决策"，如何把这种统一骨干的世界模型能力用于测试时前瞻规划或动作候选筛选，是一个自然的后续方向。是否能在更大规模、更多本体的真实机器人数据上验证统一目标带来的增益是否随数据量持续放大，也仍是未解问题。
