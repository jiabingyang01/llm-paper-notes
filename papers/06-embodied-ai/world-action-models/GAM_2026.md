## 一句话总结

GAM 把预训练几何基础模型（GFM，具体用 DA3-Giant）从中间层劈成"浅层编码器 + 深层解码器"，在劈开处插入一个因果 Transformer，在同一条 token 序列里联合预测下一步的几何隐 token 和动作 token，再让 GFM 原本的深层块把二者一起解码为未来深度图与可执行动作块；这样只用 1.4B 参数（983M 可训练）就在 LIBERO-Plus 相机扰动上把最强基线（Cosmos-Policy）的 73.4% 拉到 83.1%（+9.7pp），推理延迟压到 6.9ms（约 145Hz），比扩散式 Cosmos-Policy 快约 55 倍。

## 一、问题与动机

当前生成式机器人策略主要有三条路线，作者认为它们都没有把"3D 几何"真正吃透：

1. **VLA（视觉-语言-动作模型）**：把预训练 VLM 的视觉-语言对齐能力迁移到动作预测，但骨干本身只在 2D 图像上训练，深度、尺度、遮挡等 3D 线索都是隐式的单目线索，需要动作解码器自己去解耦，导致相机视角、机器人初始状态变化时泛化差。
2. **Video WAM（视频式世界-动作模型，如 Cosmos-Policy、Fast-WAM、WorldVLA）**：复用预训练视频生成模型联合预测未来帧和动作，训练时的"预测未来"信号确实有用，但视频骨干本质上还是在 2D 像素空间里运作，深度/尺度/遮挡依然没有被显式建模。
3. **Geometry-aware VLA（如 Spatial Forcing、ROCKET）**：通过表示对齐把 GFM（VGGT、DA3 等）的特征蒸馏进 VLA backbone，或者在 GFM 最终特征上接一个轻量动作头。这类方法确实引入了几何先验，但只是把 GFM 当作一个静态特征抽取器使用——GFM 内部多层的几何结构从未被真正复用为策略自身的时序建模和动作生成基质。

GAM 的核心问题是：**能不能直接把 GFM 本体重新用作策略网络，而不是仅仅蒸馏它的特征？** 具体做法是把 GFM 从某个中间层"劈开"：浅层继续做观测编码器，插入一个因果预测器在劈开处联合预测未来的几何隐 token 和动作 token,再让 GFM 原本的深层块（原本是用来把浅层特征精加工成 3D 几何的）承担"世界模型解码器"的角色，同时解码出未来几何和最终动作。整个设计只需要在预训练 GFM 上做"最小架构改动"（插入一个因果预测器 + 两个轻量头），就能让语言条件的时序世界建模能力内生地长在几何基础模型里。

## 二、核心方法

### 2.1 预备知识：几何基础模型（GFM）

GFM（如 VGGT、DA3）是一个把 $V$ 张 RGB 图 $\mathcal{I} = \{I_v\}_{v=1}^V$ 映射到稠密 3D 几何（per-pixel 深度 $D_v$ 或点图 $P_v$）以及相机内外参 $(K_v, \xi_v)$ 的前馈 Transformer。每张图切成 $P$ 个不重叠 patch，经过 patch embedding 得到

$$\mathbf{z}_v^{(0)} = [\mathbf{c}_v, \mathbf{x}_v^1, \dots, \mathbf{x}_v^P] \in \mathbb{R}^{(1+P)\times d}$$

其中 $\mathbf{c}_v$ 是相机 token，$\{\mathbf{x}_v^j\}$ 是 patch token。$M$ 层 Transformer block 交替使用两种注意力：frame-wise attention 只在单张图内部的 $1+P$ 个 token 上算，global attention 在所有视角的 $V(1+P)$ 个 token 上联合算：

$$\mathbf{Z}^{(m)} = f^{(m)}(\mathbf{Z}^{(m-1)}), \quad f^{(m)} \in \{f^{(m)}_{\text{frame}}, f^{(m)}_{\text{global}}\}$$

最终从若干选定中间层 $\mathcal{S} = \{m_1, m_2, m_3, m_4\}$ 的隐状态经 DPT 头解码出稠密几何。

**用大白话说**：GFM 就是一个"看多张图猜 3D 结构"的模型，内部若干层 Transformer 交替做"单图内部理解"和"跨视角信息融合"，几何结果是从中间几层的隐状态里经过一个专门的解码头（DPT）读出来的，而不是只从最后一层读。

### 2.2 问题形式化

策略在长度为 $H$ 的历史观测/状态/动作窗口和语言指令条件下，预测长度为 $C$ 的未来动作块：

$$\pi_\theta: \left(\{o_{t-H+1},\dots,o_t\}, \{s_{t-H+1},\dots,s_t\}, \{a_{t-H},\dots,a_{t-1}\}, \ell\right) \mapsto \hat{a}_t$$

### 2.3 三阶段架构

**(1) 观测编码器**：直接复用 GFM 浅层 $E_{\le L_s} = f^{(L_s)}\circ\cdots\circ f^{(1)}$，把每个时间步的多视角 RGB 独立编码到劈开层表示 $\mathbf{Z}_{t'}^{(L_s)}$。劈开层 $L_s$ 的选择有一个硬约束：既要足够深以提取到丰富的视觉特征，又必须浅于 DPT 头用到的最早层（$L_s < m_1$），这样预测出的未来 token 才能被同一套 DPT 头正确解码成未来几何。

**(2) 因果未来预测器** $g_\phi$：插在劈开层的因果 Transformer，把本体感知 $\mathbf{p}_{t'} = \psi_s(s_{t'})$、上一步动作 $\mathbf{q}_{t'} = \psi_a(a_{t'-1})$ 和 GFM 的劈开层 token 拼成每步 token 块

$$\mathbf{U}_{t'} = [\mathbf{p}_{t'}; \mathbf{q}_{t'}; \mathbf{Z}_{t'}^{(L_s)}]$$

再与冻结文本编码器（T5）产生的语言 token $\mathbf{L}_\ell$ 拼成整条序列 $\mathbf{X} = [\mathbf{L}_\ell; \mathbf{U}_{t'-H+1}; \dots; \mathbf{U}_{t'}]$，经 block-causal self-attention 处理（保证不看到未来）。在预测器最后一层，"几何槽"位置读出预测的未来几何隐 token $\tilde{\mathbf{Z}}_{t'+1}^{(L_s)}$，"上一步动作槽"位置读出预测的下一动作 token $\tilde{a}_{t'} \in \mathbb{R}^d$——这与因果语言模型的 next-token prediction 是直接类比,只是这里同时预测几何和动作两类 next-token。

**用大白话说**：这一步相当于给 GFM 半成品的视觉特征接上一个小型因果语言模型，输入"过去几步的几何特征 + 机器人状态 + 上一步动作 + 语言指令"，输出"下一步的几何特征该长什么样"和"下一步该怎么动"，两者在同一段因果注意力序列里被同时预测出来,动作因此天然和空间几何表征绑在一起,而不是分开两个网络各算各的。

**(3) 特征传播与动作解码**：预测出的单个动作 token 复制 $V$ 份（每个视角一份），与预测的几何 token 拼接后送入 GFM 剩余的深层块 $D_{>L_s}$：

$$\tilde{\mathbf{Z}}_{t'+1}^{(M)} = \left(f^{(M)}\circ\cdots\circ f^{(L_s+1)}\right)\left(\left[[\tilde{\mathbf{Z}}_{1,t'+1}^{(L_s)};\tilde{a}_{1,t'}],\dots,[\tilde{\mathbf{Z}}_{V,t'+1}^{(L_s)};\tilde{a}_{V,t'}]\right]\right)$$

为防止未来信息泄露，这一传播过程延续因果预测器的因果掩码策略，扩展到 GFM 剩余的 global attention 层。最终两个头做解码：轻量动作头 $h_{\text{act}}$ 在 context window 上聚合动作 token 回归出可执行动作块 $\hat{a}_{t'}$；GFM 原本的深度头（DPT head）$h_{\text{depth}}$ 把几何 token 解码成与动作对齐的未来深度图。

**用大白话说**：GFM 原本的深层网络是用来把半成品视觉特征精加工成最终 3D 几何的流水线，GAM 原样借用这条流水线，只是现在喂进去的是"预测出的未来几何半成品"——于是 GFM 深层里蕴含的 3D 先验知识被直接复用成了世界模型的解码器，而不用另外训练一个新解码器；动作 token 也搭便车走了这条流水线，相当于借助深层几何精加工过程"顺便"把动作表征也精炼了一遍。

### 2.4 训练目标

三项损失联合优化：

$$\mathcal{L}_{\text{total}} = \lambda_{\text{act}}\mathcal{L}_{\text{act}} + \lambda_{\text{feat}}\mathcal{L}_{\text{feat}} + \lambda_{\text{depth}}\mathcal{L}_{\text{depth}}$$

- **动作损失**：解码动作块 $\hat{a}_{t'}$ 与专家动作 $a_{t'}$ 在整个 context window 上的 $\ell_1$ 回归。
- **未来特征损失**（锚定几何时序动态）：

$$\mathcal{L}_{\text{feat}} = \sum_{t' \in \mathcal{H}} \left\| \tilde{\mathbf{Z}}_{t'+1}^{(L_s)} - \mathbf{Z}_{t'+1}^{(L_s)} \right\|_1$$

把预测的未来几何隐 token 对齐"冻结 GFM 直接在真实下一帧上算出来的"劈开层 token。**用大白话说**：这是让因果预测器学会预测"下一帧真实会是什么样的几何特征"，本质是一种隐空间里的下一帧预测,完全不需要解码成图像就能监督。

- **未来深度损失**：用深度头把预测几何 token 解码出的未来深度图，对齐真实未来深度（仿真用模拟器真值；真实机器人场景下真值几何不可得，改用预训练 GFM 自己产生的伪标签深度），采用 GFM 自带的 scale-invariant + gradient-matching 惩罚项监督。

三项权重 $\lambda_{\text{act}}=3,\ \lambda_{\text{feat}}=1,\ \lambda_{\text{depth}}=3$。推理时用 KV 缓存维护历史 context，每一步只需处理新观测 $o_t$ 和上一步动作 $a_{t-1}$，单次前向即可。

### 2.5 实现细节

Backbone 采用在 Track4World 上微调过的 **DA3-Giant**，插入一个宽度 $d_g=1024$ 的 12 层因果预测器，劈开层选在 $L_s=12$（对应 GFM 里交替注意力开始的位置）。预训练 context $H=4$，post-training 时 $H=1$；动作块 $C=8$ 步，动作空间 $d_a=7$（末端位姿），本体感知 $d_s=7$。预训练数据共 784K 条单臂轨迹，来自 Open X-Embodiment（72%）、MimicGen（18%）、RoboCasa365（10%）。两阶段训练：Stage 1 在大规模混合数据上端到端联合训练预测器、动作头与 GFM backbone，64×NVIDIA GH200，batch size 1024，约 96 小时；Stage 2 在各 benchmark 官方训练集上微调整模型，16×NVIDIA GH200，batch size 160，约 48 小时。

参数量（Table 9）：总量约 1404.8M（backbone ViT-Giant 40 blocks 1136.5M + 冻结 DPT 头 50.1M + 因果预测器 210.2M + 动作头 8.0M），可训练参数约 983.2M（backbone 第 13–39 层可训练 ≈765M，第 0–12 层与 DPT 头冻结；预测器与动作头全部可训练）。

**其他消融（附录 C.1，Table 14）**：把动作损失直接加在预测器输出的动作 token 上（不经过深层 GFM 传播）得到 Orig 98.4 / Plus 84.1，弱于 GAM 完整设计的 99.6 / 89.7，说明让动作 token 也走一遍深层几何解码器能带来额外提升，尤其在相机扰动下更明显。

## 四、局限性

- **语言理解能力受限于冻结文本编码器**：作者在结论中明确指出，"GAM 的语言推理和常识能力受限于冻结的文本编码器（T5），集成大语言模型或外部推理模块是自然的下一步"。这与实验现象吻合——GAM 在 Language 扰动维度（84.8）并非最优，被 π0.5（86.5）和 Cosmos-Policy（89.3）超过。
- **并非在所有扰动维度都最优**：Robot（本体/初始状态扰动，70.0，低于 ROCKET 的 75.6）和 Layout（79.1，低于 π0.5 的 86.0、Cosmos-Policy 的 84.0）两个维度上 GAM 都不是第一，说明几何先验对相机类扰动的收益明显大于对机器人本体状态、场景布局类扰动的收益。
- **劈开层位置高度敏感、需要针对 backbone 精调**：层选择消融显示插入位置早一层或晚十几层就会导致训练完全崩溃（$L_s=0$ 或 $L_s\ge27$ 时成功率跌到个位数），且必须满足 $L_s < m_1$ 这一硬约束（早于 DPT 头使用的最早层）。这意味着换用不同层数、不同注意力模式设计的 GFM backbone 时，可能需要重新摸索最佳分割点，对方法的跨 backbone 可迁移性构成一定限制。
- **训练资源门槛较高**：预训练阶段使用 64×NVIDIA GH200 GPU 约 96 小时，后训练阶段 16×GH200 约 48 小时，对复现和后续研究有较高的算力门槛。
- **真实机器人评测规模有限**：仅 4 个任务、每设置 10 次试验（每任务共 20 trials），任务集合本身（摞盘、叠物、开抽屉类）时序跨度也偏短，长时序、多阶段接触密集型任务上的表现尚未验证。

## 五、评价与展望

**贡献与优点**：GAM 提出了一个简洁而有说服力的架构思路——与其把几何基础模型当作静态特征蒸馏源（如 Spatial Forcing、ROCKET 等 geometry-aware VLA 的做法），不如直接把 GFM 本体劈开、复用其深层解码路径作为世界模型和动作解码器。这一设计有两个关键点区别于此前范式：其一，动作与未来几何在同一条自回归 token 序列、同一次 backbone 前向里联合建模，而不是分离的头或独立的扩散过程；其二，GFM 原本用于处理"已观测"输入的深层块被显式复用来解码"预测的未来"token。实验结果表明这一设计在保持极小模型规模（1.4B，仅 983M 可训练）的同时，在相机扰动这一最考验几何理解能力的场景下取得了大幅领先（LIBERO-Plus Camera 83.1% vs 第二名 Cosmos-Policy 73.4%），推理延迟压到个位数毫秒级（6.9ms，约 145Hz），是"几何先验应该如何被复用进策略网络"这一问题上一个有分量的正面案例。

**与其他公开工作的关系**：GAM 与同期的 geometry-aware VLA 工作（Spatial Forcing、ROCKET）在"利用 GFM 提升策略几何理解"这一目标上高度相关，但路线截然不同——后两者通过表示对齐把 GFM 特征蒸馏进独立的 VLA backbone（π0.5），GFM 本身在推理时基本不参与；GAM 则直接让 GFM 本体承担时序建模和动作解码的角色。实验中 Spatial Forcing、ROCKET 在 LIBERO-Plus 上掉分幅度反而是全表最大（分别 ↓58.3pp、↓46.6pp），这一对比结果为"仅做被动特征蒸馏不足以获得鲁棒性、需要更深度地复用 GFM 结构"提供了一个具体的反例证据。与 video WAM 路线（Cosmos-Policy、Fast-WAM）相比，GAM 用几何隐空间预测替代了像素/视频隐空间预测，规避了视频生成固有的高推理开销问题，这与同期质疑"WAM 是否需要视频生成/测试时未来想象"的工作（如 Fast-WAM 本身、以及若干图像编辑式 WAM 变体）在动机上呼应，但 GAM 给出的解法是切换到几何基础模型而非图像编辑模型，提供了另一条不依赖视频/图像生成骨干的技术路径。

**开放问题与可能的改进方向**：(1) 语言理解能力受制于冻结文本编码器是论文自陈的局限，如何在不牺牲几何先验和推理效率的前提下引入更强的语言/常识推理能力（例如接入 LLM 或做指令级别的分层规划）仍是开放问题；(2) 劈开层位置的选择目前依赖网格搜索式消融，且性能对该超参数极其敏感，如果能找到更原理性的准则（例如根据 GFM 各层的几何信息量或注意力模式变化自动定位分割点），将显著提升方法对新 GFM backbone 的可迁移性；(3) 论文的真实机器人任务集合和仿真 benchmark（LIBERO、RoboCasa）时序跨度总体有限，该架构在需要长时序规划、多阶段接触约束更强的操作任务上是否依然保持优势，以及未来几何预测的误差是否会在长 horizon 上累积并反过来误导动作预测，仍需要更系统的验证；(4) 目前只验证了单一 GFM backbone（DA3-Giant），骨干几何理解能力的强弱（例如换用不同规模、不同预训练数据的 GFM）与下游策略鲁棒性之间的定量关系还缺乏细粒度分析。
