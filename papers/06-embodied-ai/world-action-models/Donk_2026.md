## 一句话总结

Donk 用同一个基于 Wan2.2 视频扩散 Transformer 的联合去噪骨干,在流匹配框架下同时对视频 token 和双手 MANO 动作 token 去噪,靠是否提供首帧图像在"观测条件动作策略(TI2VA)"与"纯文本配对视频-动作数据引擎(T2VA)"两种模式间切换;在 OakInk2 基准上手部 RMSE 降到 0.238(优于 VITRA 的 0.444 与 Being-H0 系列),同时在 EgoDex/LOME 协议下视频质量(LPIPS 0.2992)仍具竞争力,纯文本 T2VA 模式相比同规模视频基线把 FVD 从 306.2 降到 191.1。

## 一、问题与动机

现有 World Action Models(WAMs)把视频基础模型的视觉-动态先验和可执行机器人动作对齐,但通常被限定为"观测条件策略"——给定语言指令和当前观测图像,预测未来观测和动作轨迹,即建模 $p(\text{video}, \text{action} \mid \text{text}, \text{observation})$。作者指出,初始观测只是条件空间里的一种可能情形,并非视频-动作联合生成的内在要求,可以把问题重新表述为更广义的文本/图像条件视频-动作联合分布。当给定首帧图像,该分布退化为 TI2VA(text-image-to-video-action)策略;当不给图像时则是纯文本条件的 T2VA(text-to-video-action)——语言驱动的配对视频加动作生成,可以充当数据引擎。

这一点对灵巧操作尤其有意义:成对的机器人视觉-动作轨迹因遥操作、标定、精细动作标注难度大而昂贵,而大规模人-物交互视频和文本条件视频先验相对易得,T2VA 分支理论上可以把这些先验转化为结构化、与动作对齐的监督信号。实现该统一表述的难点在于:第一,把动作预测直接注入视频生成容易干扰视频 token 表示、降低视觉保真度;第二,先生成/重建视频再事后提取动作的两阶段流水线会引入脆弱的中间表示、时间错位和误差累积。因此核心挑战是在单个模型里联合学习视频与动作生成,同时保持视觉保真度、动作精度和视频-动作的时间对应关系。

## 二、核心方法

Donk 以 Wan2.2 TI2V-5B 预训练视频扩散 Transformer 为骨干,在流匹配(flow matching)范式下联合去噪视频 token 和以 MANO 手部参数序列表示的双手动作 token。

### 统一视频-动作分布

令 $c$ 为语言指令,$V_{0:T}$ 为操作视频,$A_{1:T}$ 为对齐的未来手部动作轨迹,$T>0$ 为固定时域,$I_\star \in \{I_0, \varnothing\}$ 表示是否提供首帧图像,$g_0$ 表示初始手-相机锚点(初始 MANO 手部状态加相机内参)。Donk 建模统一条件分布

$$p_\theta(V_{0:T}, A_{1:T} \mid c, I_\star, g_0)$$

用大白话说:模型本质上回答"给定文字描述、可选的首帧照片、可选的手部初始姿态,未来的视频和动作会怎样演化",首帧图像存在与否只是同一个分布的两种取值情形,而不是两个独立模型。

当提供首帧图像时,退化为 TI2VA 策略模式:

$$p_\theta^{\text{policy}}(V_{1:T}, A_{1:T} \mid I_0, g_0) \triangleq p_\theta(V_{1:T}, A_{1:T} \mid c, I_\star = I_0, g_0)$$

此时 $g_0$ 由观测到的初始手部状态与相机内参得到,模型像一个观测条件的动作策略。当不提供图像时,退化为 T2VA 数据引擎模式:

$$p_\theta^{\text{engine}}(V_{0:T}, A_{1:T} \mid c, \tilde g_0) \triangleq p_\theta(V_{0:T}, A_{1:T} \mid c, I_\star = \varnothing, \tilde g_0)$$

此时 $\tilde g_0$ 只提供一个"合理的"首帧几何脚手架(而非未来动作规划或轨迹级条件),整个交互 rollout(视频加动作)从纯语言生成。用大白话说:同一个生成器,插上图像就是策略、拔掉图像就是数据生成器,TI2VA 策略学习和 T2VA 数据生成变成同一个端到端生成模型的两种条件模式,而不是两条独立流水线。

### 联合 token 化与视频保留注意力

视频经预训练 Wan VAE 编码为 $x^\star = \mathcal E(V_{0:T})$,动作表示为归一化的连续双手 MANO 轨迹 $a^\star = A_{1:T}$(训练时对无效/缺失手做掩码)。Wan 主干把视频 latent patch 化为视频 token,另外用轻量动作编码器和锚点编码器把未来 MANO 轨迹和初始手-相机锚点嵌入为动作 token 与锚点 token,拼接得到联合序列 $z = [z^{\text{video}}, z^{\text{action}}, z^{\text{anchor}}]$。原始 Wan 头输出视频,新增的轻量动作头输出 MANO 动作。

若完全联合的注意力允许视频 token 关注新引入的动作、锚点 token,会扰动预训练视频生成先验。因此 Donk 使用"视频保留注意力掩码":视频 query 只关注视频 token(保持预训练自注意力结构不变),而动作与锚点 query 关注全序列(视频加动作加锚点)。用大白话说:视频这条主干"看不见"新加的动作分支,继续按原来的方式生成画面;动作分支则可以同时"偷看"生成的视频和初始锚点,据此对齐手部动作,像单向玻璃一样保护了预训练视频先验的稳定性,又让动作学习能利用视觉信息。

### 锚点图控制器(Anchor Map Controller)

语言和图像条件对手部在图像平面的位置、姿态控制力有限,因此引入首帧手-相机锚点 $g_0=(s_0,K)$($s_0$ 为初始 MANO 手部状态,$K$ 为相机内参)作为显式几何控制信号:渲染出配色编码的 MANO 骨架图 $M_0=\mathcal R(g_0)$,用冻结的 Wan VAE 编码得到锚点 latent $m_0=\mathcal E(M_0)$;轻量适配器 $G_{\text{anc}}$ 把 patch 化的锚点 latent 映射为共享锚点控制表征 $C$,再由逐层 MLP 为每个选定 Wan 层产生锚点提示 $H_\ell$:

$$C = G_{\text{anc}}(\text{Patch}(m_0)), \quad H_\ell = \text{MLP}_\ell(C), \ \ell \in \mathcal S$$

通过门控首帧锚点注入方式接入网络:

$$z_{\ell,0}^{\text{video}} \leftarrow z_{\ell,0}^{\text{video}} + \gamma_\ell H_\ell, \qquad z_{\ell,t>0}^{\text{video}} \leftarrow z_{\ell,t>0}^{\text{video}}$$

门控标量 $\gamma_\ell$ 初始化为零,训练从预训练 Wan 行为开始,只对首帧视频 token 注入几何提示,后续帧的手-物演化交由联合视频-动作去噪器学习。用大白话说:先给模型一张"手部骨架示意图"钉在第一帧,告诉它手一开始大概长什么样、在哪,后面怎么动完全靠联合去噪自己学,而且这个"钉图钉"的强度是从零学起,不会一上来就破坏预训练模型原有的生成能力。

对于纯文本 T2VA,没有观测到的首帧手部状态,因此额外训练一个轻量的文本条件初始化器,学习首帧手-相机状态的经验分布,用来实例化 $\tilde g_0$;该初始化状态只用来渲染初始锚点图,未来的交互视频和 MANO 轨迹仍由共享的视频-动作去噪器生成。

### 训练目标

Donk 用视频-动作流匹配、交互聚焦的视觉监督以及教师先验正则化联合训练。去噪器预测视频和动作速度 $(\hat v_x, \hat v_a)$,由对应流匹配目标 $(v_x, v_a)$ 监督:

$$\mathcal L_{\text{video}} = \|\hat v_x - v_x\|_2^2, \qquad \mathcal L_{\text{action}} = \frac{\|M_a \odot (\hat v_a - v_a)\|_2^2}{\max(\sum M_a, 1)}$$

其中 $M_a$ 掩掉无效手部维度。为强调手-物交互区域,额外用手部聚焦的视频损失 $\mathcal L_{\text{gaze}}$,对渲染手部区域附近的视频流误差加权。同时引入冻结的 Wan 教师先验来稳定视觉生成路径,教师接收相同的视频 latent 和文本条件,预测视频速度 $\hat v_x^{\text{tea}}$:

$$\mathcal L_{\text{prior}} = \|\hat v_x - \hat v_x^{\text{tea}}\|_2^2$$

该项只在保留图像条件时施加,避免纯文本分支模仿一个"看得到图像"的教师。完整损失为

$$\mathcal L_{\text{Donk}} = \lambda_v(\mathcal L_{\text{video}} + \lambda_g \mathcal L_{\text{gaze}}) + \lambda_a \mathcal L_{\text{action}} + \lambda_p \mathcal L_{\text{prior}}$$

用大白话说:视频照常按扩散/流匹配训练,动作按同样的思路训练但只算有效手部维度的误差,手附近区域额外加权让模型更关注接触和操作细节,再用一个不参与更新的"冻结老师"防止图像条件的视频生成路径跑偏。训练时冻结文本编码器、VAE、教师模型和大部分 Wan 层,只训练动作/锚点接口、锚点图适配器、动作头和一小部分 Wan 层。

推理时,TI2VA 模式接收 $(c, I_0, g_0)$,用 $I_0$ 钳制首帧视频 latent,生成未来视频-动作轨迹;T2VA 模式只接收语言指令 $c$,用文本条件初始化器实例化合理的初始手-相机锚点 $\tilde g_0$、渲染其锚点图,再用同一个去噪器生成交互视频和同步的 MANO 动作轨迹。

## 三、实验结果

训练数据为 VITRA-1M 数据集,使用 64 张 NVIDIA Hopper 架构 GPU(96GB 显存),PyTorch FSDP2,单卡处理一个 clip,有效 batch size 64,bfloat16 精度,AdamW,恒定学习率 $2\times10^{-5}$,$(\beta_1,\beta_2)=(0.9,0.999)$,$\epsilon=10^{-8}$,权重衰减 0.01,梯度裁剪 1.0。

**TI2VA 动作精度(OakInk2 第一人称基准,均越低越好,10 次采样取 best-of-K)**：

| 方法 | Hand RMSE | ADE (K5/K10) | FDE (K5/K10) | DTW-S (K5/K10) | DTW-L (K5/K10) | ROT (K5/K10) |
|---|---|---|---|---|---|---|
| VITRA | 0.444 | 0.067 / 0.065 | 0.108 / 0.105 | 0.062 / 0.060 | 0.039 / 0.038 | **15.15 / 14.64** |
| Being-H0-1B | 0.587 | 0.118 / 0.107 | 0.131 / 0.120 | 0.118 / 0.107 | 0.098 / 0.090 | 40.52 / 38.18 |
| Being-H0-8B | 0.615 | 0.082 / 0.075 | 0.098 / 0.092 | 0.081 / 0.074 | 0.064 / 0.057 | 31.16 / 29.98 |
| DreamZero-alike | 0.262 | 0.062 / 0.057 | 0.100 / 0.094 | 0.059 / 0.054 | 0.040 / 0.037 | 20.48 / 19.00 |
| **Donk-TI2VA** | **0.238** | **0.055 / 0.049** | **0.090 / 0.079** | **0.052 / 0.046** | **0.032 / 0.029** | 16.05 / 14.95 |

Donk-TI2VA 在手部 RMSE、ADE、FDE、DTW-S、DTW-L 上均取得最好成绩,只在腕部旋转 ROT 上略逊于 VITRA(15.15/14.64 对 16.05/14.95),相比 Being-H0 系列在所有指标上全面提升。

**条件消融(同一 OakInk2 基准,best-of-5/10)**：

| 变体 | Gaze | State | Hand RMSE | ADE K5/K10 | ROT K5/K10 |
|---|---|---|---|---|---|
| Donk-TI2VA(full) | 是 | 是 | 0.238 | 0.055 / 0.049 | 16.05 / 14.95 |
| Donk-TI2VA(wo Gaze) | 否 | 是 | 0.258 | 0.058 / 0.053 | 18.17 / 16.21 |
| Donk-TI2VA(base) | 否 | 否 | 0.262 | 0.062 / 0.057 | 20.48 / 19.00 |

State(锚点图控制)条件本身已经全面提升 base 模型;再加上 gaze(手部聚焦视频损失)进一步降低手部 RMSE、轨迹误差和旋转误差,gaze 的增益比 state 小但方向一致。

**视频质量(EgoDex/LOME 协议,1000 样本,17 帧,832×480 分辨率)**：

| 方法 | PSNR↑ | SSIM↑ | LPIPS↓ | CLIP-I↑ | CLIP-S↑ | tLPIPS↓ | FVD↓ |
|---|---|---|---|---|---|---|---|
| Wan2.2-TI2V-5B | 19.50 | 0.7855 | 0.3061 | 0.9119 | 0.2004 | 0.0429 | 81.87 |
| Wan2.1-I2V-14B | 19.47 | 0.7742 | 0.3252 | 0.9090 | 0.2053 | 0.0472 | **68.97** |
| Wan2.1-VACE-14B | 17.16 | 0.7220 | 0.4067 | 0.8551 | **0.2187** | **0.0197** | 103.85 |
| **Donk-TI2VA** | **19.84** | **0.7908** | **0.2992** | **0.9172** | 0.1982 | 0.0340 | 75.13 |

Donk-TI2VA 在同时输出动作轨迹的前提下,PSNR、SSIM、LPIPS、CLIP-I 四项均超过对比的纯视频基线(部分基线参数量更大,如 Wan2.1 系列为 14B,而 Donk 基于 5B 的 Wan2.2 TI2V);FVD 和 tLPIPS 上纯视频基线仍占优,说明动作流联合去噪未明显损害视频侧质量,但也未在全部视频指标上超越专用视频生成模型。

**T2VA 纯文本数据引擎(EgoDex 100 样本,VLM 0–5 分打分对齐度)**：

| 方法 | 输入 | FVD↓ | VLM judge↑ | CLIP-S↑ | tLPIPS↓ |
|---|---|---|---|---|---|
| Wan2.2-5B-I2V | text | 306.2 | 1.59 | 0.2508 | **0.0147** |
| **Donk-T2VA** | text | **191.1** | **2.37** | **0.2572** | 0.0215 |

纯文本条件下,Donk-T2VA 相比同规模的开源 Wan2.2-5B-I2V 基线大幅降低 FVD(306.2 到 191.1)、提升 VLM 判定的指令对齐分(1.59 到 2.37)与 CLIP-S,仅 tLPIPS(时间闪烁)略逊。定性上,T2VA 能为超出典型实验室采集分布的场景生成配对视频-动作(如野外动物交互、灭火等场景),该基线不输出动作故只比较视频指标。

## 四、局限性

论文正文未设独立的 Limitations 小节,但从方法与实验设计可归纳出以下局限:

- **T2VA 首帧锚点依赖辅助初始化器**:纯文本模式下没有观测到的初始手部状态,需要额外训练一个轻量文本条件初始化器来采样 $\tilde g_0$,该初始化器的误差会传导到后续锚点图和生成质量,论文未单独报告其准确率或对下游生成的敏感性分析。
- **动作表征限定于双手 MANO 参数**:整个方法围绕灵巧手 MANO 轨迹设计(锚点渲染、动作编解码等),未展示对夹爪等其他末端执行器或全身、移动机器人动作空间的适配。
- **训练与评测数据集较单一**:训练仅用 VITRA-1M 一个数据集,评测集中在 OakInk2(动作)和 EgoDex/LOME 协议(视频),缺少跨数据集、跨具身或真实机器人闭环部署的验证。
- **视频指标并非全面领先**:在 EgoDex 视频质量对比中,FVD、CLIP-S、tLPIPS 等指标上仍有专用视频生成基线(部分参数量更大)占优,说明联合去噪在追求动作精度的同时,视频生成质量的最优性有所让渡。
- **计算成本高**:64 张 96GB 显存的 Hopper GPU 训练,门槛较高,论文未报告训练总时长或更小算力下的可复现性。

## 五、评价与展望

Donk 的核心贡献是把 World Action Model 从"仅支持观测条件策略"的单一用途,重新表述为文本/图像条件下视频-动作联合分布的两种取值情形,从而让同一套预训练-微调好的去噪骨干既能当策略(TI2VA)又能当数据引擎(T2VA)。这一"一个模型两种用途"的设计思路,相比把视频生成和动作预测分成两个独立系统(如级联的视频/动作模块,或先生成视频再离线提取动作的两阶段流水线),规避了中间表示脆弱、时间错位、误差累积等问题,是对同期统一视频-动作类工作(如 Unified World Models、Unified Video Action Model 等论文相关工作中列举的多篇)的一个自然延伸,其增量贡献在于显式的锚点几何控制信号与视频保留注意力掩码这两个工程设计,用来在联合去噪时保护预训练视频先验不被动作分支破坏。

方法设计上的亮点在于"视频保留注意力掩码"与"门控首帧锚点注入"两个机制的组合:前者用非对称注意力保证视频生成路径几乎不受动作、锚点 token 干扰,后者用零初始化门控保证从预训练行为平滑过渡到带几何控制的联合生成,二者都体现了"尽量不破坏预训练视频先验,同时用最小侵入方式注入新信号"的稳健工程哲学,这对于依赖大规模视频基础模型先验的具身智能工作有较高的参考价值。

局限与开放问题方面:第一,T2VA 作为"数据引擎"的价值目前只以视频质量和 VLM 打分间接验证,尚未有下游实验证明用 T2VA 生成的数据能否实际提升独立策略模型的训练效果,即数据引擎的下游有效性闭环未被验证,这是该方向后续最值得补充的实验;第二,文本条件初始化器本质上是给"缺失锚点"打补丁的辅助模块,若能将首帧几何的不确定性显式建模进联合分布而非独立训练一个初始化器,理论上会更优雅;第三,ROT 指标上不敌专用动作预测模型 VITRA,说明联合视频建模在角度精度上可能与位置、手指精度存在一定权衡,值得探究是否有更好的旋转表征或损失设计;第四,评测目前局限于离线基准(OakInk2、EgoDex),尚缺乏真实机器人闭环部署或与真人遥操作对比的结果,这也是几乎所有同类 WAM 工作共同面临的验证鸿沟。整体而言,Donk 提供了一个把策略学习和数据合成统一到同一生成先验下的可行范式,其锚点控制和注意力掩码设计具有较好的可迁移性,但作为数据引擎的实际价值仍需下游任务验证。
