## 一句话总结

Moto 用 VQ-VAE 风格的 Latent Motion Tokenizer 把连续视频帧间的运动无监督地压缩成离散 token 序列，再用 GPT 式自回归模型（Moto-GPT）在海量无动作标注的视频上预训练运动先验，最后通过插入 action query token 的 co-fine-tuning 策略把运动先验迁移为真实机器人动作；仅 98M 参数、纯静态 RGB 输入即在 CALVIN(ABC→D) 上以平均完成 3.10/5 个任务超过用了额外 gripper 视角与本体感知的 GR-1（3.06），SIMPLER 总体成功率 0.614 优于全部对比基线。

## 一、问题与动机

LLM 依靠海量文本上的 next-token 预测取得成功，而机器人学习长期受限于动作标注数据的高昂成本；与此同时互联网和机器人视频数据极为丰富。核心问题是：如何为视频设计一种适合自回归预训练、又能有效迁移到操作任务的表示？此前视频预训练工作（如 R3M、GR-1 等）多聚焦帧级静态视觉细节的重建，作者认为更有效的做法是聚焦"运动"这一与低层机器人动作紧密关联、且与具体机器人硬件无关（hardware-agnostic）的信息，因此提出把帧间视觉运动编码为离散的 latent motion token，作为连接视频与机器人动作的桥接"语言"。

## 二、核心方法

方法分三阶段：(1) 无监督训练 Latent Motion Tokenizer；(2) 在视频上做运动 token 自回归预训练 Moto-GPT；(3) 在动作标注数据上 co-fine-tune，转化为机器人策略。

**1. Latent Motion Tokenizer（VQ-VAE 架构）**：编码器 M-Former 是一个多层 transformer，输入为冻结 ViT 提取的当前帧 $o_t$ 与前一帧 $o_{t-1}$ 的 patch 特征，并拼接 8 个可学习 query embedding，query 间做自注意力，输出特征经 VQ codebook（词表大小 128）量化为离散 latent motion token（每帧对 8 个 token）。解码器为 ViT Decoder：把量化后 motion token 拼接经 MLP 压缩为 1 个条件 embedding，加到 $o_{t-1}$ 每个 patch embedding 上重建出 $o_t$——该条件 embedding 构成编解码器间的信息瓶颈，迫使 token 只编码"变化"而非静态外观。训练目标为标准 VQ-VAE 损失（重建 MSE + 向量量化损失 + commitment 损失）。

**2. Motion Token 自回归预训练（Moto-GPT）**：给定视频片段 $[o_0, o_1, \dots, o_T]$，把每对相邻帧的 latent motion token chunk 按时间顺序拼成序列，前置冻结 T5 编码的语言指令特征 $l$ 与冻结 ViT 提取的初始帧视觉特征 $v$ 作为 prompt，用 GPT 式 transformer 做 next-motion-token 预测：

$$\mathcal{L}_{motion} = -\sum_{i=1}^{M} \log P(m_i \mid l, v, m_{<i}; \Theta)$$

用大白话说：让模型看一句指令和第一帧画面，像接龙一样把后续每一步"发生了什么运动"的离散编码依次猜出来，$M=K\times T$（$K{=}8$ 为每帧运动 token 数，$T$ 为视频长度）。

**3. Co-fine-tuning 迁移到机器人动作**：在每个时间步的 latent motion token chunk 后插入 $N$ 个可学习 action query token（$N$ 为该帧对间实际发生的机器人动作数），用 MLP 动作头把每个 query token 的隐状态映射到真实动作空间；位移/旋转分量用 Smooth-L1 损失，夹爪开合用二元交叉熵损失：

$$\mathcal{L}_{action} = \mathcal{L}(\Delta x) + \mathcal{L}(\Delta \theta) + \mathcal{L}(\Delta grip)$$

微调阶段保留运动 token 预测目标以维持已学到的先验，总损失为两者之和 $\mathcal{L}_{ft} = \mathcal{L}_{motion} + \mathcal{L}_{action}$。关键技巧是训练时以 50% 概率随机屏蔽 action query token 对 latent motion token 的注意力：既促使知识经直接注意力路径迁移、降低对"真值运动 token 条件"的依赖，又使推理时可跳过生成运动 token、直接用 padding 占位查询动作，提升推理效率。

## 三、关键结果

**运动 token 的语义可解释性**（CALVIN 34 任务视频分类）：仅用"初始帧 + 7 个后续 latent motion token chunk"作特征，分类准确率达 0.797，接近"初始帧 + 7 个后续完整帧"的 0.828，远高于"仅初始帧"（0.292）或"初始帧重复 8 次"（0.283），说明 8 个离散 token 已能承载与 196 维原始 patch 特征相当的语义信息。

**SIMPLER**（Google Everyday Robot；pick coke can / move near / open-close drawer；109k OXE 视频预训练 + 73k RT-1 轨迹微调）：

| 方法 | Pick Coke Can Avg | Move Near | Drawer Avg | Overall |
|---|---|---|---|---|
| RT-1-X | 0.567 | 0.317 | 0.597 | 0.534 |
| RT-2-X（PaLI-X 55B） | 0.787 | 0.779 | 0.250 | 0.607 |
| OpenVLA（Prismatic-7B，fine-tuned） | 0.363 | 0.542 | 0.231 | 0.349 |
| Moto w/o Motion Token | 0.503 | 0.554 | 0.398 | 0.480 |
| **Moto**（98M 参数） | 0.740 | 0.604 | 0.431 | **0.614** |

Moto 仅 98M 参数、预训练全程无动作标签，总体成功率超过全部对比基线，与用了 55B/7B 参数的 RT-2-X、OpenVLA 相当甚至更优。

**CALVIN(ABC→D)**，长时序 5 连击任务，Moto 仅用静态 RGB（无 gripper 视角、无本体感知）：

| 方法 | 观测空间 | 1 | 2 | 3 | 4 | 5 | Avg Len |
|---|---|---|---|---|---|---|---|
| GR-1 | Static+Gripper RGB+Proprio | 0.854 | 0.712 | 0.596 | 0.497 | **0.401** | 3.06 |
| Moto w/o Motion Token | Static RGB | 0.779 | 0.555 | 0.380 | 0.256 | 0.167 | 2.14 |
| **Moto** | Static RGB | **0.897** | **0.729** | **0.601** | 0.484 | 0.386 | **3.10** |

在观测信息更少的情况下，Moto 平均完成任务数（3.10）超过使用更多观测的 GR-1（3.06）。

**数据效率**：CALVIN 微调仅用 1% 有标签数据时，Moto 达 52.5% 成功率，而从零训练的 Moto w/o Motion Token 为 0%，说明预训练学到的运动先验大幅降低了对动作标注数据量的需求。

**真实机器人**（FANUC LR Mate 200iD；pick-place banana / close laptop / disassembly；90 条遥操作演示微调）：Moto 把平均成功率从 23.33%（w/o Motion Token）提升到 60%；在 Visual Distractor 和 Novel Object 两种泛化场景下分别再提升约 20% 和 30% 的平均表现。

**人类视频跨具身迁移**：加入过滤后的 105k Something-Something V2 人类视频与 109k OXE 视频联合预训练（Moto OXE+SSV2），在 SIMPLER Move Near 任务上成功率从 Moto w/o Motion Token 的 55.4%、Moto（仅 OXE）的 60.4%，提升到 78.3%（+17.9pt），显示 latent motion token 可作为统一"语言"把人手运动语义迁移到机器人动作。此外论文用预训练 Moto-GPT 的输出似然区分 CALVIN 中成功/失败/随机三类轨迹，似然曲线可明显分开三者，提示可作为轨迹打分/奖励信号的初步探索。消融显示去掉运动 token 预测损失（Moto-IML）或去掉运动 token 输入（Moto-DM）均比完整 Moto 差，但仍优于从零训练基线，说明预训练本身与微调阶段保留运动 token 序列都各自贡献增益。

## 四、评价与展望

**优点**：(1) Latent Motion Tokenizer 把"运动"与静态外观解耦，通过重建瓶颈迫使 token 只编码帧间变化，天然具备跨具身（人手到机械臂）迁移潜力，这是相对 R3M、GR-1 等聚焦帧级像素预测方法的一个清晰差异化设计；(2) 整个预训练阶段不需要任何动作标签，可直接扩展到 Internet-scale 人类视频，数据效率实验（1% 标注数据达 52.5% vs 0%）有力支持了这一动机；(3) co-fine-tuning 通过 50% 随机 mask 注意力的技巧，同时兼顾知识迁移与推理效率，设计较为巧妙。

**局限**：(1) 8 个 token、词表 128 的离散化粒度能否支撑更精细、长程或高自由度（如双臂、灵巧手）的操作任务，论文未做验证，CALVIN/SIMPLER 任务集本身以单臂平面抓取、开关抽屉为主，复杂度有限；(2) 人类视频实验只是"preliminary study"（105k 经过滤的 SSV2 视频），尚未验证在更大规模、更多样人类视频上的可扩展性，论文结论部分也承认这一点；(3) 高层语义理解依赖冻结的 T5/ViT 特征，未联合微调视觉-语言 backbone，可能限制对新场景、新指令的泛化；(4) 真实机器人实验任务数量（3 个）和平台（单一 FANUC 机械臂）较少，尚未展示在更复杂真实环境或长时序真实任务上的表现。

**与相关工作的关系**：Moto 与 Genie（2D 游戏场景无监督 latent action）、LAPA（单步未来 latent action）、IGOR（latent action 作为目标表征）等思路同源，但 Moto 强调自回归预测一整段轨迹的运动 token 序列而非单步预测，建模更贴近视频的时序连续性；相较于 GR-1 等依赖像素级视频预测做 world model 的路线，Moto 用离散化的运动语言替代像素预测，在计算和表示上更紧凑（可解释性实验显示仅 8 个 token 即可逼近全帧特征的语义分类准确率）。

**开放问题**：离散运动词表的粒度/规模应如何随任务复杂度和自由度增加而扩展；运动先验能否与显式的视频生成式世界模型结合以获得更强的规划能力；论文初步展示的 log-likelihood 轨迹打分思路若结合在线强化学习或轨迹重排序，是否能进一步提升策略性能，仍待完整实验验证。
