## 一句话总结

ViPRA 用一个预训练-微调框架把无动作标签的人类/机器人视频转化为机器人策略:先从人类和机器人视频中联合学习运动中心的离散潜动作(光流一致性 + LPIPS + 像素重建监督),再用这些潜动作扩展一个大型视频语言模型 LWM-Chat-1M 做"未来观测 + 潜动作序列"联合预训练,最后仅用 100~200 条真实演示微调一个 flow matching 解码器把潜动作块映射为连续动作块;在 SIMPLER 上离散/连续两种设定均取得当类最佳(ViPRA-AR 平均成功率 69.8% 比最强基线 LAPA 高 16.7 个百分点,ViPRA-FM 平均成功率 62.5% 比最强连续基线 UniVLA 高约 19.8 个百分点),真实世界三任务平均成功率 54.1%(π0 为 40.1%,从零训练基线 Scratch-FM 为 23.8%),并借助动作分块把闭环控制频率做到最高 22 Hz。

## 一、问题与动机

一个通用机器人智能体需要同时具备精确的低层控制能力和环境无关的高层智能。视频生成模型天然适合提供后者:它们在互联网规模的视频上训练,已经展现出对物体动态与精细物理交互的强先验,但绝大多数视频(YouTube、Something-Something、Ego4D 等)都没有动作标签,难以直接用于机器人模仿学习。这就引出核心问题:**能否把一个视频预测模型直接转化为机器人控制策略?**

现有相关路线各有局限:

- **VLA 模型**(OpenVLA、π0 等)在预训练视觉语言模型上加动作头,用有标签的机器人示范微调,但训练数据仍依赖成本高昂的大规模动作标注(如 Open X-Embodiment 约 10000 小时),难以进一步扩展。
- **潜动作预训练方法**(LAPA、UniVLA、Moto 等)从无标签视频中学习离散/连续的"潜动作"token,规避了动作标注需求,但通常把预训练当作纯粹的(自回归)策略学习,不显式建模视频预测,且潜动作往往是时间上粗粒度、任务中心的表示,难以捕捉高频控制所需的精细运动动态。
- **视频规划方法**(UniPI、SuSIE 等)显式生成未来视频帧作为规划蓝图,再靠逆动力学模型或姿态跟踪转成动作,但多步视频生成推理延迟高,难以支撑高频闭环控制,且这类方法的评测大多局限在仿真或较受限的真实任务上。

ViPRA 的思路是把"预测什么会发生"(未来视觉观测)和"如何发生"(运动中心的潜动作)统一到同一个多模态视频语言模型里联合预训练,并用光流一致性约束潜动作学习物理上合理的运动语义,再用一个数据高效的 flow matching 解码器把这套(在动作无标签视频上学到的)高层先验对齐到机器人具体的连续动作空间,试图同时兼顾数据可扩展性(无需动作标注)、跨具身泛化能力,以及高频、平滑的连续控制。

## 二、核心方法

ViPRA 分三个阶段构建(如 Fig. 2 所示):(1)从无动作视频中学习离散潜动作;(2)用潜动作扩展一个多模态视频语言模型做联合预训练;(3)用少量有标签演示微调 flow matching 解码器实现连续高频控制。

### 2.1 从无动作视频中学习运动中心的离散潜动作

给定长度为 $L+1$ 的观测序列 $o_{t:t+L}$(可来自人类视频或机器人视频,均无动作标签),目标是学习潜动作序列 $z_{t:t+L-1}$,其中每步 $z_k = [z_k^1, z_k^2, \ldots, z_k^{N_{\text{latent}}}]$ 由 $N_{\text{latent}}=8$ 个离散 token 组成,每个 token 从一个共享码本 $\mathcal{C}$(码本大小 $\lvert\mathcal{C}\rvert=8$)中量化得到。

逆动力学编码器 $I_\beta(z_{t:t+L-1} \mid o_{t:t+L})$ 以 DINOv2 初始化的时空 Transformer 为骨干,**非因果地**看到整段视频(过去+未来)来预测每一步潜动作——用大白话说:让编码器"看完整段视频再回头总结每一步发生了什么运动",这样能靠前后文消除歧义(比如区分"拿起"和"放下"),迫使潜动作 token 只需编码解释局部状态转移所必需的最小信息。量化采用 NSVQ(噪声替代向量量化),用随机噪声注入代替传统 VQ-VAE 的直通估计器(straight-through estimator),无需额外码本损失即可稳定训练、缓解码本坍塌。

前向解码器 $F_\alpha(\hat o_{t+k+1} \mid o_{t:t+k}, z_{t:t+k})$ 因果地重建下一帧,联合训练使潜动作包含足够信息解释场景动态。总损失为像素级 L1 重建、LPIPS 感知损失,再加上一个光流一致性损失:

$$\mathcal{L}_{\text{flow}} = \frac{1}{L}\sum_{k=t+1}^{t+L}\lVert \mathrm{OF}(\hat o_k,\hat o_{k-1}) - \mathrm{OF}(o_k,o_{k-1})\rVert_1 + \frac{1}{L}\sum_{k=t}^{t+L-1}\lVert \mathrm{OF}(\hat o_k,\hat o_{k+1}) - \mathrm{OF}(o_k,o_{k+1})\rVert_1,$$

其中 $\mathrm{OF}(a,b)$ 是 RAFT 计算的帧间光流。用大白话说:不仅要求重建的帧像真实帧,还要求重建帧之间"动起来的方式"(光流场)要和真实视频的运动模式一致,这道约束在训练若干步(warm-up 之后)才启用,避免早期重建质量差时引入不稳定梯度。总损失:

$$\mathcal{L}_{\text{latent}} = \mathcal{L}_{\text{rec}} + \lambda_{\text{LPIPS}}\mathcal{L}_{\text{LPIPS}} + \mathbb{I}(\text{step} > \alpha_{\text{flow}})\lambda_{\text{flow}}\mathcal{L}_{\text{flow}}.$$

消融(见下文 Table 4)证实:去掉光流一致性损失会让码本困惑度从 5.01 升到 5.63、熵从 1.59 升到 1.74、下游动作探针(冻结骨干)的动作 MSE 从 0.84 升到 0.92,说明光流监督确实促使潜空间编码运动而非背景外观。

### 2.2 用潜动作做多模态视频语言模型预训练

学到离散潜动作后,ViPRA 在公开的指令微调视频语言模型 **LWM-Chat-1M** 基础上扩展两个轻量模块:潜动作嵌入头 $E_\phi$(把每个码本 token 映射进模型 token 空间,维度 4096)和潜动作 token 解码器 $H_\psi$(单层 MLP,隐藏维 2048),用于自回归预测下一个潜动作 token。

给定最近两帧观测 $(o_{t-1}, o_t)$ 和任务描述 $c$,模型联合预测 $H=14$ 步之后的未来帧 $o_{t+H}$(通过 VQ-VAE token 分类损失)以及通向该未来帧的潜动作序列 $z_{t:t+H-1}$(通过 token 级交叉熵),采用 teacher forcing:

$$\mathcal{L}_{\text{pretrain}} = \underbrace{\sum_{i=1}^{N_{\text{tokens}}}\mathrm{CE}(\hat x_{t+H}^i, x_{t+H}^i)}_{\mathcal{L}_{\text{img}}} + \underbrace{\sum_{k=t}^{t+H-1}\sum_{i=1}^{N_{\text{latent}}}\mathrm{CE}(\hat z_k^i, z_k^i)}_{\mathcal{L}_{\text{act}}}.$$

用大白话说:模型一边学"这段视频接下来长什么样"(未来帧的视觉 token),一边学"要经历怎样一串运动 token 才能到达那个未来"——多步预测视野(而不是单步)迫使模型学到有意义、可区分的场景变化,为下游动作推断提供更稳的条件信息。该阶段在 198k 条 Something-Something v2 人类视频 + OpenX 子集(87k Fractal、25.4k BridgeV2、85.6k Kuka 机器人视频)上训练,全程无需任何动作标签。

### 2.3 Flow Matching 解码器:连续高频控制的微调

预训练后的视频语言模型提供了强的语义化物理先验,但缺乏低层控制所需的精度。ViPRA 引入两个动作专用组件:噪声动作编码器 $E_\gamma$ 把带噪声的时间插值动作块 $u_s$ 嵌入 token 空间,flow 解码器 $H_\eta$ 预测把噪声动作输运到真实动作块的速度场。训练时对目标动作块 $a_{t:t+H-1}\in\mathbb{R}^{H\times D}$ 采样噪声 $u_0 \sim \mathcal{N}(0,I)$,插值系数 $s\sim\mathrm{Beta}(1.5,1.0)$:

$$u_s = s\cdot u_0 + (1-s)\cdot a_{t:t+H-1},$$

再让模型基于图像 token、语言 prompt 和噪声动作预测速度场 $\hat g = H_\eta(G_\theta(c, x_{t-1}, x_t, f_s))$,监督目标为:

$$\mathcal{L}_{\text{FM}} = \lVert a_{t:t+H-1} - u_0 - (1-s)\cdot \hat g\rVert_2^2.$$

推理时用前向 Euler 积分从 $s=0$ 迭代到 $s=1$(共 10 步)求解连续动作块。用大白话说:这一层相当于给冻结的高层"想象力"模型接上一个专用的"翻译器",只用几百条真机演示就能把抽象潜动作精确翻译成机器人真实的关节/末端指令,而不需要重新训练整个视频模型。全部训练/微调数据集、码本规模等细节见 Table 5-7:潜动作学习阶段 8×H100 训练 168 小时(24 万步,批大小 128);多模态预训练阶段 8×H100 训练 144 小时(5 万步,批大小 512);flow matching 微调仅需约 1.2 万步(SIMPLER 设置),仅依赖 100-200 条 teleoperated 演示。

|---|---|
| UniPI | 0.00 |
| OpenVLA | 0.54 |

| UniVLA | 0.92 |

ViPRA-FM(79%)明显超过 OpenVLA 和 π0-FAST,但落后于专门针对 LIBERO 优化的 UniVLA(92%)/UVA(90%)。论文将差距归因于 delta 末端执行器动作空间上的漂移(delta-EEF drift):细小预测偏差随时间累积,导致对直径敏感的精密抓取(如圆柱罐)失准,而 ViPRA 在没有本体感知/腕部相机输入的情况下仍超过同等感知条件的 OpenVLA。

**真实世界(双臂 Franka Emika Panda,单前置 ZED 单目相机,GELLO 遥操作采集,三个单臂任务:Cover-Object / Pick-Place / Stack-Cups)**

ViPRA-FM 平均全任务成功率 **54.1%**,显著优于 π0(**40.1%**)和从零训练基线 Scratch-FM(**23.8%**)。由于离散策略(bin 化动作预测)在真机上出现不稳定尖峰、偶发触发 Franka 安全急停,离散基线被排除在真实世界评测之外。论文观察到 ViPRA 具备"失败后重试"的鲁棒行为(如 Cover-Object 中反复尝试抓取滑落的布料),归因于时序预训练带来的长时程动态与可恢复性建模。

**双臂任务(Appendix H,置于同一策略下联合控制两臂)**:Place-in-Bowl 18 次试验成功 10 次(失败主因是自制 3D 打印夹爪跨度/顺应性不足,而非双臂协调本身——所有成功抓取均无碰撞地完成放置);Mix-with-Whisk 12 次试验成功 8 次。

**消融研究(SIMPLER 4 任务平均成功率,拆解"未来状态预测"与"潜动作分块"的贡献)**

| 变体 | 预训练目标 | 微调目标 | AR 成功率 | FM 成功率 |
|---|---|---|---|---|
| LAPA(纯潜动作基线) | 1-step L | 1-step A | 53.1 | – |
| ViPRA(完整) | FS + H-step L | H-step A | **69.8** | **62.5** |
| −AC(去动作分块) | FS + 1-step L | 1-step A | 59.2 | 44.8 |
| −SP2(去预训练状态预测) | H-step L only | H-step A | 59.4 | 53.2 |
| −LA(纯状态、无潜动作) | FS only | H-step A | 60.7 | – |
| +SP3(把状态预测挪到微调阶段) | H-step L | FS + H-step A | 53.1 | 31.3 |

关键发现:即便只用单步潜动作,加入未来状态预测也能把 LAPA 的 53.1% 提升到 59.2%;去掉预训练阶段的状态预测明显掉点(AR 69.8→59.4,FM 62.5→53.2);仅状态预测而无潜动作不够(60.7% < 69.8%);而把状态预测**挪到微调阶段**(即让动作预测与视频预测在自回归结构中耦合)会显著损害性能,FM 上从 62.5% 骤降到 31.3%,原因是自回归耦合带来的复合误差在分布外状态上不可逆地累积——这是本文论证"预训练期联合预测、微调期解耦"这一设计决策的关键证据。

**其它消融**:光流一致性损失(Table 4)降低码本困惑度/熵、降低下游动作探针 MSE(0.92→0.84);数据构成消融(Table 3,LIBERO-10)显示人机协同训练效果最好(成功率 0.79、动作 MSE 0.84),仅机器人视频次之(0.72 / 0.91),仅人类视频最弱但仍有正迁移(0.69 / 0.99),说明人类视频贡献运动多样性、机器人视频贡献运动学对齐,二者互补。

**推理效率(Table 9)**:ViPRA(7B)单次前向产出 14 步动作块耗时 510 ms(启用 KV 缓存,语言/图像注意力状态跨 flow matching 积分步复用,仅重算动作 token),对应约 1.95 Hz 的原始推理频率;通过动作分块摊销,闭环控制频率可达 **22 Hz**,据作者所知这一高频水平在 7B 参数量级模型中仅有一个同期工作(π0.5/Kim et al., 2025)与之相当。相比之下 LAPA(7B,单步动作)220 ms/步、OpenVLA(7B,单步)190 ms/步、π0(3.3B,16 步)90 ms、UniPI(16 步)高达 24000 ms。

## 四、局限性

论文第 6 节明确列出以下方向,并结合实验现象补充说明:

- **具身与灵巧度**:方法在 WidowX、Franka(含双臂)等具身间验证了泛化性,但尚未涉及多指灵巧手或人形机器人等需要更精细接触动力学和多臂协同的场景;论文认为可能需要触觉/力反馈或具身特定适配器。
- **数据范围**:预训练虽已用上大规模无动作视频,但多样性仍有限,论文提出可扩展至更丰富的第一人称数据集(人手交互、工具使用、导航片段),并加入腕部相机、本体感知、触觉、深度等额外模态。
- **扩展规律未知**:被动视频预训练如何随数据规模扩展、在何处出现收益递减,以及无标签视频与有标签演示的最优配比,论文明确列为开放问题,未给出定量扩展曲线。
- **delta 动作空间漂移**:在 LIBERO-10 这类长时程任务上,ViPRA 因缺乏本体感知/腕部相机的"再锚定"信号,在需要精密直径对齐的抓取上出现误差累积,落后于利用额外传感信号的 π0、以及专门优化过 LIBERO 的 UniVLA/UVA。
- **双臂抓取瓶颈来自硬件而非算法**:Appendix H 中双臂任务的失败主要归因于自制夹爪的跨度/顺应性限制,而非策略的空间推理或双臂协调能力,提示当前评测规模下算法与硬件因素尚未完全解耦。
- **推理延迟仍由骨干主导**:单次 14 步动作块前向耗时 510 ms,虽靠动作分块摊销到 22 Hz,但原始单次前向频率仅约 2 Hz,对更高频、更精细的接触密集型任务是否足够仍待验证。

## 五、评价与展望

ViPRA 的方法论贡献在于清晰地把"视频预测"和"潜动作预训练"两条此前分立的路线统一到同一个多模态视频语言模型里联合训练,并用消融(尤其是 +SP3 变体在 FM 设置下从 62.5% 骤降到 31.3%)有力地证明了一个此前未被充分讨论的设计原则:**未来状态预测应当留在预训练阶段与潜动作联合建模,而不应该在微调阶段与真实动作预测耦合**,否则自回归结构会把视频预测的复合误差带入分布外状态、拖垮动作精度。这一发现对后续同类"视频+动作"联合训练工作(如 UVA、HiF-VLA 一类混合预测式 VLA)具有一般性的参考价值。

与同期公开工作相比,ViPRA 相对 LAPA/UniVLA 这类纯潜动作预训练方法的增量在于显式引入了带光流一致性约束的多步视频预测目标(而非把预训练简化为纯策略学习),相对 UniPI/SuSIE 这类显式视频规划方法的增量在于用一次 flow matching 前向 + 动作分块取代了昂贵的多步视频采样,从而把控制频率从 UniPI 的亚 Hz 级别(24000 ms/16 步)提升到 22 Hz。不过在需要强精细抓取对齐的 LIBERO-10 上,ViPRA(79%)仍不及专门针对该基准调优、且额外利用本体感知/腕部相机的 UniVLA(92%)与 UVA(90%),说明"利用无动作视频学到的隐式几何/运动先验"目前还难以完全替代显式的本体感知反馈,这与同期强调显式深度/状态输入的工作(如 π0 系列)形成有趣对照,也是该方向一个尚待厘清的开放问题:潜动作蒸馏得到的隐式表征,到底能在多大程度上弥补显式传感信号的缺失。

开放问题与可能的改进方向:第一,论文尚未给出预训练数据规模-下游性能的定量扩展曲线,难以判断当前 198k 人类视频 + 数十万机器人视频的规模是否已接近收益递减点,系统性的数据缩放实验会是重要的后续工作;第二,codebook 大小仅为 8(8 个 token/步)是经验设定,未讨论该超参数对下游任务复杂度的敏感性,更大或自适应的潜动作词表是否能进一步缩小与显式状态方法的差距值得探索;第三,当前流程完全依赖单目 RGB 观测,如何在不牺牲"从无标注视频学习"这一核心卖点的前提下融合深度/多视角信息,是弥合与 LIBERO-10 上专用方法差距的可能路径;第四,真实世界评测(3 个单臂任务 + 2 个双臂任务)规模仍偏小,长时程、杂乱场景、多物体交互等更具挑战性的真实世界泛化性尚未系统验证,双臂结果中硬件(夹爪)与算法瓶颈的解耦也需要更完善的实验设计。
