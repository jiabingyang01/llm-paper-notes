## 一句话总结

UMA 把 3D 物体运动轨迹当作跨真实机器人数据、人类视频、仿真数据的共享接口,用 masked diffusion transformer 将物体运动与机器人动作建模为一对"共同演化"的变量,同一套预训练权重仅靠改变 mask 模式即可切换视觉运动控制、动力学建模、few-shot 任务适应三种推理模式,在真实桌面操作上 zero-shot 视觉运动控制比最强专用基线高 20-25 个百分点,动力学预测 MSE 从 0.054 降到 0.042。

## 一、问题与动机

现有方法把物体运动(3D 关键点轨迹)固定成单一角色:运动条件化策略(如 [7,8])只能把运动当条件预测动作,用不上无动作标签的视频;运动预测模型(如 [9,10])只能从视频预测运动轨迹,拿到动作标签后也没有自然的方式利用它们做下游控制。结果是没有一个方法能同时"从无动作视频学运动"又"在有动作标签时用上动作监督"。论文的目标是设计一种表示与建模方式,让物体运动同时可以作为条件输入和预测目标,从而让人类视频、仿真数据、真实机器人示范三类异构数据在同一目标函数下联合预训练,并支持部署时灵活切换任务。

## 二、核心方法

**统一运动-动作建模。** 记当前观测 $o_t$、物体运动序列 $x_{t:t+H}$、动作序列 $a_{t:t+H}$,UMA 建模条件分布

$$p_\theta\big(x_{\mathcal P_x}, a_{\mathcal P_a} \mid o_t, c, x_{\mathcal G_x}, a_{\mathcal G_a}\big),$$

其中 $\mathcal G_x,\mathcal G_a$ 是给定(条件)token 的下标集合,$\mathcal P_x,\mathcal P_a$ 是要预测的目标 token 下标集合,$c$ 是"任务潜变量"(task latent),抽象地概括交互意图、与具体场景和动作空间无关。大白话说:模型看到的是一串带 mask 的"运动+动作"token 序列,训练时随机遮住一部分,推理时通过决定哪些 token 被遮住来切换任务——遮动作就是控制,遮运动就是动力学预测。

**任务潜变量与参考运动。** 训练时用一段部分观测到的参考运动 $x_c = \{\tilde x_{k,t} : (k,t)\in\mathcal G_c\}$(关键点、采样数、时间间隔都可以不规则)来指定任务,任务编码器

$$c = h_\psi(o_0, x_c)$$

把初始观测(PointNet++ 点云特征 + SigLIP2 图像特征)和稀疏参考运动编码成固定数量的任务 token,要求对参考运动的关键点选取和绝对场景位置保持不变性(invariant)。这相当于把 hindsight experience replay 从"单状态目标"推广到"运动情境"目标。

**Motion-Action Diffusion Transformer。** 观测、物体运动、机器人动作都被 token 化进同一个 transformer 骨干,用 flow matching 联合去噪。注意力沿三个方向结构化分解(继承自 COIL [12]):spatial(同一时间步内运动-动作耦合)、temporal(同一关键点跨时间步)、context(目标 token 关注观测/任务 token)。为了让"干净的条件 token"和"待去噪的目标 token"共用一次 transformer 前向却不互相干扰,论文提出 **Masked DiT block**:每层对目标 token 和给定 token 分别用两套独立的 adaLN 调制参数

$$\hat z_i = \begin{cases}\gamma^{\text{tgt}} \odot \mathrm{LN}(z_i) + \beta^{\text{tgt}}, & i \in \mathcal P\ (\text{目标}),\\ \gamma^{\text{gvn}} \odot \mathrm{LN}(z_i) + \beta^{\text{gvn}}, & i \in \mathcal G\ (\text{给定}).\end{cases}$$

大白话说:同一层网络内部,"该去噪的部分"和"该原样保留的部分"各走各的缩放/平移参数,但仍共享同一次自注意力计算,这样既不破坏条件信息,又保留了目标-条件间的信息流动。

**异构监督与整体目标。** 每个样本只对"有标签的模态"计算损失:真机示范同时监督运动重建 $\mathcal L_x$ 和动作重建 $\mathcal L_a$;无动作标签的人类视频/跨embodiment 数据只监督 $\mathcal L_x$。为防止任务潜变量 $c$ 偷偷记住具体场景细节(如绝对位姿),用 SimCLR 风格的对比损失 $\mathcal L_c$,对同一 $(o_0,x_c)$ 做关键点子采样、时间步子采样、随机 SE(3) 变换构造正样本,不同任务/不同运动意图作负样本。总目标

$$\mathcal L = \lambda_x \mathcal L_x + \lambda_a \mathcal L_a + \lambda_c \mathcal L_c \qquad(\lambda_x{=}1,\ \lambda_a{=}2,\ \lambda_c{=}0.001).$$

**三种推理模式(权重全程冻结,除 soft prompt tuning)。**
1. *运动条件化视觉运动控制*:给定目标参考运动 $x^\star$,$c=h_\psi(o_0,x^\star)$,模型逐步预测动作 chunk、执行、从新观测重规划,闭环修正实际运动与目标运动的偏差。
2. *运动动力学建模*:给定候选动作序列,正向预测物体运动 rollout,用于 MPC 打分候选动作。
3. *soft prompt tuning 的任务推断*:仅把 $c$ 当自由变量优化(其余参数冻结),用少量示范(动作监督或纯运动监督均可)最小化

$$c^\star = \arg\min_c \sum_{\xi\in\mathcal D} \big[\lambda_x' \mathcal L_x(\xi;c) + \lambda_a' \mathcal L_a(\xi;c)\big] + \lambda_c' \lVert c-c_0\rVert_2^2,$$

对纯视频示范设 $\lambda_a'=0$,即整个新任务适应过程可以"无机器人数据"、只靠人类视频完成。

**训练数据与规模。** 预训练混合约 1000 万步人类视频(HOI4D + Xperience)、1000 万步真机数据(DROID)、1000 万步仿真数据(沿用 COIL 的仿真生成管线,资产来自 YCB/PartNet-Mobility/MuJoCo Scanned Objects),三者等概率采样。真实视频的 3D 关键点监督通过四阶段管线获取:MegaSaM 估相机位姿/深度 → UniDepth 做尺度对齐 → GPT-5-nano + SAM 3 做目标分割与关键点采样 → TAPIP3D 做持久 3D 点跟踪,全程不需要真值深度、相机内参或动作标签。骨干为 6 层、隐藏维 768、12 头的 transformer,预测视野 $H=16$,单样本关键点数 $K\in[1,16]$;AdamW,学习率 $1\times10^{-4}$,batch size 16,8 张 A100 训练 16 个 epoch。

## 三、关键结果

真实世界评测在 DROID 同款环境(Franka FR3 + Robotiq 2F-85,ZED 相机,15Hz)上进行,三个任务(Insertion 精细 6-DoF 插入、Sweeping 工具使用扫地、Folding 可变形牛仔裤折叠),每任务 20 次试验,预训练数据与测试环境无重叠。

**Zero-shot 视觉运动控制**:对比同数据训练的 COIL(3D 运动条件化 flow-matching 策略)和 UVA(像素空间联合视频-动作 masked 模型),UMA 在每个任务上都比最强基线高 20-25 个百分点。

**动力学预测(MSE,平方米,越低越好)**:

| 方法 | MSE |
|---|---|
| PointWorld(专用动力学模型基线) | 0.054 |
| UMA w/o Sim | 0.208 |
| UMA w/o Human | 0.044 |
| **UMA(完整)** | **0.042** |

去掉仿真数据使 MSE 上升近 5 倍,说明仿真中密集的运动-动作配对是耦合建模的主要来源;去掉人类视频对 MSE 几乎无影响(0.042→0.044),但会明显拖累 Folding 的零样本成功率(75%→65%),说明人类视频主要贡献任务多样性而非动力学精度。

**Few-shot 适应(仅优化任务潜变量 $c$,25 条目标任务示范)**:动作监督设置下,对比 $\pi_{0.5}$(LoRA 微调)和 UVA(动作监督微调),UMA 在 Insertion 上打平最强基线,在 Sweeping/Folding 上高出 10-25 个百分点,而 UMA 只更新一个低维任务潜变量,不更新模型权重。纯运动监督设置(仅用人类视频提供参考运动,无动作标签)下,UMA 比同样条件微调的 UVA(像素重建损失)在每个任务上高约 25 个百分点。

**消融**:去掉对比损失 $\mathcal L_c$,两种监督体制下 few-shot 成功率均下降 30-60 个百分点(任务潜变量退化为记住具体参考运动、无法迁移);仿真三任务上的架构消融显示,Action-only(不预测运动)明显落后于完整模型(尤其工具使用任务),独立解码头(Separate Heads)优于 action-only 但仍逊于共享头,同预算下的 dense attention 也不如论文的结构化分解注意力。失败分析显示 82% 的错误来自执行阶段(抓取后动作不精确),仅 18% 来自抓取失败,说明模型已能正确识别任务意图和抓取配置,瓶颈在动作 rollout 的精度。附录的规模分析(数据量 25/50/100% × 模型参数 150M/300M/600M)显示数据量和参数量都对成功率有贡献,全数据 600M 参数配置成功率最高(约 81%)。

## 四、评价与展望

**优点**:把物体运动定位为"输入/输出双角色"的共享变量而非单一方向的条件或目标,是一个简洁但抓住要害的设计——它直接解决了"动作标签稀缺 vs. 视频数据海量"的错位问题,使同一套目标函数能吃下三种异构数据源。soft prompt tuning 只更新任务潜变量、不动模型权重的 few-shot 适应方式,相比 LoRA 类全模型/子模块微调更轻量,也天然支持"完全无机器人数据"的纯视频适应,这是相对于 COIL、UVA、$\pi_{0.5}$ 等基线的一个实用性优势。Masked DiT block 用双 adaLN 头解决"同一 transformer 前向内条件 token 与目标 token 调制需求冲突"的问题,是对标准 DiT 的一个针对性且轻量的改动,消融也证实其收益(优于 dense attention 和独立解码头)。

**局限与开放问题**:论文自陈的三点限制值得关注——(1) 尽管数据吸收是跨 embodiment 的,但部署到新机器人本体仍需要用该本体的配对数据监督一个 embodiment-specific 的动作头,并未真正做到"训练一次、任意本体零样本部署";(2) 所有表示都活在逐 episode 的相机坐标系下,部署时依赖标定好的相机-基座外参,限制了在无标定/移动相机场景下的可用性;(3) 每步预测只依据单帧观测,没有利用多步历史或跨帧稠密点对应,这本可以是该 masked 联合建模框架自然能吸收的额外监督信号。此外,论文的评测任务集较小(3 个真实任务、20 次试验/任务),基线均为与 UMA 同数据训练的自研或近期公开方法(COIL、UVA、PointWorld、$\pi_{0.5}$),尚缺少与更广谱的通用 VLA(如 RT-2、OpenVLA、$\pi_0$)在同一测试床上的直接对比,难以判断"运动作为共享接口"相对于"直接端到端动作生成"的边际收益在更大规模、更多任务上是否依然成立。对比损失对 few-shot 迁移的贡献(30-60pp)提示任务潜变量的解耦质量是当前性能的关键杠杆,如何在更大任务分布上验证 $c$ 的语义一致性,以及去噪骨干本身是否也需要类似的正则,是值得跟进的方向。
