## 一句话总结

X-WAM 在统一世界动作模型（Unified World Action Model, UWM）中首次显式引入 3D 空间信息：通过复制预训练 DiT 最后几层构造一条"单向注意力"深度分支实现轻量 4D 建模，并用耦合噪声采样（Asynchronous Noise Sampling, ANS）对齐训练与推理的去噪分布，使动作能以远少于视频的步数被快速解码；在 RoboCasa（79.2% 对比最强基线 Cosmos Policy 的 67.1%）和 RoboTwin 2.0（Clean/Randomized 89.8%/90.7%，超过 Motus 的 88.7%/87.0%）上都取得当前最优成功率，同时在 4D 重建质量（PSNR/Chamfer Distance 等）上也全面超过两阶段"视频生成+后验深度估计"方案。

## 一、问题与动机

具身通用模型目前分裂为两条并行路线：一是 Vision-Language-Action（VLA）策略模型，擅长指令跟随但缺乏对物理动态如何演化的几何直觉；二是世界模型（World Model），擅长生成逼真的未来视觉预测但不直接产出可执行动作。近期一批"统一世界动作模型"（UWM、Motus、Cosmos Policy、DreamZero、LingBot-VA、GigaWorld-Policy 等）尝试用同一套视频扩散基座联合建模未来观测与动作，但作者指出这些工作仍留有两个共性缺陷：

1. **仍局限于 2D 像素空间**，缺乏显式 3D 空间感知。由于物理世界本质是三维的，纯 2D 建模会剥离关键几何结构，导致模型对未来的想象产生"物理不合理的幻觉"，也无法给出几何一致的 3D 重建。
2. **训练/推理的去噪分布不匹配**。已有工作为兼顾视频生成质量与动作解码效率，在训练阶段对视频、动作两种模态的噪声时间步做完全独立的随机采样；但这会产生推理阶段根本不会出现的组合（例如视频噪声比动作噪声还干净），限制了异步去噪能带来的效率收益。

X-WAM 的目标是把统一世界动作模型从"2D 像素预测器"升级为"具备空间感知的 4D 动力学模拟器"，同时兼顾四个目标：高保真视频生成、3D 空间重建、高策略成功率、高效动作执行。

## 二、核心方法

X-WAM 以预训练视频生成 DiT **Wan2.2-TI2V-5B** 为基座微调。给定语言指令 $c$、初始本体状态 $s_0$、多视角初始 RGB 观测 $O_0$，模型联合预测未来 RGB 视频 $O_{1:H}$、深度视频 $D_{1:H}$、本体状态 $s_{1:H}$ 与动作 $a_{1:K}$（其中 $H=8$，$K=32$，即动作预测频率是视频/状态的 $4$ 倍，反映动作控制需要更高时间分辨率的先验）。

**1）统一去噪序列。** RGB 用因果 VAE 编码器 $\mathcal{E}$ 得到 latent，状态/动作各自经可学习 MLP 投影到同一 latent 空间，再拼接成一条序列：

$$\mathbf{Z} = [\mathbf{z}_{O_0}, \mathbf{z}_{O_{1:H}}, \mathbf{z}_{s_0}, \mathbf{z}_{s_{1:H}}, \mathbf{z}_{a_{1:K}}]$$

用大白话说：把"生成未来视频"和"预测未来动作/状态"看成同一个去噪任务里的不同 token 段，用一个带双向全注意力的 DiT 统一处理，条件帧 $O_0$、初始状态 $s_0$ 全程视为噪声时间步为 0 的干净样本。为兼容多视角输入，X-WAM 给每个视角的 token 额外加可学习视角嵌入，而不破坏预训练的 3D RoPE 位置编码；状态/动作则复用视频的时序 RoPE，使模型能通过位置邻近性推断状态/动作与视频帧的时序对应关系。

**2）相机位姿的简化建模。** 机械臂上的相机分两类：静态（第一/第三人称视角，位姿全程不变）与动态（腕部相机，位姿由固定手眼标定矩阵决定）。X-WAM 不显式预测相机外参 token，而是直接用模型本来就要预测的末端执行器位姿反推腕部相机位姿：

$$\mathbf{T}_{\text{wrist}} = \mathbf{T}_{\text{ee}} \cdot \mathbf{T}_{\text{h2e}}$$

大白话：腕部相机位姿 = 末端执行器位姿 $\times$ 固定手眼标定矩阵，省去一路冗余的相机外参预测分支，同时保证多视角深度能被正确融合成统一 3D 点云。

**3）轻量深度自适应（Lightweight Depth Adaptation）。** 若把深度图沿序列维度拼接为额外 token 会使序列长度翻倍、注意力开销平方增长；若沿通道维度融合则会让输入分布偏离预训练 manifold，增大学习难度。X-WAM 的做法是复制预训练 DiT 的最后 $M$ 个 block（$M=10$）构造一条独立的深度预测分支，与主干（RGB/state/action）"交错"执行。设共享的前 $N-M$ 层输出隐状态 $\mathbf{H}$，令 $\mathbf{Z}_D^{(0)} = \mathbf{Z}_m^{(0)} = \mathbf{H}$，第 $j$ 层更新为：

$$\mathbf{Z}_D^{(j)} = \text{DepthBlock}_j\left(\mathbf{Z}_D^{(j-1)} \mid \mathbf{Z}_m^{(j-1)}\right), \qquad \mathbf{Z}_m^{(j)} = \text{DiTBlock}_{N-M+j}\left(\mathbf{Z}_m^{(j-1)}\right)$$

其中深度分支通过 cross-attention 在同一层读取主干分支的输入 $\mathbf{Z}_m^{(j-1)}$，而主干分支完全不受深度分支影响，作者称之为"单向注意力"（unilateral attention）。用大白话说：深度信息像一条"寄生"在主干旁边的旁路，只单向"偷看"主干的隐状态来推测深度，但绝不反过来干扰、拖慢或改变主干本身，从而在不增加序列长度、不破坏预训练视觉先验的前提下注入 3D 空间信息。深度图本身按逆深度用 MSE 回归监督，且和 RGB 共享同一个 VAE（单通道深度沿通道维复制 3 份伪装成 RGB 图送入/解出 VAE）。推理时该深度分支可按需灵活开关，不参与每一步去噪，从而不显著增加 rollout 开销。

**4）异步噪声采样（Asynchronous Noise Sampling, ANS）。** 核心洞察：合成高维视频轨迹需要很多去噪步数，而低维动作可以在噪声很大的视频 latent 条件下依然被较准确地恢复。因此推理阶段，X-WAM 为视频分配 $T_O$ 步、为状态/动作分配 $T_a$ 步去噪（$T_a < T_O$），两者都从纯噪声出发、以 $1/T_a$、$1/T_O$ 的步长独立前进；$T_a$ 步后动作即可解码完成并立即下发执行，此后可选择继续跑完剩余 $T_O - T_a$ 步来渲染高保真视频（此时动作已作为"干净模态"参与条件生成，过程自然退化为一个动作条件世界模型）。

为使训练分布匹配这种推理行为，X-WAM 摒弃"视频、动作各自独立随机采噪"的旧做法，改为耦合联合采样：

$$(t_O, t_a) \sim \begin{cases} t_a = 0,\ t_O \sim U(0,1) & \text{以概率 } p \\ t_a \sim U(0,1),\ t_O = t_a + (1-t_a)\cdot b,\ b \sim \text{Beta}(1.5,1) & \text{以概率 } 1-p \end{cases}$$

大白话说：以概率 $p$ 让动作直接是干净的（$t_a=0$），训练"动作条件下生成视频"；以概率 $1-p$，先随机采一个动作噪声水平 $t_a$，再把视频噪声水平 $t_O$ 强制限制在 $[t_a, 1]$ 区间内（用 Beta(1.5,1) 分布使其偏向更高噪声），从而保证任何时刻视频都不会比动作更"干净"——这与推理时"动作先去噪完、视频继续去噪"的顺序完全对齐，消除了训练/推理分布失配的问题。

**5）训练目标。** 沿用 flow matching 框架，网络对每个模态在噪声时间步 $t_m$ 上预测速度场：

$$\mathcal{L}_m = \left\| f_\theta^m(\mathbf{z}_m^{t_m}, t_m) - (\boldsymbol{\epsilon}_m - \mathbf{z}_m^0) \right\|^2$$

深度分支用逆深度 MSE 回归监督，总损失为四路加权和（预训练阶段 $\lambda_s=\lambda_a=\lambda_D=1.0$）：

$$\mathcal{L}_{\text{total}} = \mathcal{L}_O + \lambda_s \mathcal{L}_s + \lambda_a \mathcal{L}_a + \lambda_D \mathcal{L}_{\text{depth}}$$

**训练规模与数据。** 预训练数据汇总真机 + 仿真共 1,492,026 条轨迹、5,873.9 小时（AgibotWorld-Beta 866,562 条/2,221.5h 真机；DROID 74,734 条/280.3h 真机；InternA1-Aloha/Genie1/Lift2、RoboCasa MimicGen、RoboTwin 2.0 五个仿真数据源），深度标注全部由 Video Depth Anything 自动抽取（伪标签）。预训练用 256 块 NVIDIA H20，batch size 2048，AdamW，峰值学习率 $1\times 10^{-4}$，1000 步 warmup + cosine decay，训练 40,000 步；深度分支复制层数 $M=10$，ANS 动作条件概率 $p=0.5$。基准微调用 32 块 H20，batch 128，学习率 $3\times10^{-5}$，20,000 步。推理阶段用 UniPC 多步调度器，$T_a=10$、$T_O=50$，CFG scale 设为 1.0（更大的 guidance scale 不提升动作质量反而增加推理开销）。

## 四、局限性

论文附录 E 明确列出两点：

1. **上下文窗口固定、无历史/自回归机制**：当前框架只处理固定长度的观测窗口，不像 DreamZero 那样用 KV cache 扩展时序上下文，在需要消歧任务阶段的长时程场景中可能因缺乏历史信息而做出次优决策。
2. **推理延迟仍高于专用策略模型**：作为联合生成高维视频与低维动作的统一模型，X-WAM 的单步推理成本高于专用 VLA 与轻量 WAM（如 Fast-WAM）；即使用 8 步异步去噪 + RTC，单动作块仍需约 300ms，机器人执行的是若干帧之前计算出的预测，可能因此在快速反应场景下降低策略表现。

此外，从实验设计角度看仍有若干未覆盖之处：真机对比只用了一个自研 VLA 基线（XR-0）和一次失败的 $\pi_{0.5}$ 复现尝试，未与其他统一 WAM（如 Motus、GigaWorld-Policy）做真机对比；深度监督全部依赖 Video Depth Anything 生成的伪标签而非真实深度传感器数据，伪标签误差可能上限了重建精度；消融只在 RoboCasa 单一基准上做，未验证在 RoboTwin 2.0 或真机上的一致性。

## 五、评价与展望

**优点**：X-WAM 的核心贡献可以概括为"用最小的架构改动把 3D 感知塞进已经很强的 2D 视频扩散先验里"。轻量深度自适应分支（复制末尾几层 + 单向 cross-attention）是一个工程上很克制的设计：它不像"序列拼接深度 token"那样让计算量随视角数/时序长度平方增长，也不像"通道拼接"那样破坏预训练分布，消融实验（Table 4a）也确实证明了这一点——它在与无深度基线完全相同的延迟下拿到了接近序列拼接方案的质量。ANS 的耦合噪声采样思路同样值得称道：它指出了此前 UWM/DreamZero/Motus 等工作里一个容易被忽视的训练-推理不一致问题（独立采样会产生"视频比动作更干净"这种推理时不会出现的组合），并给出了一个用 Beta 分布实现的、简洁的联合分布重参数化方案，用消融（Table 4b）直接证明了它相对"解耦训练+异步推理"的收益。

**与其他公开工作的关系**：X-WAM 明确定位自己是在 UWM、DreamZero、Motus、Cosmos Policy 等统一世界动作模型谱系之上做的下一步——那批工作已经证明"视频生成 + 动作预测"共享一个扩散/Transformer 主干是可行的，但普遍停留在 2D 像素空间；X-WAM 则是该谱系中较早把显式 3D/4D 重建也纳入同一框架、并系统性讨论训练-推理噪声分布对齐问题的工作。相较于走"3D 表示原生建模"路线的 ManiGaussian、GWM（基于 3D Gaussian Splatting），X-WAM 选择了更轻量的"2D 视频 latent + 深度伪标签监督"路线，代价是重建保真度可能不如原生 3D 表示，但换来了能直接复用强力预训练视频基座（Wan2.2-5B）先验的优势，这是一个典型的"数据/算力效率 vs 表示原生性"权衡。

**开放问题与可能的改进方向**：
1. 深度监督完全依赖 Video Depth Anything 的伪标签，尚未验证在有真实深度传感器（如结构光/ToF）标注的数据上重新训练是否能进一步提升重建上限，也未讨论伪标签噪声对策略成功率的潜在负面影响。
2. 论文本身承认的长上下文缺失问题，是否可以像 DreamZero 一样引入 KV cache 或历史 token，同时不破坏 ANS 的耦合噪声采样假设，是一个值得深挖的架构问题——历史条件化本身可能改变"视频噪声始终不低于动作噪声"这一假设成立的边界条件。
3. 推理延迟仍是统一模型对比专用策略模型的结构性短板；论文提出的蒸馏、一致性模型（consistency model）等方向若能与 ANS 结合，理论上可以进一步压缩 $T_a$，但这可能进一步牺牲视频质量或动作精度，两者之间的帕累托前沿尚未在论文中给出系统刻画。
4. 真机评测目前局限在单一双臂平台、单一长时程任务（耳机装盒），且只对比了一个 VLA 基线，观察到的"长时程可靠性优势""泛化优势"结论的稳健性还需要更多任务、更多基线（尤其是其他统一 WAM）的真机验证来支撑。
