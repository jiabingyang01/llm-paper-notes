## 一句话总结

PAD 用统一的 Diffusion Transformer（DiT）把未来图像帧（以及可选的深度图）与机器人动作拼接为同一组 token，在同一个扩散去噪过程中联合预测，从而让预训练图像扩散先验直接反哺动作生成；在 MetaWorld 全 50 任务上单一视觉-语言条件策略取得 72.5% 平均成功率，相对最强基线 GR-1（57.4%）提升 26.3%，真实 Panda 机械臂上对未见任务的成功率相对最强基线提升 28.0%。

## 一、问题与动机

预测未来（image prediction）和执行动作（robot action）看似是两种能力，但作者认为二者高度相关，因为它们共享同一套物理世界动力学：能准确预测未来画面的模型，理应也更懂得如何生成合理的动作。已有两条技术路线各自独立：

- **扩散生成模型**（Diffusion Policy 之外的图像/视频扩散模型）在图像编辑、视频生成上表现出对物理规律的良好理解，但只用于生成图像，不直接产生动作；
- **扩散策略**（Diffusion Policy 等）把动作生成建模为去噪过程，但通常从零训练，没有利用图像扩散模型中蕴含的丰富视觉先验。

此前将两者结合的工作（如 UniPi、SuSIE）采用"两阶段"范式：先用预训练图像编辑/生成模型合成目标图像，再学习一个以该目标图像为条件的低层策略。作者指出这种两阶段方式的问题：预测和动作学习被割裂，动作模型无法利用预测模型内部编码的丰富表征，且下游策略效果强依赖于生成图像的质量。

PAD 的核心问题是：**能否用同一个去噪过程同时完成图像预测和动作生成，让两者共享同一套网络权重和中间表征，而不是先生成图像再喂给下游策略？** 由此带来三个关键收益：（1）利用预训练 DiT 的图像先验直接加速/提升动作学习；（2）该联合框架可以在缺失动作标签的大规模网络视频数据（如 BridgeData-v2）上协同训练，因为 DiT 的自注意力机制天然支持通过 mask 跳过缺失模态；（3）架构可以灵活扩展到深度图等其它模态。

## 二、核心方法

### 2.1 问题设定与预备知识

机器人数据集 $D_{robot}=\{\zeta_1,\dots,\zeta_n\}$，每条演示 $\zeta_i=(I_i,l_i,\tau_i)$ 包含语言指令 $l_i$、像素观测序列 $I_i$ 和机器人轨迹 $\tau_i=\{p_i^{1:T}\}$。此外引入无动作标注的网络视频数据集 $D_{video}=\{\zeta_j=(I_j)\}$。

PAD 建立在 latent diffusion 之上。设 $z_0=\varepsilon(x_0)$ 是真实数据编码得到的 latent，前向加噪过程为

$$
q(z_t\mid z_{t-1})=\mathcal N\!\left(z_t;\sqrt{\alpha_t}\,z_{t-1},(1-\alpha_t)\mathbb I\right),
$$

记 $\bar\alpha_t=\prod_{i=1}^t\alpha_i$，则 DDPM 给出 $z_t=\sqrt{\bar\alpha_t}z_0+\sqrt{1-\bar\alpha_t}\epsilon_t$。反向去噪过程（论文式 1）为

$$
p(z_{t-1}\mid z_t,c)=\mathcal N\!\left(z_{t-1};\sqrt{\bar\alpha_{t-1}}\,\mu_\theta(z_t,t,c),(1-\bar\alpha_{t-1})\mathbb I\right),\quad
\mu_\theta(z_t,t,c)=\big(z_t-\sqrt{1-\bar\alpha_t}\,\epsilon_\theta(z_t,t,c)\big)/\sqrt{\alpha_t}.
$$

噪声估计网络 $\epsilon_\theta$ 用来逼近数据分布对数密度的梯度（式 2）：

$$
\epsilon_\theta(z_t,t,c)\approx -\sqrt{1-\bar\alpha_t}\,\nabla_{z_t}\log p(z_t\mid c).
$$

**大白话说**：扩散模型训练一个网络，学会在任意噪声强度 $t$ 下，把加了噪声的样本"往干净方向"修正一点点；这个修正方向本质上就是数据分布的得分函数（score）。PAD 把这套机制同时用在图像 latent 和机器人动作 latent 上。

### 2.2 联合预测框架

PAD 同时建模三类模态：RGB 图像模态 $M_I$、机器人动作模态 $M_A$（末端执行器位置、姿态与夹爪状态构成的位姿），以及可扩展的额外模态 $M_E$（论文中用深度图实例化）。预测和动作都以当前多模态观测 $c_I,c_A,c_E$（RGB、机器人位姿、深度）和语言指令 $l$ 为条件，模型一次输出未来 $k$ 步的预测 $x_I^{1:k},x_A^{1:k},x_E^{1:k}$，其中只有第一步动作 $x_A^1$ 会被真正执行，随后触发下一轮闭环重规划——即模型做的是 $k$ 步的短期规划，但只执行第一步。

**输入编码**：RGB 图像 $x_I$ 先经冻结的 VAE 编码器 $\varepsilon_I$ 得到 latent，再 patchify 成 token 序列 $t_I$；机器人位姿 $x_A$ 经 MLP 编码器 $\varepsilon_A$ 并线性变换为与图像 token 同维度 $h$ 的 token $t_A$；深度图（若存在）同理编码为 $t_E$；语言指令经冻结 CLIP 编码器得到文本嵌入 $c_l$。

**条件生成的拼接方式**：将条件 latent 与噪声 latent 沿通道维拼接，例如图像分支 $L_I=[\varepsilon_I(c_I),z_t^I]$，动作分支 $L_A=[\varepsilon_A(c_A),z_t^A]$，深度分支 $L_E=[\varepsilon_E(c_E),z_t^E]$；若图像编码 latent 形状为 $c\times d\times d$，堆叠 $k$ 个未来帧的噪声 latent $z_t^I$ 形状为 $kc\times d\times d$，拼接后 $L_I$ 形状为 $(k+1)c\times d\times d$。

拼接后的各模态 latent 分别 tokenize（图像走与 DiT 相同的 patchify，动作走简单线性投影），再把三种模态的 token 序列**在序列维度上拼接**，送入多层 DiT block 做**联合自注意力去噪**，最终解码出未来帧和未来动作的 latent 表示。

**大白话说**：把"现在的图像/位姿/深度"当条件，把"未来的图像/位姿/深度加噪声"当待去噪目标，图像和动作被切成同一批 token 塞进同一个 Transformer，一次去噪同时把图像的噪声和动作的噪声都去掉——图像 token 和动作 token 互相做注意力，图像里学到的物理规律因此可以直接"渗透"进动作预测。

### 2.3 缺失模态与视频协同训练

当输入数据缺少某个模态时（例如网络视频数据集没有机器人动作），DiT 输入直接省去对应 token（如只保留 $t_I$），并在自注意力模块中引入 **masked-attention**，把无效/padding 部分的 token 从输出中排除，只保留有效预测结果。这一设计使 PAD 能同时在纯 RGB 视频数据集（BridgeData-v2）与带动作标注的机器人数据集上联合训练。

### 2.4 初始化与训练目标

PAD 权重从 ImageNet 类别条件图像生成任务预训练的 DiT 初始化：丢弃原有类别标签嵌入层，为文本嵌入新增层做零初始化，把图像 latent tokenizer 权重复制 $k+1$ 份以编码堆叠的未来帧，机器人状态的编码器/解码器同样零初始化。

训练目标是同时预测三种模态各自加入的噪声 $\epsilon^I,\epsilon^A,\epsilon^E$，用 DDPM 损失（式 3）：

$$
\mathcal L_{diff}^{\delta}(\theta)=\mathbb E_{\epsilon^\delta\sim\mathcal N(0,1),t,C,l}\Big[\big\|\epsilon^\delta-\epsilon_\theta^\delta(z_t^\delta,t,C,l)\big\|_2^2\Big],\quad \delta\in\{I,A,E\},
$$

总损失为三者加权和（式 4）：

$$
\mathcal L(\theta)=\lambda_I\mathcal L_{diff}^{I}+\lambda_A\mathcal L_{diff}^{A}+\lambda_E\mathcal L_{diff}^{E}.
$$

**大白话说**：三个模态各算各的"去噪 MSE 损失"，再按权重加起来一起训练；训练时图像损失权重 $\lambda_I$ 全程保持 1.0（因为预训练已经打好图像先验），$\lambda_A,\lambda_E$ 在适配阶段（后 10 万步）从 0 线性升到 2.0，让模型先巩固图像预测能力，再逐步侧重动作精度。

### 2.5 训练与推理细节

- **两阶段训练**：先在 BridgeData-v2（约 6 万条轨迹）上预训练 200k 步建立图像预测先验，再针对每个具体机器人域（MetaWorld、真实 Panda）用机器人演示继续适配训练 100k 步；整个预训练+适配过程在 4 张 NVIDIA A100 上约需 2 天 1 小时。
- **预测/执行**：预测步长 $k=3$，帧间隔 $i=4$；策略执行时用 75 步 DDIM 采样得到 $k$ 步未来图像和动作，只执行第一步预测动作，随后用简单的线性插值运动规划器把机械臂移动到目标位姿，再触发下一轮预测循环（闭环控制）。

## 三、实验结果

实验围绕三个问题展开：（1）联合预测能否在数据有限时提升策略学习；（2）能否借助大规模网络视频协同训练提升泛化；（3）扩大计算量能否进一步提升性能。评测环境为 MetaWorld 仿真基准（50 个任务，单一视觉-语言条件策略，每任务 50 条演示，"corner2" 相机视角，4 维状态：末端位置+夹爪）和真实世界 Panda 机械臂操作（6 类任务：按按钮、开抽屉、拉线缆、抓取摆放等，每任务 200 条遥操作演示，腕部相机，7 维位姿）。

### 3.1 MetaWorld 全 50 任务（Table 1 / Table 5 节选）

| 方法 | 平均成功率（50 任务） |
|---|---|
| Diffusion Policy | 27.9% |
| SuSIE | 41.0% |
| RT-1 | 34.6% |
| RT-2*（复现） | 52.2% |
| GR-1 | 57.4% |
| **PAD（ours）** | **72.5%** |
| PAD w/o img（去掉图像预测） | 43.6% |
| PAD w/o co-train（不用视频协同训练） | 59.2% |

PAD 相对最强基线 GR-1 的绝对提升为 15.1 个百分点，相对提升 26.3%，与摘要给出的数字一致。部分典型任务（如 assembly、basketball、peg-insert-side、pick-place-wall 等偏"精细操作"任务）PAD 优势尤为明显（如 basketball-v2 上 PAD 0.84 vs 次优 SuSIE 0.24；assembly-v2 上 PAD 0.88 vs 次优 RT-2* 0.24）。

### 3.2 真实世界 Panda——已见任务（Table 2）

| 方法 | 平均成功率（6 类任务，每任务 50 次 rollout） |
|---|---|
| Diffusion Policy | 0.38 |
| SuSIE | 0.49 |
| RT-1 | 0.43 |
| RT-2*（复现） | 0.69 |
| **PAD（ours）** | **0.72** |
| **PAD-Depth（ours，含深度模态）** | **0.78** |

引入深度模态后进一步提升成功率（0.72→0.78），验证了框架对额外模态的可扩展性。

### 3.3 真实世界未见任务泛化（Figure 5，三档难度）

专家数据集仅含彩色方块和圆盘，测试时引入此前未见过的水果/蔬菜玩具及新背景，设三档难度：easy（1–4 个干扰物）、middle（5–15 个干扰物）、hard（未见物体+5–15 个干扰物或未见背景）。PAD 在难度递增时的相对优势持续扩大，成功处理草莓、胡萝卜、茄子等分布外物体及新背景，而基线方法在 hard 档基本失效；摘要给出的整体数字是相对最强基线 **28.0% 的成功率提升**。

### 3.4 消融实验

- **去掉图像预测**（PAD w/o img）：MetaWorld 平均成功率从 72.5% 降到 43.6%，说明图像预测分支提供的物理知识对动作学习是关键增益，而非可有可无的辅助任务；单纯预测低维位姿容易过拟合训练数据。
- **去掉视频协同训练**（PAD w/o co-train）：平均成功率从 72.5% 降到 59.2%，且预测图像质量明显下降（论文 Figure 8 显示未协同训练时预测图像中物体细节缺失，如蓝色方块消失）。
- **深度模态**：如上，带来真实任务成功率的进一步提升。

## 四、局限性

论文在结论部分明确指出两点局限：

1. **模态种类有限**：目前只验证了 RGB 图像、机器人动作、深度图三类模态，触觉等其它模态留待未来工作。
2. **控制频率不高**：由于需要联合去噪图像和动作（图像去噪发生在高维像素/latent 空间，计算代价高），PAD 的闭环控制频率受限；作者提出未来可探索直接利用预测图像的中间 latent 表征而非高维像素空间来加速。

此外，从实验设计看还存在几点可议之处（论文未明说，但可从数据中读出）：真实机器人实验和 MetaWorld 实验的性能提升幅度存在差异（MetaWorld 上相对提升 26.3%，真实已见任务上 PAD 相对 RT-2* 只提升约 4 个百分点），说明联合预测的收益在仿真高保真渲染环境下可能被放大；RT-2* 为按论文规格自行复现（用 InstructBlip-7B 替代不可获取的 PaLM 主干），GR-1 也是基于作者开源推理代码自行训练，两者复现的公平性依赖于复现质量，无法完全排除复现差异对比较结果的影响。

## 五、评价与展望

**优点**：PAD 的核心贡献是把"图像预测"和"动作生成"两个此前分离的扩散过程，通过 DiT 的 token 拼接 + 掩码注意力机制真正融合进同一次去噪迭代，而不是像 UniPi/SuSIE 那样先生成目标图像再训练一个独立的逆动力学/低层策略。这一设计选择带来两个直接好处：其一，动作分支可以复用 DiT 内部为图像预测学到的中间表征，而不仅仅把生成图像当作外部条件信号；其二，masked-attention 使框架天然支持异构数据协同训练（有动作标注的机器人数据 + 只有 RGB 的网络视频），为利用互联网规模视频数据学习操作先验提供了一条架构上干净的路径。消融实验（w/o img、w/o co-train）也确实证明了这两个机制各自独立贡献了显著增益，而不只是堆叠工程技巧。

**与其他公开工作的关系**：PAD 与 GR-1（自回归架构同时生成图像和动作）构成直接对比，论文的定性结果（Figure 6）显示 PAD 预测的未来图像比 GR-1 更清晰精确，作者将其归因于扩散模型相较自回归模型在图像生成任务上的固有优势；与 UniPi、SuSIE 的两阶段路线相比，PAD 属于"单阶段联合建模"路线，这是继 GR-1 之后该路线的又一个扩散版本实现，二者共同代表了"预测即策略"（predict-then/while-act）研究方向从自回归到扩散的演进。

**开放问题与可能的改进方向**：（1）论文承认的控制频率瓶颈是该类"联合预测扩散策略"共同面临的实际部署障碍，如何在不牺牲图像先验收益的前提下把去噪压缩到更低维、更快的空间，是后续值得跟进的方向；（2）当前 $k=3$ 步的预测/单步执行范式仍是相对粗粒度的重规划频率，是否可以引入图像预测置信度或不确定性估计来自适应决定重规划间隔，论文未涉及；（3）Figure 7 展示了模型在 Bridge 数据集上预测未来时会产生"幻觉"（如把 banana 想象成 pear），这种预测不确定性目前只是被展示而未被显式建模或用于策略决策（如加权、拒绝执行等），是一个可深入的开放问题；（4）深度模态的加入方式仍是与 RGB/动作对称的独立 token 分支，未来可探索模态间更紧密耦合的几何一致性约束（例如图像与深度联合渲染一致性）以进一步提升样本效率。
